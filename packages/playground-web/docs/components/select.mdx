# Select

An accessible dropdown select component built on Radix UI primitives with full keyboard navigation, search functionality, and Tekton design token integration.

## Overview

The Select component provides a rich dropdown selection experience. Built on Radix UI's Select primitive, it offers advanced features like keyboard navigation, grouping, disabled items, and proper accessibility support.

**Key Features:**
- Full keyboard navigation (arrow keys, type-ahead, Enter/Space)
- Grouping support with labels
- Disabled items support
- Scrollable long lists with scroll buttons
- Portal rendering for proper z-index stacking
- Animated open/close transitions
- Composable API with 9 sub-components
- TypeScript support with full type inference

## Installation

```bash
npm install @framingui/ui
```

## Basic Usage

```tsx
import {
  Select,
  SelectContent,
  SelectItem,
  SelectTrigger,
  SelectValue,
} from '@framingui/ui';

export default function Example() {
  return (
    <Select>
      <SelectTrigger className="w-[180px]">
        <SelectValue placeholder="Select a fruit" />
      </SelectTrigger>
      <SelectContent>
        <SelectItem value="apple">Apple</SelectItem>
        <SelectItem value="banana">Banana</SelectItem>
        <SelectItem value="orange">Orange</SelectItem>
      </SelectContent>
    </Select>
  );
}
```

## Props/API

### Select

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | `undefined` | Controlled selected value |
| `defaultValue` | `string` | `undefined` | Uncontrolled default value |
| `onValueChange` | `(value: string) => void` | `undefined` | Callback when value changes |
| `disabled` | `boolean` | `false` | Disable entire select |
| `name` | `string` | `undefined` | Form field name |
| `required` | `boolean` | `false` | Required field validation |

### SelectTrigger

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | `undefined` | Additional CSS classes |
| `children` | `ReactNode` | `undefined` | Trigger content (typically SelectValue) |
| `id` | `string` | `undefined` | HTML id for label association |

### SelectValue

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `placeholder` | `string` | `undefined` | Text shown when no value selected |

### SelectContent

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | `undefined` | Additional CSS classes |
| `children` | `ReactNode` | `undefined` | Select options |
| `position` | `'popper' \| 'item-aligned'` | `'popper'` | Positioning strategy |

### SelectGroup

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | `undefined` | Grouped items |

### SelectLabel

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | `undefined` | Additional CSS classes |
| `children` | `ReactNode` | `undefined` | Label text |

### SelectItem

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | **required** | Item value (must be unique) |
| `disabled` | `boolean` | `false` | Disable this item |
| `className` | `string` | `undefined` | Additional CSS classes |
| `children` | `ReactNode` | `undefined` | Item display text |

## Best Practices

### ✅ DO

- Always use SelectValue with placeholder for clear UX
- Associate selects with labels using `htmlFor` and `id`
- Use SelectGroup and SelectLabel for categorizing many options
- Provide clear, concise option text (avoid very long labels)
- Use controlled state when select value affects other UI

### ❌ DON'T

- Don't use selects for fewer than 3 options (use radio buttons instead)
- Avoid nested selects or very deep option hierarchies
- Don't forget placeholder text (users need context)
- Avoid disabling items without explaining why
- Don't use select for navigation (use proper links/buttons)

## Accessibility Notes

- Select uses proper ARIA roles and attributes
- Keyboard navigation: Arrow keys to navigate, Enter/Space to select
- Type-ahead search: Type characters to jump to matching options
- Disabled items are properly marked with `aria-disabled`
- Selected item shows checkmark icon indicator
- Focus management handled automatically
- Screen reader announces selected value and total options

## Controlled vs Uncontrolled

### Controlled Select

```tsx
import { Select, SelectContent, SelectItem, SelectTrigger, SelectValue } from '@framingui/ui';
import { useState } from 'react';

export default function ControlledExample() {
  const [value, setValue] = useState('');

  return (
    <Select value={value} onValueChange={setValue}>
      <SelectTrigger className="w-[180px]">
        <SelectValue placeholder="Select..." />
      </SelectTrigger>
      <SelectContent>
        <SelectItem value="option1">Option 1</SelectItem>
        <SelectItem value="option2">Option 2</SelectItem>
      </SelectContent>
    </Select>
  );
}
```
