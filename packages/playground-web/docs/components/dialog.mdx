# Dialog

An accessible modal dialog component built on Radix UI primitives with full keyboard navigation, focus management, and Tekton design token integration.

## Overview

The Dialog component displays content in a layer above the main application. Built on Radix UI's Dialog primitive, it provides robust accessibility features including focus trapping, scroll locking, and proper ARIA attributes.

**Key Features:**
- Full keyboard and screen reader accessibility
- Automatic focus management and restoration
- Scroll lock when open
- Click outside to close
- ESC key to dismiss
- Composable API with 8 sub-components
- Animated entry/exit transitions
- Portal rendering for proper z-index stacking

## Installation

```bash
npm install @tekton-ui/ui
```

## Basic Usage

```tsx
import {
  Dialog,
  DialogContent,
  DialogDescription,
  DialogHeader,
  DialogTitle,
  DialogTrigger,
} from '@tekton-ui/ui';
import { Button } from '@tekton-ui/ui';

export default function Example() {
  return (
    <Dialog>
      <DialogTrigger asChild>
        <Button variant="outline">Open Dialog</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>Are you absolutely sure?</DialogTitle>
          <DialogDescription>
            This action cannot be undone.
          </DialogDescription>
        </DialogHeader>
      </DialogContent>
    </Dialog>
  );
}
```

## Variants

### Confirmation Dialog

Dialog for confirming destructive actions.

```tsx

  Dialog,
  DialogContent,
  DialogDescription,
  DialogFooter,
  DialogHeader,
  DialogTitle,
  DialogTrigger,
} from '@tekton-ui/ui';

export default function ConfirmationExample() {
  return (
    <Dialog>
      <DialogTrigger asChild>
        <Button variant="destructive">Delete Account</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>Delete Account</DialogTitle>
          <DialogDescription>
            Are you sure you want to delete your account? This action cannot be undone.
          </DialogDescription>
        </DialogHeader>
        <DialogFooter>
          <Button variant="outline">Cancel</Button>
          <Button variant="destructive">Delete</Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  );
}
```

## Props/API

### Dialog

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | `undefined` | Controlled open state |
| `defaultOpen` | `boolean` | `false` | Uncontrolled default open state |
| `onOpenChange` | `(open: boolean) => void` | `undefined` | Callback when open state changes |
| `modal` | `boolean` | `true` | Whether to render as modal with overlay |

### DialogTrigger

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | Render as child component |
| `children` | `ReactNode` | `undefined` | Trigger button content |

### DialogContent

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | `undefined` | Additional CSS classes |
| `children` | `ReactNode` | `undefined` | Dialog content |
| `onEscapeKeyDown` | `(event: KeyboardEvent) => void` | `undefined` | Custom ESC key handler |
| `onPointerDownOutside` | `(event: PointerEvent) => void` | `undefined` | Custom click outside handler |

### DialogHeader, DialogFooter

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | `undefined` | Additional CSS classes |
| `children` | `ReactNode` | `undefined` | Header/footer content |

### DialogTitle

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | `undefined` | Additional CSS classes |
| `children` | `ReactNode` | `undefined` | Title text |

### DialogDescription

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | `undefined` | Additional CSS classes |
| `children` | `ReactNode` | `undefined` | Description text |

## Best Practices

### ✅ DO

- Always include DialogTitle for accessibility (required by screen readers)
- Use DialogDescription to provide context for the dialog purpose
- Place primary action on the right in DialogFooter
- Use `asChild` on DialogTrigger to avoid nested buttons
- Implement proper form submission handlers in form dialogs

### ❌ DON'T

- Don't nest dialogs inside other dialogs (creates confusing UX)
- Avoid very large content without scroll handling
- Don't forget to handle dialog close in form submissions
- Avoid missing DialogTitle (accessibility violation)
- Don't use dialogs for simple notifications (use Toast instead)

## Accessibility Notes

- Dialog uses proper ARIA roles (`dialog`, `alertdialog`)
- DialogTitle uses `aria-labelledby` for screen readers
- DialogDescription uses `aria-describedby` for context
- Focus is trapped within dialog when open
- ESC key closes dialog (can be customized)
- Focus returns to trigger element on close
- Click outside overlay closes dialog (can be prevented)
- Close button has `sr-only` text for screen readers

## Controlled vs Uncontrolled

### Controlled Dialog

```tsx
import { Dialog, DialogContent, DialogTrigger } from '@tekton-ui/ui';
import { Button } from '@tekton-ui/ui';
import { useState } from 'react';

export default function ControlledExample() {
  const [open, setOpen] = useState(false);

  return (
    <Dialog open={open} onOpenChange={setOpen}>
      <DialogTrigger asChild>
        <Button>Open</Button>
      </DialogTrigger>
      <DialogContent>
        <p>Dialog content</p>
        <Button onClick={() => setOpen(false)}>Close</Button>
      </DialogContent>
    </Dialog>
  );
}
```
