# Card

A composable card container component with header, content, and footer sections. Built with Tekton design tokens for consistent styling and visual hierarchy.

## Overview

The Card component provides a flexible container for grouping related content. It's composed of multiple sub-components (CardHeader, CardContent, CardFooter, etc.) that can be combined to create various card layouts.

**Key Features:**
- Composable architecture with 6 sub-components
- Semantic HTML structure for accessibility
- Flexible layout options (header, content, footer)
- Consistent styling via Tekton tokens
- Built-in shadow and border styling
- Responsive design support

## Installation

```bash
npm install @framingui/ui
```

## Basic Usage

```tsx
import { Card, CardHeader, CardTitle, CardDescription, CardContent, CardFooter } from '@framingui/ui';
import { Button } from '@framingui/ui';

export default function Example() {
  return (
    <Card>
      <CardHeader>
        <CardTitle>Card Title</CardTitle>
        <CardDescription>Card description goes here.</CardDescription>
      </CardHeader>
      <CardContent>
        <p>Card content goes here.</p>
      </CardContent>
      <CardFooter>
        <Button>Action</Button>
      </CardFooter>
    </Card>
  );
}
```

## Best Practices

### ‚úÖ DO

- Use CardHeader for consistent title and description layout
- Combine CardFooter with Button components for actions
- Apply width constraints (e.g., `w-[350px]`) for predictable layouts
- Use CardDescription for secondary information below titles
- Group related cards in responsive grids for dashboard layouts

### ‚ùå DON'T

- Don't nest Card components inside each other (creates visual confusion)
- Avoid very wide cards without content constraints (hard to read)
- Don't put too much content in a single card (break into multiple cards)
- Avoid missing CardHeader when title is important for context
- Don't use Card for simple dividers (use Separator component instead)

## Accessibility Notes

- Card uses semantic `<div>` elements with proper hierarchy
- CardTitle renders as `<h3>` for proper heading structure
- Interactive cards should include `role="button"` and keyboard handlers
- Use sufficient color contrast for CardDescription text
- Ensure card content is keyboard navigable when interactive
