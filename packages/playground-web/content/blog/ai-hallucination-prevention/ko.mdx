## 당신이 몰랐던 AI 환각(Hallucination) 문제

Claude Code에게 버튼을 만들어달라고 요청하면 이렇게 생성합니다:

```tsx
<button className="bg-blue-600 hover:bg-blue-700 text-white px-6 py-3 rounded-lg">
  클릭하세요
</button>
```

다시 다른 컴포넌트를 요청하면 이번엔 이렇게 만듭니다:

```tsx
<button className="bg-indigo-500 hover:bg-indigo-600 text-white px-4 py-2 rounded-md">
  제출
</button>
```

**두 개의 버튼. 서로 다른 파란색. 다른 패딩. 다른 border radius.**

이것이 바로 UI에서의 AI 환각(hallucination)이며, 조용히 당신의 디자인 일관성을 파괴하고 있습니다.

## UI에서의 AI 환각이란?

AI 환각은 전통적으로 LLM이 사실을 지어내는 것을 의미합니다. 하지만 UI 코드 생성에서는 더 미묘하고 악랄한 형태가 있습니다:

**디자인 환각(Design Hallucination)** — AI가 실제 디자인 시스템과 무관하게 시각적 디자인 결정을 임의로 만들어내는 것.

### UI 환각의 세 가지 유형

1. **색상 환각** — AI가 임의의 Tailwind 색상을 선택 (`blue-600`, `indigo-500`, `sky-400`)
2. **간격 환각** — 일관성 없는 padding/margin (`p-4`, `px-5 py-3`, `gap-6`)
3. **패턴 환각** — 각 컴포넌트마다 고유한 일회성 구현을 생성

결과는? **AI가 생성한 모든 컴포넌트가 달라 보입니다.** 심지어 동일해야 하는 것들도요.

## 왜 이런 일이 발생할까? (기술적 진실)

LLM은 당신의 디자인 시스템을 "보지" 못합니다. 수백만 개의 오픈소스 저장소로 학습되었고, 각각은 완전히 다른 스타일을 가지고 있습니다.

당신이 이렇게 프롬프트하면:
```
primary 버튼을 만들어줘
```

AI의 내부 추론은 이렇습니다:
- "Primary"는 보통 파란색을 의미해
- 파란색은 `blue-500`, `blue-600`, `indigo-600`, `sky-500`...가 될 수 있어
- 학습 데이터에서 가장 자주 등장하는 `blue-600`을 선택할게
- 패딩은 `px-6 py-3`이 일반적이야... 하지만 `px-4 py-2`도 괜찮아

**근거가 없습니다.** AI는 통계적 패턴을 기반으로 추측하는 것이지, 당신의 실제 디자인 의도가 아닙니다.

## 디자인 환각의 숨겨진 비용

### 1. 천 개의 미세한 불일치로 인한 죽음

AI가 생성한 각 컴포넌트는 미묘한 차이를 도입합니다:
- 앱 전체에 15가지 다른 파란색
- 8가지 다른 버튼 크기
- 12가지 다른 border radius 값

시간이 지나면서, 앱은 마치 한 번도 만나지 못한 위원회가 디자인한 것처럼 보입니다.

### 2. 리팩토링 세금

환각된 모든 컴포넌트는 수동 수정이 필요합니다:
```tsx
// AI가 이렇게 생성
<div className="bg-blue-50 border border-blue-200 rounded-lg p-4">

// 당신은 이렇게 바꿔야 함
<div className="bg-[var(--color-background-secondary)] border border-[var(--color-border-default)] rounded-[var(--radius-default)] p-[var(--spacing-4)]">
```

**손실 시간:** 컴포넌트당 2-5분. 100개 이상의 컴포넌트 × 시간 = 몇 시간의 낭비.

### 3. 디자인 부채는 복리로 불어남

3개월 후, primary 파란색을 바꾸고 싶다면:
- 디자인 토큰 없이: 50개 이상의 하드코딩된 `blue-600` 값을 찾아서 바꾸기
- 디자인 토큰 사용: CSS 변수 하나만 변경

**가드레일이 없다면 AI 환각으로 인한 기술 부채는 영구적입니다.**

## FramingUI 솔루션: AI 가드레일로서의 디자인 토큰

FramingUI는 간단한 원칙으로 AI 환각을 해결합니다:

**AI가 디자인 토큰만 사용하여 코드를 생성할 수 있다면, 색상이나 스타일을 환각할 수 없습니다.**

### 작동 방식

1. **시맨틱 토큰이 원시 값을 대체**
   - `bg-blue-600` 대신 → `var(--bg-primary)`
   - `p-4` 대신 → `var(--spacing-4)`
   - `rounded-lg` 대신 → `var(--radius-lg)`

2. **타입 안전 컴포넌트 API**
   - Button은 `className="bg-red-500"`을 허용하지 않음
   - `variant="primary" | "secondary" | "destructive"`만 허용

3. **프로젝트 컨텍스트에 내장된 AI 지침**
   - `.claude/project-knowledge.md`가 AI에게 디자인 시스템을 가르침
   - 모든 프롬프트가 자동으로 토큰을 참조

- 🔴 5개의 임의 간격 값 (`p-6`, `p-8`, `gap-4`, `px-6 py-3`, `mt-6`)
- 🔴 3개의 임의 border radius (`rounded-xl`, `rounded-full`, `rounded-lg`)

**문제:** 다크모드, 브랜드 색상 변경, 간격 시스템 업데이트에 적응하지 못함.

### ✅ FramingUI 사용 (AI가 디자인 토큰 사용)

**같은 프롬프트:**
```
FramingUI를 사용해서 사용자 프로필 카드가 있는 설정 패널을 만들어줘

**AI 생성 결과:**

import { Card, CardHeader, CardContent, Avatar, Button } from '@framingui/ui';

export function SettingsPanel({ user }: SettingsPanelProps) {

            />
            <div>
              <h2 className="text-2xl font-bold">{user.name}</h2>

            </div>
          </div>

            프로필 수정

        </CardContent>
      </Card>

      <Card className="mt-[var(--spacing-6)]">
        <CardHeader>
          <h3 className="text-xl font-bold">환경설정</h3>

```

**환각 카운트:**
- ✅ 0개의 하드코딩된 색상 (모두 시맨틱 토큰 사용)
- ✅ 0개의 임의 간격 (모두 `var(--spacing-*)` 사용)
- ✅ 0개의 임의 border radius (Card 컴포넌트에서 상속)

**장점:** 테마 변경, 브랜드 업데이트, 반응형 브레이크포인트에 자동 적응.

## AI 환각 방지 설정하기

### 1단계: FramingUI 설치

```bash
pnpm add @framingui/ui @framingui/core
```

### 2단계: Claude Code용 프로젝트 지식 생성

`.claude/project-knowledge.md` 생성:

```markdown
# FramingUI 디자인 시스템 규칙

## 중요: 환각 방지 규칙

1. **절대 색상을 하드코딩하지 않기** — `var(--*)` 토큰만 사용
2. **Tailwind 색상을 임의로 만들지 않기** — `bg-blue-600`, `text-red-500` 등 금지
3. **항상 @framingui/ui에서 import** — 컴포넌트를 재생성하지 않기
4. **시맨틱 variant 사용** — `className="bg-blue-600"` 대신 `variant="primary"`

## 사용 가능한 디자인 토큰

### 색상 (시맨틱)
- `--bg-primary` — Primary 브랜드 색상
- `--bg-secondary` — Secondary 액션
- `--bg-destructive` — 에러/삭제
- `--bg-muted` — Muted 배경
- `--bg-accent` — 강조
- `--text-primary` — 메인 텍스트
- `--text-muted` — 보조 텍스트

### 간격 (일관된 스케일)
- `--spacing-1`부터 `--spacing-16`까지
- 사용: `p-[var(--spacing-4)]`

### Border Radius
- `--radius-sm`, `--radius-md`, `--radius-lg`, `--radius-xl`

## 컴포넌트 라이브러리

`@framingui/ui`에서 사용 가능:
- 레이아웃: `Card`, `Sheet`, `Dialog`, `Tabs`, `Separator`
- 폼: `Button`, `Input`, `Label`, `Select`, `Switch`, `Checkbox`
- 데이터: `Table`, `Avatar`, `Badge`
- 피드백: `Toast`, `Tooltip`, `Progress`
```

export default function RootLayout({ children }) {

### 4단계: 프롬프트로 테스트

Claude Code에서 이 프롬프트를 시도해보세요:

```
사용자 수, 매출, 세션, 전환율을 보여주는 4개의 통계 카드가 있는 대시보드를 만들어줘.
FramingUI 컴포넌트와 디자인 토큰만 사용해줘.
```

이제 Claude는 다음과 같이 생성합니다:

```tsx
import { Card, CardContent } from '@framingui/ui';
import { Users, DollarSign, Activity, TrendingUp } from 'lucide-react';

const stats = [
  { title: '총 사용자', value: '12,345', icon: Users, change: '+12%' },
  { title: '매출', value: '$54,321', icon: DollarSign, change: '+8%' },
  { title: '활성 세션', value: '1,234', icon: Activity, change: '+23%' },
  { title: '전환율', value: '3.2%', icon: TrendingUp, change: '+0.5%' },
];

export function StatsGrid() {
  return (
    <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-[var(--spacing-4)]">
      {stats.map((stat) => (
        <Card key={stat.title}>
          <CardContent className="p-[var(--spacing-6)]">
            <div className="flex items-center justify-between">
              <div>
                <p className="text-sm text-[var(--text-muted)]">
                  {stat.title}
                </p>
                <p className="text-2xl font-bold mt-[var(--spacing-2)]">
                  {stat.value}
                </p>
                <p className="text-sm text-[var(--text-primary)] mt-[var(--spacing-1)]">
                  {stat.change}
                </p>
              </div>
              <div className="p-[var(--spacing-3)] bg-[var(--bg-primary)]/10 rounded-[var(--radius-lg)]">
                <stat.icon className="w-6 h-6 text-[var(--bg-primary)]" />
              </div>
            </div>
          </CardContent>
        </Card>
      ))}
    </div>
  );
}
```

**주목:** 모든 색상, 모든 간격 값, 모든 radius가 디자인 토큰을 사용합니다. 환각이 불가능합니다.

## 고급: v0, Cursor, Bolt에서 환각 방지

FramingUI의 환각 방지 패턴은 모든 AI 도구에서 작동합니다:

### v0.dev의 경우

v0 프로젝트 지침에 추가:

```
@framingui/ui 컴포넌트를 디자인 토큰과 함께 사용:
- 모든 색상은 var(--*) 토큰 사용
- 하드코딩된 Tailwind 색상 금지
- @framingui/ui에서 컴포넌트 import
```

### Cursor의 경우

`.cursorrules`에 추가:

```
# 디자인 시스템 규칙
- @framingui/ui에서만 UI 컴포넌트 import
- 모든 색상과 간격에 var(--*) 디자인 토큰 사용
- 색상 하드코딩 금지: bg-blue-600, text-red-500 등 사용 불가
- 시맨틱 variant 사용: className="bg-blue-600" 대신 variant="primary"
```

### Bolt.new의 경우

첫 메시지에 붙여넣기:

```
FramingUI 디자인 시스템 사용:
1. 모든 색상은 var(--*) 토큰 사용
2. @framingui/ui에서 컴포넌트 사용
3. Tailwind 색상 하드코딩 금지
4. 시맨틱 variant만 사용
```

## 성공 측정: 환각 감사

디자인 환각을 확인하려면 이 명령을 실행하세요:

```bash
# 하드코딩된 색상 찾기 (0개여야 함)
grep -r "bg-\(blue\|red\|green\|indigo\|purple\|pink\|yellow\)" src/

# 색상 hex 코드 찾기 (0개여야 함)
grep -r "#[0-9A-Fa-f]\{6\}" src/

# 디자인 토큰 사용 찾기 (모든 곳에 있어야 함)
grep -r "var(--" src/ | wc -l
```

**목표:** 하드코딩된 색상 0개, 토큰 사용 100%.

## AI 환각 방지의 ROI

### 절약된 시간

- **수동 정리 불필요:** AI 생성 컴포넌트당 2-5분 절약
- **디자인 부채 없음:** 브랜드 변경 시 리팩토링 불필요
- **불일치 없음:** 임의의 파란색 추적에 낭비되는 시간 없음

### 디자인 일관성

- **단일 진실 공급원:** 모든 색상이 한 곳에 정의됨
- **자동 테마화:** 다크모드, 브랜드 색상이 즉시 작동
- **예측 가능한 출력:** AI가 항상 일관된 코드 생성

### 개발자 경험

- **빠른 반복:** 프롬프트 → 생성 → 배포 (정리 단계 없음)
- **더 나은 프롬프트:** 시각적 세부사항이 아닌 기능에 집중
- **낮은 인지 부하:** 모든 색상을 감사하지 않고 AI 출력 신뢰

## 흔한 실수 (그리고 피하는 방법)

### 실수 1: 토큰과 하드코딩 값 혼합

```tsx
// ❌ 나쁨: 반은 토큰, 반은 하드코딩
<div className="bg-[var(--bg-primary)] p-4 rounded-lg">
```

```tsx
// ✅ 좋음: 모두 토큰
<div className="bg-[var(--bg-primary)] p-[var(--spacing-4)] rounded-[var(--radius-lg)]">
```

### 실수 2: AI에게 시스템을 가르치지 않음

토큰을 사용하라고 말하지 않으면 AI는 알 수 없습니다. 항상 프로젝트 지식이나 Cursor 규칙을 포함하세요.

### 실수 3: AI의 첫 번째 출력을 그대로 수용

토큰을 사용해도 처음 몇 번의 생성은 감사하세요. AI가 패턴을 학습하면 일관성을 유지합니다.

## FramingUI가 DIY 토큰 시스템을 능가하는 이유

직접 디자인 토큰을 만들 수도 있습니다. 하지만 FramingUI는 다음을 제공합니다:

1. **사전 구축된 컴포넌트 라이브러리** — 이미 토큰화된 40개 이상의 컴포넌트
2. **AI 최적화 네이밍** — LLM 이해를 위해 설계된 토큰 이름
3. **자동 다크모드** — 모든 토큰이 `prefers-color-scheme`에 적응
4. **타입 안전성** — TypeScript가 토큰 올바른 사용 보장
5. **검증된 패턴** — 500명 이상의 개발자가 프로덕션에서 사용

## 다음 단계: 환각 없이 빌드하기

1. **FramingUI 설치** — `pnpm add @framingui/ui`
2. **프로젝트 지식 생성** — AI에게 디자인 시스템 가르치기
3. **프롬프트로 테스트** — 컴포넌트 생성하고 하드코딩된 색상 없는지 확인
4. **기존 코드 감사** — 환각된 값을 토큰으로 교체

**FramingUI와 함께라면, AI는 디자인 시스템 인식 코드 생성기가 됩니다.** 더 이상 임의의 파란색도, 불일치한 간격도 없습니다. 오직 예측 가능하고 프로덕션 준비된 UI만.
