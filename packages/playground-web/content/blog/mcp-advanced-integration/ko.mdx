# MCP 심화 가이드: 커스텀 AI 도구 만들기

Model Context Protocol(MCP)은 AI 도구가 코드베이스를 이해하는 방식을 바꾸고 있습니다. AI를 진정으로 유용하게 만드는 커스텀 연동을 구축하는 방법을 알아보세요.

## MCP란?

MCP는 AI 어시스턴트가 외부 도구와 데이터에 접근할 수 있게 해주는 표준 프로토콜입니다. AI가 프로젝트에 대해 추측하는 대신, 실제 정보를 조회할 수 있습니다.

## 왜 디자인 시스템에 MCP인가?

디자인 시스템은 MCP 연동에 완벽합니다:

- **구조화된 데이터** — 토큰이 이미 잘 정리되어 있음
- **명확한 제약** — AI가 따라야 할 규칙
- **예측 가능한 출력** — 일관된 컴포넌트 생성

## MCP 서버 구축하기

### 기본 설정

```typescript
import { Server } from '@modelcontextprotocol/sdk/server/index.js'

const server = new Server({
  name: 'my-design-system',
  version: '1.0.0',
})
```

### 디자인 토큰 노출하기

```typescript
server.setRequestHandler('tools/call', async (request) => {
  if (request.params.name === 'get-color') {
    const { colorName } = request.params.arguments
    return {
      content: [{
        type: 'text',
        text: JSON.stringify(tokens.colors[colorName])
      }]
    }
  }
})
```

### 도구 등록하기

```typescript
server.setRequestHandler('tools/list', async () => ({
  tools: [{
    name: 'get-color',
    description: '색상 토큰 값 가져오기',
    inputSchema: {
      type: 'object',
      properties: {
        colorName: { type: 'string' }
      }
    }
  }]
}))
```

## 고급 패턴

### 컴포넌트 생성

전체 컴포넌트를 생성하는 도구 구축:

```typescript
{
  name: 'generate-component',
  description: '적절한 토큰을 사용해 컴포넌트 생성',
  inputSchema: {
    type: 'object',
    properties: {
      componentType: { enum: ['button', 'card', 'input'] },
      variant: { type: 'string' }
    }
  }
}
```

### 테마 전환

AI가 테마 시스템을 이해하게 하기:

```typescript
{
  name: 'apply-theme',
  description: '생성된 코드에 테마 적용',
  inputSchema: {
    type: 'object',
    properties: {
      themeName: { type: 'string' }
    }
  }
}
```
