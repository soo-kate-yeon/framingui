# AI 디자인 워크플로우: 토큰에서 프로덕션 UI까지

AI로 UI를 만든다고 해서 일관성 없는 결과물을 받아들일 필요는 없습니다. 매번 프로덕션 수준의 컴포넌트를 생성하는 워크플로우를 구축하는 방법을 알아보세요.

## AI 생성 UI의 문제점

대부분의 AI 코딩 도구는 디자인을 추측합니다. 색상, 여백, 타이포그래피—브랜드와 맞지 않는 값을 환각(hallucinate)합니다.

**결과는?** 디자인 시스템에 맞추기 위해 생성된 코드를 수정하는 데 몇 시간을 소비하게 됩니다.

## 더 나은 접근법: 토큰 우선 디자인

AI가 추측하게 두지 말고, 정답을 미리 제공하세요:

1. **토큰 정의** — 색상, 여백, 타이포그래피를 구조화된 데이터로
2. **MCP로 연결** — AI 도구에서 토큰을 사용할 수 있게
3. **제약 조건으로 생성** — AI가 상상이 아닌 규칙을 따르게

## 워크플로우 설정하기

### 1단계: 토큰 구조화

```typescript
// tokens/colors.ts
export const colors = {
  primary: {
    50: 'oklch(0.97 0.02 250)',
    500: 'oklch(0.55 0.15 250)',
    900: 'oklch(0.25 0.10 250)',
  },
  // ... 더 많은 색상
}
```

### 2단계: AI 도구에 연결

tekton/ui의 MCP 서버를 사용하면 Claude, Cursor 등의 AI 도구에서 토큰을 사용할 수 있습니다:

```bash
npx @anthropic-ai/claude-code mcp add tekton
```

### 3단계: 일관된 UI 생성

이제 AI에게 컴포넌트를 만들어달라고 하면 실제 디자인 값을 사용합니다:

```
"호버 상태가 있는 primary 버튼 만들어줘"
```

AI는 임의의 파란색이 아닌, 여러분의 primary 색상인 `oklch(0.55 0.15 250)`을 알고 있습니다.

## 실제 결과

토큰 우선 AI 워크플로우를 사용하는 팀들의 성과:
- 컴포넌트 개발 속도 **70% 향상**
- 생성된 코드의 **디자인 불일치 제로**
- **더 행복한 디자이너** (드디어 결과물을 신뢰할 수 있게 됨)
