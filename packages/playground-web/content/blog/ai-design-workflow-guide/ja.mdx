# AI デザインワークフロー: トークンから本番 UI まで

AI で UI を作るときも、結果をばらつかせる必要はありません。毎回本番品質に近いコンポーネントを作るワークフローを紹介します。

## AI 生成 UI で起きやすい問題

多くの AI コーディングツールは、色・余白・タイポグラフィを推測で決めます。するとブランドに合わない値が混ざります。

**その結果**、生成後の修正に時間が取られます。

## より良い方法: Token-First 設計

AI に推測させる代わりに、最初から答えを渡します。

1. **トークンを定義する** — 色、余白、タイポを構造化データで管理
2. **MCP で接続する** — トークンを AI ツールから参照可能にする
3. **制約付きで生成する** — AI は想像ではなくルールに従って出力する

## ワークフロー構築手順

### Step 1: トークンを構造化する

```typescript
// tokens/colors.ts
export const colors = {
  primary: {
    50: 'oklch(0.97 0.02 250)',
    500: 'oklch(0.55 0.15 250)',
    900: 'oklch(0.25 0.10 250)',
  },
  // ... more colors
}
```

### Step 2: AI ツールと接続する

framingui の MCP サーバーを使うと、Claude や Cursor などからトークンを直接参照できます。

```bash
npx @anthropic-ai/claude-code mcp add tekton
```

### Step 3: 一貫した UI を生成する

コンポーネントを依頼すると、AI は実際のデザイン値を使います。

```
"Create a primary button with hover state"
```

たとえば primary 色が `oklch(0.55 0.15 250)` なら、その値を基準にコードを組み立てます。
