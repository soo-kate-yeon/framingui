# OKLCH カラーシステムガイド

OKLCH は、人間の見え方に近い色空間です。同じ明度なら、色相が違っても明るさの体感差を抑えやすくなります。

## OKLCH の基本

- `L`: 明度
- `C`: 彩度
- `H`: 色相

```css
--brand-500: oklch(0.62 0.16 252);
```

## なぜ実務で有利か

1. 明度設計がしやすい
2. ダークモード調整が安定する
3. コントラスト設計と相性が良い

## パレット設計の手順

1. 中心色（500）を決める
2. 明度を段階化して 50〜900 を作る
3. 各段階でコントラストを確認する

## 運用時の注意

- 彩度を上げすぎない
- ダークモードで `L` を再調整する
- 重要色は semantic token で管理する

## まとめ

OKLCH は見た目のきれいさだけでなく、運用の再現性を上げるための選択です。トークン運用と組み合わせると効果が大きくなります。
