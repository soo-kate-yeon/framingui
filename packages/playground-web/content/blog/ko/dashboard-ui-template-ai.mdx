# FramingUI와 AI로 대시보드 레이아웃 만들기

대시보드 레이아웃은 디자인 토큰 규율이 가장 눈에 띄게 성과를 내는 곳이에요. 사이드바, 헤더, 통계 카드 그리드, 데이터 테이블—각각 따로 만들더라도 결국 하나의 일관된 화면이 되어야 합니다. 디자인 시스템 없이 AI 도구가 이 컴포넌트들을 생성하면 불일치가 빠르게 쌓여요.

이 글에서는 FramingUI 컴포넌트로 대시보드의 각 레이어를 만드는 방법을 단계별로 살펴보고, AI 보조 코드가 인터페이스 나머지 부분과 일관되게 유지되도록 프로젝트를 구조화하는 방법도 함께 다룹니다.

## 시작하기

패키지 설치:

```bash
pnpm add @framingui/ui @framingui/core @framingui/tokens tailwindcss-animate
```

AI 도구 통합을 위한 MCP 서버 설정:

```bash
npx -y @framingui/mcp-server@latest init
```

이 명령어 하나가 전체 런타임 계약을 잡아줍니다. 프로바이더 설정, CSS 임포트, Claude Code MCP 설정까지요. Claude Code를 재시작하면 AI 생성 컴포넌트도 직접 작성한 컴포넌트와 동일한 디자인 토큰을 참조합니다.

## 레이아웃 셸

바깥 구조가 레이아웃의 형태를 만들어요. 왼쪽의 고정 사이드바, 오른쪽의 flex 컬럼이 헤더와 스크롤 가능한 콘텐츠를 담습니다.

```tsx
import { Separator } from '@framingui/ui'

export function DashboardLayout({ children }: { children: React.ReactNode }) {
  return (
    <div className="flex h-screen">
      <aside className="w-64 border-r border-[var(--border-default)] bg-[var(--bg-card)]">
        <div className="p-[var(--spacing-6)]">
          <h2 className="text-lg font-bold">MyApp</h2>
        </div>
        <Separator />
      </aside>

      <div className="flex-1 flex flex-col">
        <header className="h-16 border-b border-[var(--border-default)] bg-[var(--bg-card)] flex items-center px-[var(--spacing-6)]">
          <h1 className="text-xl font-semibold">대시보드</h1>
        </header>
        <main className="flex-1 overflow-auto p-[var(--spacing-6)] bg-[var(--bg-background)]">
          {children}
        </main>
      </div>
    </div>
  )
}
```

간격과 테두리 값이 CSS 커스텀 프로퍼티에서 옵니다. 테마가 바뀌면 모든 치수가 자동으로 업데이트돼요.

## 사이드바 내비게이션

사이드바는 활성 상태 추적이 필요해요. FramingUI `Button` 컴포넌트의 variant 전환을 사용하면 커스텀 CSS 없이 활성 항목을 시각적으로 구분할 수 있습니다.

```tsx
'use client'

import { useState } from 'react'
import { Home, Users, BarChart3, Settings, ChevronDown } from 'lucide-react'
import { Button, Separator } from '@framingui/ui'
import { cn } from '@framingui/ui/lib/utils'

const navItems = [
  { icon: Home,      label: '대시보드', href: '/dashboard' },
  { icon: Users,     label: '사용자',   href: '/users' },
  { icon: BarChart3, label: '분석',     href: '/analytics' },
  { icon: Settings,  label: '설정',     href: '/settings' },
]

export function DashboardSidebar() {
  const [activeItem, setActiveItem] = useState('/dashboard')

  return (
    <aside className="w-64 border-r border-[var(--border-default)] bg-[var(--bg-card)] flex flex-col h-screen">
      <div className="p-[var(--spacing-6)]">
        <h2 className="text-lg font-bold">MyApp 관리자</h2>
      </div>
      <Separator />

      <nav className="flex-1 p-[var(--spacing-4)] space-y-[var(--spacing-2)]">
        {navItems.map((item) => (
          <Button
            key={item.href}
            variant={activeItem === item.href ? 'default' : 'ghost'}
            className={cn(
              'w-full justify-start',
              activeItem === item.href && 'bg-[var(--bg-primary)] text-[var(--bg-primary-foreground)]'
            )}
            onClick={() => setActiveItem(item.href)}
          >
            <item.icon className="mr-[var(--spacing-2)] h-4 w-4" />
            {item.label}
          </Button>
        ))}
      </nav>

      <Separator />
      <div className="p-[var(--spacing-4)]">
        <Button variant="ghost" className="w-full justify-between">
          <div className="flex items-center">
            <div className="w-8 h-8 rounded-full bg-[var(--bg-primary)] flex items-center justify-center text-sm font-bold text-[var(--bg-primary-foreground)] mr-[var(--spacing-3)]">
              홍길동
            </div>
            <div className="text-left">
              <p className="text-sm font-medium">홍길동</p>
              <p className="text-xs text-[var(--bg-muted-foreground)]">관리자</p>
            </div>
          </div>
          <ChevronDown className="h-4 w-4" />
        </Button>
      </div>
    </aside>
  )
}
```

## 통계 카드 그리드

통계 그리드는 토큰 기반 간격이 왜 중요한지 잘 보여주는 예시예요. 모든 gap, padding, 아이콘 배경이 동일한 디자인 토큰 스케일을 사용하기 때문에, 카드들이 각자 스타일링된 요소가 아니라 하나의 통합된 시스템처럼 느껴집니다.

```tsx
import { Card, CardContent } from '@framingui/ui'
import { Users, DollarSign, ShoppingCart, TrendingUp } from 'lucide-react'
import { cn } from '@framingui/ui/lib/utils'

const stats = [
  { title: '총 수익',   value: '₩45,231,000', change: '+20.1%', icon: DollarSign,   trend: 'up'   },
  { title: '총 사용자', value: '2,350',        change: '+12.5%', icon: Users,        trend: 'up'   },
  { title: '총 주문',   value: '1,234',        change: '-4.3%',  icon: ShoppingCart, trend: 'down' },
  { title: '전환율',    value: '3.24%',        change: '+0.5%',  icon: TrendingUp,   trend: 'up'   },
] as const

export function StatsGrid() {
  return (
    <div className="grid gap-[var(--spacing-4)] md:grid-cols-2 lg:grid-cols-4">
      {stats.map((stat) => (
        <Card key={stat.title}>
          <CardContent className="p-[var(--spacing-6)]">
            <div className="flex items-center justify-between space-x-[var(--spacing-4)]">
              <div className="space-y-[var(--spacing-2)]">
                <p className="text-sm font-medium text-[var(--bg-muted-foreground)]">
                  {stat.title}
                </p>
                <div className="flex items-baseline space-x-[var(--spacing-2)]">
                  <p className="text-2xl font-bold">{stat.value}</p>
                  <span className={cn(
                    'text-xs font-medium',
                    stat.trend === 'up' ? 'text-[var(--bg-primary)]' : 'text-[var(--bg-destructive)]'
                  )}>
                    {stat.change}
                  </span>
                </div>
              </div>
              <div className="p-[var(--spacing-3)] bg-[var(--bg-primary)]/10 rounded-[var(--radius-lg)]">
                <stat.icon className="h-5 w-5 text-[var(--bg-primary)]" />
              </div>
            </div>
          </CardContent>
        </Card>
      ))}
    </div>
  )
}
```

반응형 그리드가 컬럼 수를 자동으로 처리해요. 모바일 1열, 태블릿 2열, 데스크톱 4열로.

## 데이터 테이블

테이블을 `Card`로 감싸면 위의 통계 카드들과 시각적으로 일관됩니다. 상태 배지는 디자인 시스템이 적절한 색상으로 매핑하는 시맨틱 variant를 사용해요.

```tsx
import {
  Table, TableBody, TableCell, TableHead, TableHeader, TableRow,
  Card, CardHeader, CardTitle, CardContent, Badge, Button,
} from '@framingui/ui'
import { MoreHorizontal } from 'lucide-react'

const orders = [
  { id: '#3210', customer: '김철수', amount: '₩250,000', status: 'completed',  date: '2026-02-27' },
  { id: '#3211', customer: '이영희', amount: '₩120,500', status: 'pending',    date: '2026-02-27' },
  { id: '#3212', customer: '박민수', amount: '₩89,990',  status: 'completed',  date: '2026-02-26' },
  { id: '#3213', customer: '최지원', amount: '₩340,000', status: 'processing', date: '2026-02-26' },
]

export function RecentOrdersTable() {
  return (
    <Card>
      <CardHeader>
        <CardTitle>최근 주문</CardTitle>
      </CardHeader>
      <CardContent>
        <Table>
          <TableHeader>
            <TableRow>
              <TableHead>주문 ID</TableHead>
              <TableHead>고객</TableHead>
              <TableHead>금액</TableHead>
              <TableHead>상태</TableHead>
              <TableHead>날짜</TableHead>
              <TableHead className="w-[50px]" />
            </TableRow>
          </TableHeader>
          <TableBody>
            {orders.map((order) => (
              <TableRow key={order.id}>
                <TableCell className="font-medium">{order.id}</TableCell>
                <TableCell>{order.customer}</TableCell>
                <TableCell>{order.amount}</TableCell>
                <TableCell>
                  <Badge variant={
                    order.status === 'completed' ? 'default' :
                    order.status === 'pending'   ? 'secondary' : 'outline'
                  }>
                    {order.status}
                  </Badge>
                </TableCell>
                <TableCell className="text-[var(--bg-muted-foreground)]">{order.date}</TableCell>
                <TableCell>
                  <Button variant="ghost" size="icon">
                    <MoreHorizontal className="h-4 w-4" />
                  </Button>
                </TableCell>
              </TableRow>
            ))}
          </TableBody>
        </Table>
      </CardContent>
    </Card>
  )
}
```

## 모바일 사이드바

소형 화면에는 드로어 형태의 사이드바가 필요해요. FramingUI의 `Sheet` 컴포넌트가 오버레이를 처리합니다.

```tsx
'use client'

import { useState } from 'react'
import { Menu } from 'lucide-react'
import { Button, Sheet, SheetContent, SheetTrigger } from '@framingui/ui'

export function MobileSidebar({ children }: { children: React.ReactNode }) {
  const [open, setOpen] = useState(false)

  return (
    <>
      <div className="lg:hidden">
        <Sheet open={open} onOpenChange={setOpen}>
          <SheetTrigger asChild>
            <Button variant="ghost" size="icon">
              <Menu className="h-5 w-5" />
            </Button>
          </SheetTrigger>
          <SheetContent side="left" className="p-0 w-64">
            {children}
          </SheetContent>
        </Sheet>
      </div>
      <div className="hidden lg:block">{children}</div>
    </>
  )
}
```

## AI로 대시보드 확장하기

MCP를 설정하면 Claude Code에게 추가 대시보드 섹션을 생성해달라고 요청할 수 있고, 결과물이 위의 컴포넌트들과 동일한 토큰을 참조해요. 이런 프롬프트가 잘 작동합니다.

```
대시보드에 수익 차트 섹션을 추가해줘.
래퍼로 FramingUI Card를 사용해줘.
차트 선 색상으로 var(--bg-primary)를 사용해줘.
내부 패딩에 var(--spacing-6)를 사용해줘.
차트 라이브러리는 Recharts를 사용해줘.
```

프롬프트에 구체적인 토큰 참조를 넣는 게 핵심이에요. 없으면 AI가 `stroke="#3b82f6"`이나 `padding: "24px"`를 선택할 수 있는데, 지금은 맞아 보여도 테마가 바뀌면 깨지는 값들이거든요.

## 최종 페이지 조합

```tsx
import { DashboardSidebar }  from './dashboard-sidebar'
import { StatsGrid }         from './stats-grid'
import { RecentOrdersTable } from './recent-orders-table'
import { Button }            from '@framingui/ui'

export default function DashboardPage() {
  return (
    <div className="flex h-screen">
      <DashboardSidebar />
      <div className="flex-1 flex flex-col">
        <header className="h-16 border-b border-[var(--border-default)] bg-[var(--bg-card)] flex items-center justify-between px-[var(--spacing-6)]">
          <h1 className="text-xl font-semibold">대시보드 개요</h1>
          <Button variant="default">보고서 다운로드</Button>
        </header>
        <main className="flex-1 overflow-auto p-[var(--spacing-6)] bg-[var(--bg-background)]">
          <div className="space-y-[var(--spacing-6)]">
            <StatsGrid />
            <RecentOrdersTable />
          </div>
        </main>
      </div>
    </div>
  )
}
```

각 섹션이 독립적으로 만들어졌고 여기서 조합됩니다. 모든 컴포넌트가 동일한 토큰 세트를 참조하기 때문에, 컴포넌트 사이의 명시적인 조율 없이도 시각적으로 일관된 결과가 나와요.

실용적인 장점은 기능을 추가할 때 드러납니다. 새 섹션도 같은 패턴을 따르면—`@framingui/ui`에서 임포트하고, 모든 스타일 값에 CSS 커스텀 프로퍼티 사용—자동으로 나머지 대시보드와 맞아떨어져요.
