로그인 폼은 대부분의 사용자가 제품과 처음 만나는 지점이에요. 포커스 링이 깨져 있거나, 에러 메시지가 스크린 리더에게 전달되지 않거나, 로딩 중에 버튼이 아무 반응도 안 하는 것들 — 이런 디테일이 제품을 보여주기도 전에 신뢰를 깎아요.

FramingUI의 폼 컴포넌트는 접근성 계약을 자동으로 처리해줘요. 레이블 연결, aria-describedby 연결, 에러 role 알림이 컴포넌트 구조에서 나오기 때문에 따로 유지보수할 커스텀 코드가 필요 없어요.

## 의존성 설치

```bash
pnpm add @framingui/ui @framingui/core @framingui/tokens tailwindcss-animate
```

검증을 위해 react-hook-form과 zod도 추가해요.

```bash
pnpm add react-hook-form zod @hookform/resolvers
```

## 기본 로그인 폼 만들기

아래 컴포넌트는 검증, 로딩 상태, 접근 가능한 에러 메시지를 최소한의 코드로 연결해요.

```tsx
'use client';

import { useState } from 'react';
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { z } from 'zod';
import {
  Card,
  CardHeader,
  CardTitle,
  CardDescription,
  CardContent,
  CardFooter,
  Button,
  Input,
  Form,
  FormField,
  FormItem,
  FormLabel,
  FormControl,
  FormMessage,
} from '@framingui/ui';

const loginSchema = z.object({
  email: z.string().email('올바른 이메일을 입력하세요'),
  password: z.string().min(8, '비밀번호는 최소 8자 이상이어야 합니다'),
});

type LoginFormData = z.infer<typeof loginSchema>;

export function LoginForm() {
  const [isLoading, setIsLoading] = useState(false);

  const form = useForm<LoginFormData>({
    resolver: zodResolver(loginSchema),
    defaultValues: { email: '', password: '' },
  });

  async function onSubmit(data: LoginFormData) {
    setIsLoading(true);
    try {
      const response = await fetch('/api/auth/login', {
        method: 'POST',
        body: JSON.stringify(data),
      });
      if (!response.ok) throw new Error('잘못된 자격 증명');
    } finally {
      setIsLoading(false);
    }
  }

  return (
    <Card className="w-full max-w-md">
      <CardHeader>
        <CardTitle>로그인</CardTitle>
        <CardDescription>계정에 접근하려면 자격 증명을 입력하세요</CardDescription>
      </CardHeader>
      <Form {...form}>
        <form onSubmit={form.handleSubmit(onSubmit)}>
          <CardContent className="space-y-[var(--spacing-4)]">
            <FormField
              control={form.control}
              name="email"
              render={({ field }) => (
                <FormItem>
                  <FormLabel>이메일</FormLabel>
                  <FormControl>
                    <Input type="email" placeholder="you@example.com" {...field} />
                  </FormControl>
                  <FormMessage />
                </FormItem>
              )}
            />
            <FormField
              control={form.control}
              name="password"
              render={({ field }) => (
                <FormItem>
                  <FormLabel>비밀번호</FormLabel>
                  <FormControl>
                    <Input type="password" {...field} />
                  </FormControl>
                  <FormMessage />
                </FormItem>
              )}
            />
          </CardContent>
          <CardFooter className="flex flex-col space-y-[var(--spacing-4)]">
            <Button type="submit" className="w-full" disabled={isLoading}>
              {isLoading ? '로그인 중...' : '로그인'}
            </Button>
            <div className="text-sm text-center">
              <a
                href="/forgot-password"
                className="text-[var(--foreground-accent)] hover:underline"
              >
                비밀번호를 잊으셨나요?
              </a>
            </div>
          </CardFooter>
        </form>
      </Form>
    </Card>
  );
}
```

구현에서 알아두면 좋은 점 몇 가지예요. 간격은 모두 `var(--spacing-*)` CSS 변수를 사용해요. `FormMessage` 컴포넌트는 각 필드의 검증 상태와 연결되고, `role="alert"`으로 렌더링돼서 제출 시 스크린 리더가 에러를 알려줘요. 버튼은 비동기 호출 중에 비활성화돼서 중복 제출을 막아요.

## 접근성이 내부적으로 작동하는 방식

FramingUI의 `Form` 컴포넌트는 전체 ARIA 관계를 자동으로 만들어요. 이렇게 쓰면:

```tsx
<FormLabel>이메일</FormLabel>
<FormControl>
  <Input type="email" {...field} />
</FormControl>
<FormMessage />
```

렌더링된 HTML은 모든 요소를 연결해요.

```html
<label for="email-form-item">이메일</label>
<input
  id="email-form-item"
  aria-describedby="email-form-item-message"
/>
<p id="email-form-item-message" role="alert">
  올바른 이메일을 입력하세요
</p>
```

에러 메시지의 `role="alert"`는 추가 설정 없이도 제출 시 검증 실패를 스크린 리더가 알려주게 해줘요.

## API 에러 처리

검증 스키마 에러는 자동으로 표시돼요. 서버 측 에러는 별도 상태가 필요해요.

```tsx
const [apiError, setApiError] = useState('');

async function onSubmit(data: LoginFormData) {
  setIsLoading(true);
  setApiError('');
  try {
    const response = await fetch('/api/auth/login', {
      method: 'POST',
      body: JSON.stringify(data),
    });
    if (!response.ok) throw new Error('잘못된 자격 증명');
  } catch (error) {
    setApiError(error instanceof Error ? error.message : '문제가 발생했습니다');
  } finally {
    setIsLoading(false);
  }
}

// JSX에서 제출 버튼 위에:
{apiError && (
  <p className="text-sm text-[var(--border-error)]" role="alert">
    {apiError}
  </p>
)}
```

폼을 페이지 중앙에 배치하는 건 레이아웃 래퍼 하나면 충분해요.

```tsx
<div className="flex min-h-screen items-center justify-center p-[var(--spacing-4)]">
  <EnhancedLoginForm />
</div>
```

위 구조는 출발점이에요. 실제 작업은 `onSubmit`을 인증 프로바이더 — NextAuth.js, Supabase, Firebase, 또는 커스텀 API — 와 연결하는 거예요. 컴포넌트가 UI 계약을 담당하고, 인증 로직은 직접 연결해야 해요.
