제약 없이 두면 Claude Code는 실행할 때마다 다른 디자인 시스템을 만들어내요. 월요일에 카드 컴포넌트, 수요일에 테이블, 금요일에 폼을 요청하면 색상 팔레트가 세 가지, 여백 리듬이 세 가지, 버튼 해석이 세 가지예요. 코드는 각각 작동하지만 제품 화면은 따로 노는 느낌이 됩니다.

핵심 원인은 Claude가 우리 시스템이 아니라 "그럴듯함"을 최적화한다는 거예요. 빠진 컨텍스트를 우리 디자인 결정이 아닌 학습 데이터, 즉 공개 코드베이스로 채웁니다. Claude에게 실제 토큰과 컴포넌트에 대한 구조화된 접근권을 주면 "그럴듯함"의 기준 자체가 달라지거든요.

## FramingUI가 제공하는 것

FramingUI는 Claude가 작업할 수 있는 구체적인 어휘를 제공해요:

- `@framingui/ui`에서 열거된 variant와 size를 가진 **타입 안전 컴포넌트**
- 의도를 담은 `var(--bg-primary)`, `var(--spacing-4)` 같은 **시맨틱 CSS 변수**
- 생성 단계에서 제약을 만드는 **일관된 컴포넌트 API**

Claude가 이 어휘를 가지면 새로운 빌딩 블록을 발명하는 대신 여러분의 것을 조합하게 됩니다.

## Claude에게 디자인 시스템 가르치기

가장 빠른 방법은 Claude가 세션 시작마다 읽는 프로젝트 지식 파일이에요. `.claude/project-knowledge.md`를 만드세요:

```markdown
# 프로젝트 디자인 시스템

이 프로젝트는 shadcn-ui 기반에 디자인 토큰을 통합한 컴포넌트 라이브러리 FramingUI를 사용합니다.

## 컴포넌트

`@framingui/ui`에서 임포트:
- Layout: Card, Sheet, Dialog, Tabs, Separator
- Forms: Button, Input, Label, Checkbox, RadioGroup, Select, Switch, Textarea
- Data: Table, Avatar, Badge
- Feedback: Toast, Tooltip, Progress

## 디자인 토큰

모든 스타일링은 CSS 커스텀 속성을 사용합니다:

**색상:** `var(--bg-primary)`, `var(--bg-secondary)`, `var(--bg-destructive)`,
`var(--bg-muted)`, `var(--bg-card)`, `var(--bg-accent)`

**여백:** `var(--spacing-1)` ~ `var(--spacing-8)`

**테두리 반경:** `var(--radius-sm)`, `var(--radius-md)`, `var(--radius-lg)`

## 코드 생성 규칙

- 색상을 하드코딩하지 마세요. 항상 `var(--*)` 토큰을 사용하세요.
- 로컬 파일이 아닌 `@framingui/ui`에서 임포트하세요.
- 시맨틱 이름을 사용하세요: `blue`가 아닌 `primary`.
- `<button>` 대신 `<Button>`을 사용하세요.
```

### 패키지 설치

```bash
pnpm add @framingui/ui @framingui/core @framingui/tokens tailwindcss-animate
```

### 토큰 임포트

```tsx
// app/layout.tsx
import '@framingui/ui/styles';
```

## 컨텍스트가 만드는 차이

같은 프롬프트가 FramingUI 컨텍스트 유무에 따라 어떻게 달라지는지 보여드릴게요.

**프롬프트:** `이름, 이메일, 편집 버튼이 있는 사용자 프로필 카드 만들어줘`

### FramingUI 컨텍스트 없이

```tsx
export function ProfileCard({ name, email }: ProfileCardProps) {
  return (
    <div className="bg-white p-6 rounded-lg shadow-md border border-gray-200">
      <h3 className="text-xl font-bold text-gray-900">{name}</h3>
      <p className="text-gray-600 text-sm mt-1">{email}</p>
      <button className="mt-4 bg-blue-600 text-white px-4 py-2 rounded-md hover:bg-blue-700">
        프로필 수정
      </button>
    </div>
  );
}
```

하드코딩된 색상, 임의 여백, 테마 변경에 적응 불가예요.

### FramingUI 컨텍스트 있으면

```tsx
import { Card, CardHeader, CardTitle, CardContent, CardFooter, Button } from '@framingui/ui';

export function ProfileCard({ name, email }: ProfileCardProps) {
  return (
    <Card>
      <CardHeader>
        <CardTitle>{name}</CardTitle>
      </CardHeader>
      <CardContent>
        <p className="text-sm text-[var(--bg-muted-foreground)]">{email}</p>
      </CardContent>
      <CardFooter>
        <Button variant="default">프로필 수정</Button>
      </CardFooter>
    </Card>
  );
}
```

시맨틱 토큰, FramingUI 프리미티브, 자동 테마 적응이에요.

## 두 번의 프롬프트로 대시보드 만들기

제약 있는 프롬프트를 쓰면 복잡한 UI도 일관성 있게 만들 수 있어요.

**프롬프트 1: 통계 카드**

```
총 사용자, 수익, 활성 세션, 전환율을 표시하는 통계 카드 4개 만들어줘.
FramingUI Card 컴포넌트 사용해. 여백이랑 색상은 전부 디자인 토큰 써.
```

Claude가 생성:

```tsx
import { Card, CardContent } from '@framingui/ui';
import { Users, DollarSign, Activity, TrendingUp } from 'lucide-react';

const stats = [
  { title: '총 사용자', value: '12,345', icon: Users, change: '+12%' },
  { title: '수익', value: '$54,321', icon: DollarSign, change: '+8%' },
  { title: '활성 세션', value: '1,234', icon: Activity, change: '+23%' },
  { title: '전환율', value: '3.2%', icon: TrendingUp, change: '+0.5%' },
];

export function StatsGrid() {
  return (
    <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-[var(--spacing-4)]">
      {stats.map((stat) => (
        <Card key={stat.title}>
          <CardContent className="p-[var(--spacing-6)]">
            <div className="flex items-center justify-between">
              <div>
                <p className="text-sm text-[var(--bg-muted-foreground)]">{stat.title}</p>
                <p className="text-2xl font-bold mt-[var(--spacing-2)]">{stat.value}</p>
                <p className="text-sm text-[var(--bg-primary)] mt-[var(--spacing-1)]">{stat.change}</p>
              </div>
              <div className="p-[var(--spacing-3)] bg-[var(--bg-primary)]/10 rounded-[var(--radius-lg)]">
                <stat.icon className="w-6 h-6 text-[var(--bg-primary)]" />
              </div>
            </div>
          </CardContent>
        </Card>
      ))}
    </div>
  );
}
```

모든 여백이 토큰을 써요. 색상도 시맨틱하죠.

**프롬프트 2: 사용자 테이블**

```
통계 아래에 이름, 이메일, 역할, 상태 컬럼 있는 사용자 테이블 추가해줘.
FramingUI Table이랑 Badge 컴포넌트 사용해.
```

Claude가 일관된 결과를 냅니다. 같은 토큰 어휘, 같은 컴포넌트 프리미티브, 통계 카드와 시각적으로 어울리는 결과예요.

## 커스텀 토큰 확장

앱별 레이아웃 상수는 FramingUI 토큰 임포트 옆에 정의하세요:

```css
/* globals.css */
@import '@framingui/ui/styles';

:root {
  --app-sidebar-width: 280px;
  --app-header-height: 64px;
}
```

이걸 프로젝트 지식 파일에 추가하면 레이아웃 프롬프트에서도 올바르게 활용해요:

```
커스텀 레이아웃 토큰을 사용해서 사이드바와 헤더가 있는 대시보드 레이아웃 만들어줘.
```

`var(--app-sidebar-width)`, `var(--app-header-height)`를 정확하게 참조한 결과가 나옵니다.

## 효과적인 프롬프트 패턴

일관되게 품질을 높이는 패턴 몇 가지예요.

**컴포넌트 이름을 명시하세요.** "모달 만들어줘"보다 "FramingUI Dialog로 모달 만들어줘"가 훨씬 나아요.

**토큰을 직접 참조하세요.** "텍스트를 회색으로"보다 "보조 텍스트에 `var(--bg-muted-foreground)` 써줘"가 명확합니다.

**variant 이름을 쓰세요.** "버튼 빨갛게"보다 "Button variant='destructive'"가 의도를 정확히 전달해요.

**일관성 체크를 추가하세요.** 복잡한 컴포넌트를 만든 후에 "코드 검토해서 하드코딩된 색상이나 여백을 시맨틱 토큰으로 바꿔줘"를 붙이세요.

## 토큰 사용 현황 확인

생성 코드에서 토큰 준수 여부를 점검하세요:

```bash
# 하드코딩 색상 (일치 없어야 함)
grep -r "bg-blue\|text-red\|#[0-9A-Fa-f]\{6\}" src/

# 토큰 사용 (많이 있어야 함)
grep -r "var(--" src/
```

큰 생성 세션 후에 한 번씩 실행해보세요. 위반이 발견되면 커밋 전에 Claude에게 수정을 요청하세요.
