## Tailwind 함정

Tailwind를 좋아합니다. 빠르고 직관적이며 HTML을 벗어나지 않고 UI를 만들 수 있습니다.

```tsx
<button className="bg-blue-600 hover:bg-blue-700 text-white px-4 py-2 rounded-lg">
  Click me
</button>
```

배포. 완료. 아름답습니다.

하지만 6개월 후에는:
- 23가지 다른 파란색 shade
- `px-4 py-2`, `px-5 py-2.5`, `px-6 py-3`, `px-4 py-1.5`인 버튼들
- `gap-4`를 쓰는 폼, `gap-6`을 쓰는 카드, `gap-3`을 쓰는 그리드

**Tailwind가 디자인 시스템이라고 생각했습니다. 아닙니다.**

Tailwind는 **유틸리티 툴킷**입니다. 디자인 시스템은 **결정**의 집합입니다. 이 글은 Tailwind의 속도를 잃지 않고 Tailwind 위에 결정을 레이어링하는 방법을 보여줍니다.

## 핵심 요약

- Tailwind는 유틸리티를 제공하며, 디자인 제약은 아닙니다
- 토큰 없이는 팀이 일관성 없음으로 표류합니다 (Tailwind를 써도)
- Design tokens은 Tailwind 유틸리티 클래스 위에 브랜드 결정을 레이어링합니다
- FramingUI는 토큰을 Tailwind 설정에 연결하여 구조와 속도를 모두 제공합니다
- `bg-primary`, `px-md`를 계속 씁니다—하지만 이제 Tailwind 기본값이 아닌 당신의 결정입니다

## Tailwind만으로는 디자인 시스템이 아닌 이유

### 문제 1: 너무 많은 선택지

Tailwind는 각 색상의 22가지 shade를 제공합니다:

```javascript
colors: {
  blue: {
    50: '#eff6ff',
    100: '#dbeafe',
    // ...
    900: '#1e3a8a',
    950: '#172554'
  }
}
```

**질문: `blue-600` vs. `blue-700`을 언제 쓰나요?**

답이 "좋아 보이는 걸로"라면, 디자인 시스템이 없는 겁니다. 예쁜 그라데이션이 있는 혼돈입니다.

### 문제 2: 복사-붙여넣기 복권

개발자 1이 primary 버튼을 만듭니다:
```tsx
<button className="bg-blue-600 hover:bg-blue-700 ...">
```

개발자 2가 다른 파일에서 primary 버튼이 필요합니다:
```tsx
<button className="bg-blue-500 hover:bg-blue-600 ...">
```

둘 다 독립적으로는 "괜찮아" 보입니다. 하지만 나란히? 같은 목적의 다른 버튼입니다.

### 문제 3: 브랜드 색상 변경은 악몽

디자이너가 말합니다: "파란색에서 인디고로 바꿔봐요."

**원시 Tailwind 클래스**에서는 찾기-바꾸기를 합니다:
- `blue-600` → `indigo-600`
- `blue-700` → `indigo-700`
- `blue-500` → `indigo-500` (잠깐, 이건 500이어야 하나 600이어야 하나?)

**소요 시간:** 2~4시간  
**인스턴스 놓칠 위험:** 높음  
**뭔가 망가질 확률:** 중간

### 문제 4: Dark Mode가 재앙이 됨

```tsx
// Light mode는 blue-600 사용
<div className="bg-blue-600 text-white">

// Dark mode... 어떤 색이어야 하지?
<div className="bg-blue-600 dark:bg-blue-400 text-white dark:text-black">
// 잠깐, 텍스트가 검정이어야 하나? 아니면 gray-900? 아니면 흰색?
```

**시맨틱 토큰** 없이는 매번 명암비와 시각적 계층을 추측합니다.

## Design Tokens이 실제로 해결하는 것

Design tokens은 값에 매핑되는 **이름 붙은 결정**입니다:

```typescript
// 이 대신:
bg-blue-600

// 이렇게 씁니다:
bg-brand-primary

// 이게 매핑됩니다:
colors.brand.primary = '#2563EB'  // (또는 blue-600, indigo-700, 무엇이든)
```

**마법:** `colors.brand.primary`를 변경하면 모든 컴포넌트가 자동으로 업데이트됩니다.

### 시맨틱 네이밍 = 디자인 의도

```typescript
// ❌ 구현 중심 (Tailwind 기본값)
colors: {
  blue: { 600: '#2563EB' }
}

// ✅ 의도 중심 (Design Tokens)
colors: {
  brand: {
    primary: '#2563EB',      // 메인 CTA, 링크
    primaryHover: '#1D4ED8', // Hover 상태
  },
  semantic: {
    success: '#10B981',
    error: '#EF4444',
    warning: '#F59E0B',
  },
  surface: {
    base: '#FFFFFF',         // 카드 배경
    elevated: '#F9FAFB',     // 모달, 팝오버
  }
}
```

이제 `bg-surface-elevated`를 보면 그 색상이 **왜** 존재하는지 알 수 있고, 단순히 **무엇**인지만 아는 게 아닙니다.

## 빠진 연결고리: Tailwind + Tokens

Tailwind의 속도와 디자인 시스템의 일관성 중 선택할 필요가 없습니다. 둘 다 가질 수 있습니다.

### 아키텍처

```
Design Tokens (source of truth)
         ↓
   Tailwind Config (유틸리티)
         ↓
   Your Components (클래스)
```

### 1단계: 토큰 정의 (YAML, JSON, 또는 TypeScript)

```typescript
// tokens/colors.ts
export const colorTokens = {
  brand: {
    primary: '#4338CA',
    primaryHover: '#3730A3',
    secondary: '#7C3AED',
  },
  neutral: {
    50: '#F9FAFB',
    100: '#F3F4F6',
    200: '#E5E7EB',
    // ... 전체 스케일
    900: '#111827',
  },
  semantic: {
    success: '#10B981',
    successBg: '#D1FAE5',
    error: '#EF4444',
    errorBg: '#FEE2E2',
  }
}
```

### 2단계: Tailwind Config에 토큰 주입

```javascript
// tailwind.config.js
import { colorTokens } from './tokens/colors'

export default {
  theme: {
    extend: {
      colors: {
        brand: colorTokens.brand,
        neutral: colorTokens.neutral,
        semantic: colorTokens.semantic,
      }
    }
  }
}
```

### 3단계: 시맨틱 클래스 사용

```tsx
// ❌ 이전 (원시 Tailwind)
<button className="bg-blue-600 hover:bg-blue-700 text-white">
  Save
</button>

// ✅ 이후 (Token 기반 Tailwind)
<button className="bg-brand-primary hover:bg-brand-primaryHover text-white">
  Save
</button>
```

**이제 브랜드 색상 변경은 한 줄 편집입니다:**

```diff
// tokens/colors.ts
export const colorTokens = {
  brand: {
-   primary: '#4338CA',
+   primary: '#7C3AED',  // 보라색으로 전환
  }
}
```

모든 버튼, 링크, CTA가 즉시 업데이트됩니다. 찾기-바꾸기 제로.

## Spacing, Typography 등

색상은 시작일 뿐입니다. 토큰은 **모든 디자인 결정**에 작동합니다.

### Spacing Tokens

```typescript
// tokens/spacing.ts
export const spacingTokens = {
  '3xs': '0.125rem',  // 2px
  '2xs': '0.25rem',   // 4px
  'xs': '0.5rem',     // 8px
  'sm': '0.75rem',    // 12px
  'md': '1rem',       // 16px
  'lg': '1.5rem',     // 24px
  'xl': '2rem',       // 32px
  '2xl': '3rem',      // 48px
  '3xl': '4rem',      // 64px
}

// tailwind.config.js
export default {
  theme: {
    spacing: spacingTokens,
  }
}
```

**사용:**
```tsx
// ❌ 이전: 임의 값
<div className="px-4 py-3 gap-5">

// ✅ 이후: 이름 붙은 결정
<div className="px-md py-sm gap-lg">
```

### Typography Tokens

```typescript
// tokens/typography.ts
export const typographyTokens = {
  fontFamily: {
    sans: ['Inter', 'system-ui', 'sans-serif'],
    mono: ['JetBrains Mono', 'monospace'],
  },
  fontSize: {
    'xs': ['0.75rem', { lineHeight: '1rem' }],
    'sm': ['0.875rem', { lineHeight: '1.25rem' }],
    'base': ['1rem', { lineHeight: '1.5rem' }],
    'lg': ['1.125rem', { lineHeight: '1.75rem' }],
    'xl': ['1.25rem', { lineHeight: '1.75rem' }],
    '2xl': ['1.5rem', { lineHeight: '2rem' }],
  },
  fontWeight: {
    normal: '400',
    medium: '500',
    semibold: '600',
    bold: '700',
  }
}

// tailwind.config.js
export default {
  theme: {
    extend: {
      fontFamily: typographyTokens.fontFamily,
      fontSize: typographyTokens.fontSize,
      fontWeight: typographyTokens.fontWeight,
    }
  }
}
```

**사용:**
```tsx
<h1 className="font-sans text-2xl font-bold">
  Heading
</h1>
<p className="font-sans text-base font-normal text-neutral-700">
  일관된 line height를 가진 본문 텍스트.
</p>
```

## Tokens과 함께하는 Dark Mode

토큰이 진정으로 빛나는 곳입니다. dark mode 색상을 추측하는 대신 **시맨틱 역할**을 정의합니다:

```typescript
// tokens/colors.ts
export const colorTokens = {
  surface: {
    base: {
      light: '#FFFFFF',
      dark: '#111827',
    },
    elevated: {
      light: '#F9FAFB',
      dark: '#1F2937',
    }
  },
  text: {
    primary: {
      light: '#111827',
      dark: '#F9FAFB',
    },
    secondary: {
      light: '#6B7280',
      dark: '#9CA3AF',
    }
  }
}
```

**CSS 변수를 사용한 Tailwind 설정:**

```javascript
// tailwind.config.js
export default {
  theme: {
    extend: {
      colors: {
        surface: {
          base: 'var(--color-surface-base)',
          elevated: 'var(--color-surface-elevated)',
        },
        text: {
          primary: 'var(--color-text-primary)',
          secondary: 'var(--color-text-secondary)',
        }
      }
    }
  }
}
```

**CSS:**

```css
:root {
  --color-surface-base: #FFFFFF;
  --color-surface-elevated: #F9FAFB;
  --color-text-primary: #111827;
  --color-text-secondary: #6B7280;
}

.dark {
  --color-surface-base: #111827;
  --color-surface-elevated: #1F2937;
  --color-text-primary: #F9FAFB;
  --color-text-secondary: #9CA3AF;
}
```

**컴포넌트:**
```tsx
<div className="bg-surface-base text-text-primary">
  이것은 자동으로 light/dark mode에 적응합니다
</div>
```

`dark:bg-gray-900 dark:text-white`가 필요 없습니다. 토큰이 처리합니다.

## FramingUI 방식

FramingUI는 이 전체 워크플로우를 자동화합니다.

### 1. 토큰 한 번 정의

```typescript
// framingui.config.ts
import { defineConfig } from '@framingui/core'

export default defineConfig({
  tokens: {
    colors: {
      brand: {
        primary: '#4338CA',
        secondary: '#7C3AED',
      },
      // ... 색상
    },
    spacing: {
      sm: '0.5rem',
      md: '1rem',
      lg: '1.5rem',
    },
    typography: {
      fontFamily: {
        sans: 'Inter, system-ui',
      }
    }
  }
})
```

### 2. Tailwind Config 자동 생성

```bash
npx framingui generate
```

출력:
- 토큰이 있는 `tailwind.config.js`
- dark mode용 CSS 변수
- 자동완성용 TypeScript 타입

### 3. 컴포넌트에서 사용

```tsx
import { tokens } from '@/framingui'

// Tailwind 클래스 (자동완성 작동)
<button className="bg-brand-primary px-md py-sm">

// 또는 프로그래밍 방식 접근
<button style={{ backgroundColor: tokens.colors.brand.primary }}>
```

### 4. 토큰 변경, 즉시 재빌드

```diff
// framingui.config.ts
colors: {
  brand: {
-   primary: '#4338CA',
+   primary: '#DC2626',  // 빨간색 리브랜딩
  }
}
```

```bash
npx framingui generate
```

`bg-brand-primary`를 사용하는 모든 컴포넌트가 업데이트됩니다. 수동 찾기-바꾸기 없음.

## 실제 사례: 5분 만에 리브랜딩

**시나리오:** 스타트업이 인수됩니다. 새 브랜드 색상, 새 타이포그래피, 새 간격 시스템.

**이전 (원시 Tailwind):**
1. `blue-600`의 모든 인스턴스 찾기 → 2시간
2. `px-4`의 모든 인스턴스를 찾아 유지할 것 결정 → 3시간
3. 타이포그래피 클래스 업데이트 → 1시간
4. 놓친 인스턴스 QA → 2시간
**총: 8시간**

**이후 (Tailwind + Tokens):**
1. `framingui.config.ts` 업데이트 (10분)
2. `npx framingui generate` 실행 (5초)
3. 시각적으로 QA (20분)
**총: 30분**

**절약된 시간: 7.5시간**

## 흔한 반론

### "Tailwind 클래스는 이미 충분히 시맨틱해요"

`bg-blue-600`은 **무엇**(파란색의 shade)을 말하지, **왜**(primary 브랜드 색상? 링크 색상? 장식 악센트?)는 말하지 않습니다.

`bg-brand-primary`는 **의도**를 말합니다. 코드에서 보면 역할을 알 수 있습니다.

### "이건 추가 추상화를 더해요"

맞지만, 올바른 종류입니다. **구현**이 아니라 **결정**을 추상화합니다.

토큰이 있으면 마음을 바꾸는 것이 쌉니다. 토큰 없이는 마음을 바꾸는 것이 비쌉니다.

### "Tailwind의 임의 값이 좋아요"

일회용으로 유지하세요:
```tsx
<div className="w-[47.5%]">  // 고유 레이아웃에는 괜찮음
```

하지만 반복되는 결정(브랜드 색상, 간격 스케일)에는 토큰을 사용하세요.

### "우리 팀은 이걸 하기엔 너무 작아요"

작은 팀이 가장 큰 이득을 봅니다. 관료주의 없이 기반을 설정할 수 있는 완벽한 크기입니다.

**2인 팀:** 당신과 공동 창업자가 표류하는 것을 방지
**5인 팀:** 신입을 즉시 온보딩
**10인 팀:** 일관성에 필수

## 마이그레이션 전략: 모든 것을 다시 작성하지 않고 토큰 추가

하룻밤 사이에 전체 코드베이스를 리팩터링할 필요가 없습니다.

### 1단계: 토큰 정의 (1시간)

가장 많이 쓰는 색상, 간격, 타이포그래피를 토큰으로 추출합니다.

### 2단계: Tailwind Config 확장 (30분)

Tailwind 테마에 토큰을 추가합니다. 기존 클래스는 여전히 작동합니다.

### 3단계: 점진적으로 채택

**새 컴포넌트:** 토큰 사용 (`bg-brand-primary`)
**기존 컴포넌트:** 그대로 두기 (`bg-blue-600`)

둘 다 동시에 작동합니다. 깨지는 변경 없음.

### 4단계: Hot Path 리팩터링 (진행 중)

다른 이유로 파일을 수정할 때 클래스를 토큰 사용으로 업데이트합니다.

2~3개월에 걸쳐 코드베이스가 자연스럽게 수렴합니다.

## 미래: AI 기반 토큰 관리

FramingUI + MCP를 사용하면 AI 도구(Cursor, Claude Code)가:

1. 코드 생성 전 **토큰 쿼리**
2. 값을 반복적으로 사용할 때 **토큰 추가 제안**
3. 하드코딩된 값에 대해 **코드베이스 감사**하고 마이그레이션 제안

**예시:**
```
Cursor AI: "5곳에서 #F59E0B를 사용한 것 같습니다. 
colors.semantic.warning으로 추가할까요?"
```

AI가 디자인 시스템을 알면 Tailwind가 더욱 빨라집니다.

## 결론

**Tailwind는 속도를 줍니다.**  
**Design tokens은 일관성을 줍니다.**  
**함께하면 둘 다 얻습니다.**

빠르게 움직이는 것과 일관된 디자인 시스템 구축 사이에서 선택할 필요가 없습니다.

Tailwind 유틸리티 클래스 위에 시맨틱 토큰을 레이어링하세요. 좋아하는 DX를 유지하고 필요한 구조를 추가하세요.
