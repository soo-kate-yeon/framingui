## AI + Shadcn UI의 문제점

Shadcn UI는 사람 개발자에게는 훌륭합니다. 컴포넌트를 복사해서 프로젝트에 붙여넣고 필요에 따라 커스터마이징하면 됩니다. 하지만 Claude Code나 Cursor 같은 AI 도구가 UI 코드를 생성할 때는 중요한 문제에 직면합니다: **수백 개의 생성된 컴포넌트 간 일관성을 어떻게 보장할 것인가?**

AI는 디자인 시스템과 맞지 않는 색상 값, 간격, 스타일 패턴을 환각할 수 있습니다. 한 컴포넌트는 `bg-blue-500`을 사용하고, 다른 컴포넌트는 `bg-indigo-600`을, 또 다른 컴포넌트는 `#4F46E5`를 사용하는데 모두 "primary" 색상을 의도한 것입니다.

## FramingUI: 빠진 다리

FramingUI는 shadcn UI를 포크하고 모든 컴포넌트에 **디자인 토큰**을 직접 통합하여 이 문제를 해결합니다. 임의의 Tailwind 클래스 대신 모든 스타일이 시맨틱 토큰을 참조합니다:

```tsx
// ❌ Shadcn UI (원본)
<button className="bg-blue-500 hover:bg-blue-600 text-white">
  클릭하세요
</button>

// ✅ FramingUI (토큰 기반)
<Button variant="default">
  클릭하세요
</Button>

// 내부 구현:
// bg-[var(--tekton-bg-primary)]
// hover:bg-[var(--tekton-bg-primary)]/90
```

## 실제 예제: Button 컴포넌트

FramingUI의 Button 컴포넌트가 토큰을 사용하는 방법:

```tsx
import { Button } from '@framingui/ui';

function LoginForm() {
  return (
    <div className="space-y-[var(--tekton-spacing-4)]">
      <Button variant="default">로그인</Button>
      <Button variant="outline">계정 생성</Button>
      <Button variant="destructive">삭제</Button>
    </div>
  );
}
```

모든 버튼은 자동으로 테마의 색상, 간격, 테두리 반경을 상속받습니다 — **하드코딩된 값 없이**.

## 디자인 토큰이 AI 환각을 방지하는 방법

AI가 FramingUI를 사용하여 코드를 생성할 때 색상을 환각할 수 없는 이유:

1. **단일 진실 공급원**: 모든 색상이 `var(--tekton-*)` 토큰에서 나옵니다
2. **시맨틱 네이밍**: `--tekton-bg-primary`는 명확합니다
3. **OKLCH 기반**: 색상이 지각적으로 균일하여 일관된 밝기를 보장합니다

### 예제: AI 생성 카드

```tsx
import { Card, CardHeader, CardTitle, CardDescription, CardContent } from '@framingui/ui';

// FramingUI를 사용한 AI 생성 컴포넌트
function ProductCard({ title, description, price }: ProductCardProps) {
  return (
    <Card>
      <CardHeader>
        <CardTitle>{title}</CardTitle>
        <CardDescription>{description}</CardDescription>
      </CardHeader>
      <CardContent>
        <p className="text-2xl font-bold text-[var(--tekton-bg-primary)]">
          {price}원
        </p>
      </CardContent>
    </Card>
  );
}
```

AI가 `text-blue-600`이나 `#3B82F6`을 추측하는 대신 토큰(`var(--tekton-bg-primary)`)을 사용하는 것에 주목하세요.

## OKLCH: 비밀 소스

FramingUI는 디자인 토큰에 **OKLCH 색상 공간**을 사용하여 다음을 제공합니다:

- **지각적 균일성**: L=0.5는 모든 색조에서 동일하게 밝게 보입니다
- **더 넓은 색역**: sRGB보다 더 생생한 색상에 접근 가능
- **WCAG 준수**: AA/AAA 대비 비율을 더 쉽게 달성

HSL 같은 전통적인 색상 공간은 여기서 실패합니다 — `hsl(220, 100%, 50%)` (파란색)은 `hsl(60, 100%, 50%)` (노란색)보다 더 어둡게 보이는데도 둘 다 50% 밝기를 갖습니다.

## AI 도구와의 통합

### Cursor IDE 설정

```bash
# FramingUI 설치
pnpm add @framingui/ui @framingui/core

# 브랜드 색상에서 디자인 토큰 생성
npx framingui generate --color "#FF6B35"
```

그런 다음 Cursor 규칙(`.cursorrules`)에 추가하세요:

```
UI 컴포넌트 생성 시:
- 항상 @framingui/ui에서 임포트
- 색상, 간격, 반경에 var(--tekton-*) 토큰 사용
- 색상 값이나 간격을 하드코딩하지 말 것
```

### Claude Code 설정

`.claude/skills/framingui.md` 파일을 생성하세요:

```markdown
## FramingUI 컴포넌트 라이브러리

@framingui/ui에서 컴포넌트 임포트:
- Button, Input, Card, Form, Dialog 등

디자인 토큰 (Tailwind 색상 대신 사용):
- 색상: var(--tekton-bg-primary), var(--tekton-bg-secondary)
- 간격: var(--tekton-spacing-2), var(--tekton-spacing-4)
- 반경: var(--tekton-radius-md), var(--tekton-radius-lg)
```

## Shadcn UI에서 마이그레이션

이미 shadcn UI를 사용 중이신가요? FramingUI는 드롭인 대체제입니다:

```tsx
// 이전 (shadcn)
import { Button } from '@/components/ui/button';

// 이후 (FramingUI)
import { Button } from '@framingui/ui';
// 동일한 API, 토큰 기반 내부 구현
```
