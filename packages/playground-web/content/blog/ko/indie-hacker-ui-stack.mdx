## 솔로 개발자의 딜레마

혼자(또는 공동 창업자 1명과) SaaS 제품을 만들고 있습니다. 필요한 것:

- 빠르게 출시 (투자자/사용자는 기다리지 않음)
- 전문적으로 보이기 (일반적인 UI는 전환하지 않음)
- 일관성 유지 (허술한 UI는 신뢰를 해침)
- 비용 절감 ($50k 디자인 에이전시 없음)
- 유지보수 지옥 피하기 (당신이 유일한 엔지니어)

기존 조언: "Material-UI나 Chakra 같은 컴포넌트 라이브러리 사용."

문제: 일반 라이브러리는 일반적으로 보입니다. 모든 MUI 앱이 똑같이 보입니다. 랜딩 페이지가 "템플릿"이라고 외칩니다.

**2026 답변:** 속도와 브랜드 차별화를 모두 제공하는 **레이어드 스택**.

## 핵심 요약: 스택

```
Layer 1: Tailwind CSS (유틸리티 프레임워크)
         ↓
Layer 2: FramingUI (design tokens + 시스템)
         ↓
Layer 3: shadcn/ui (복사-붙여넣기 컴포넌트)
         ↓
Layer 4: Custom components (차별화 요소)
```

**설정 시간:** 2~3시간
**유지보수:** 월 1시간
**비용:** $0 (모두 오픈소스)
**결과:** 독특하게 보이고, 빠르게 출시되고, 일관성을 유지하는 전문 UI

## 왜 이 스택이 2026년에 이기는가

### Material-UI / Chakra와 비교

| | Material-UI / Chakra | FramingUI + shadcn |
|---|---|---|
| **설정** | `npm install` → 완료 | 2시간 (tokens + config) |
| **커스터마이징** | 테마 설정 (제한적) | 완전한 제어 (코드 소유) |
| **번들 크기** | 100-300kb | 20-50kb (tree-shakeable) |
| **브랜드 차별화** | 다른 모든 MUI 앱처럼 보임 | 브랜드에 독특하게 보임 |
| **AI 호환성** | 중간 (AI가 일반 props 사용) | 높음 (AI가 토큰 사용) |
| **학습 곡선** | 중간 (라이브러리별 API) | 낮음 (표준 HTML + Tailwind) |

### 처음부터 구축과 비교

| | 처음부터 | FramingUI + shadcn |
|---|---|---|
| **첫 컴포넌트까지 시간** | 4~8시간 | 5분 |
| **접근성** | 직접 구현 필요 | 내장 (shadcn이 Radix 사용) |
| **일관성** | 쉽게 표류 | 토큰으로 강제 |
| **Dark mode** | 수동 구현 | 자동 (토큰 기반) |
| **유지보수** | 모두 당신 몫 | 커뮤니티 업데이트 |

## 스택, 레이어별로

### Layer 1: Tailwind CSS (기반)

**역할:** HTML을 떠나지 않고 스타일링하는 유틸리티 클래스

**필수인 이유:**
- CSS 작성보다 10배 빠름
- "이 클래스 이름을 뭐라고 할까?" 결정 피로 제거
- 작은 번들 (사용된 클래스만 포함)
- 업계 표준 (채용 쉬움, 학습 쉬움)

**설정:**
```bash
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
```

**판결:** 협상 불가. 모든 현대 인디 해커는 Tailwind를 써야 합니다.

---

### Layer 2: FramingUI (디자인 시스템)

**역할:** Tailwind 위에 위치하는 design tokens + 규칙

**필요한 이유:**
- Tailwind는 1000개 옵션을 제공; FramingUI는 당신의 옵션으로 좁힙니다
- 앱 전체에 일관성 강제
- 리브랜딩을 5분 작업으로 만듦 (코드가 아니라 토큰 변경)
- AI 도구(Cursor, Claude)가 브랜드에 맞는 코드 생성 가능

**얻는 것:**
```typescript
// 코드화된 디자인 결정
const tokens = {
  colors: {
    brand: {
      primary: '#4338CA',    // 무작위 파란색이 아닌 브랜드 색상
      secondary: '#7C3AED',
    }
  },
  spacing: {
    sm: '0.75rem',           // 임의 px 값이 아닌 간격 스케일
    md: '1rem',
    lg: '1.5rem',
  },
  typography: {
    fontFamily: {
      sans: 'Inter, system-ui',  // 시스템 기본값이 아닌 폰트
    }
  }
}
```

**설정:**
```bash
npm install @framingui/core
```

**시간 투자:** 토큰 정의에 1~2시간

**판결:** 가치 있음. 이것이 코드 형태의 디자인 시스템입니다.

---

### Layer 3: shadcn/ui (컴포넌트 라이브러리)

**역할:** 복사-붙여넣기 React 컴포넌트 (NPM 패키지가 아님)

**인디 해커에게 완벽한 이유:**
- 코드 소유 (vendor lock-in 없음)
- 완전히 커스터마이징 가능 (소스 직접 편집)
- 기본적으로 접근 가능 (Radix UI primitives 기반)
- Tailwind + FramingUI와 완벽하게 작동

**얻는 것:**
- 약 40개 프로덕션 준비 컴포넌트 (Button, Dialog, Form, Table 등)
- TypeScript 지원
- 키보드 탐색 및 스크린 리더 지원
- 기본 제공 dark mode

**설정:**
```bash
npx shadcn@latest init
```

**컴포넌트 추가:**
```bash
npx shadcn@latest add button
npx shadcn@latest add dialog
npx shadcn@latest add form
```

컴포넌트가 당신의 코드베이스의 `components/ui/`에 추가됩니다. 자유롭게 편집 가능합니다.

**판결:** 게임 체인저. 번들 크기나 lock-in 없이 컴포넌트 라이브러리의 80%를 얻습니다.

---

### Layer 4: Custom Components (차별화 요소)

**역할:** 제품 고유의 컴포넌트

**필요한 이유:**
- shadcn은 일반 UI 제공 (버튼, 폼, 다이얼로그)
- 제품에는 도메인별 컴포넌트 필요 (분석 차트, 사용자 프로필, 가격 테이블)

**예시:**
```tsx
// components/features/PricingCard.tsx
// components/features/AnalyticsDashboard.tsx
// components/features/UserAvatar.tsx
```

FramingUI 토큰과 shadcn 컴포넌트를 빌딩 블록으로 사용:

```tsx
// components/features/PricingCard.tsx
import { Button } from '@/components/ui/button'
import { tokens } from '@/lib/tokens'

export function PricingCard({ plan, price, features }) {
  return (
    <div className="bg-surface-base border border-neutral-200 rounded-lg p-lg">
      <h3 className="text-2xl font-bold text-text-primary">{plan}</h3>
      <p className="text-4xl font-bold text-brand-primary my-md">${price}/mo</p>
      <ul className="space-y-sm mb-lg">
        {features.map(f => (
          <li key={f} className="flex items-center gap-sm">
            <Check className="w-5 h-5 text-semantic-success" />
            <span className="text-text-secondary">{f}</span>
          </li>
        ))}
      </ul>
      <Button variant="primary" className="w-full">
        Get Started
      </Button>
    </div>
  )
}
```

**판결:** 여기가 브랜드가 사는 곳입니다. 일반 컴포넌트가 80%까지 데려감; 커스텀 컴포넌트가 100%로 만듭니다.

## 실제 예시: "GitHub Insights" SaaS 만들기

### 제품
GitHub 저장소 분석, 분석 표시, 인사이트 생성.

### 시간 분석

| 작업 | 시간 |
|------|------|
| 설정 (Tailwind + FramingUI + shadcn) | 2시간 |
| 랜딩 페이지 | 3시간 |
| 인증 화면 (Login, Signup) | 1.5시간 |
| 대시보드 shell | 2시간 |
| 분석 컴포넌트 (커스텀 차트) | 4시간 |
| 설정 페이지 | 1시간 |
| Dark mode | 1시간 |
| 반응형 + 다듬기 | 2시간 |
| **총** | **16.5시간** |

**결과:** 투자자나 초기 사용자에게 보여줄 준비가 된 전문적인 SaaS.

**Material-UI만 사용:** 아마 12~14시간, 하지만 일반적으로 보임.

**처음부터:** 40~60시간, 그리고 일관성 없을 가능성.

## 흔한 함정 (그리고 피하는 방법)

### 함정 1: shadcn 컴포넌트를 너무 일찍 커스터마이징

**실수:** shadcn 설치 후 즉시 Button.tsx 편집 시작

**문제:** 업스트림 업데이트와 버그 수정을 잃음

**해결:** shadcn을 2~4주 동안 그대로 사용. 명확하고 반복적인 필요가 생길 때만 커스터마이징.

### 함정 2: 토큰을 미리 정의하지 않음

**실수:** 컴포넌트 구축 시작, 진행하면서 색상 결정

**문제:** 일관성이 빠르게 침투

**해결:** 1시간을 들여 정의:
- 2~3개 브랜드 색상
- 1개 완전한 neutral 스케일 (50~900)
- 3개 시맨틱 색상 (success, error, warning)
- 5개 spacing 값 (xs, sm, md, lg, xl)
- 2개 font families (sans, mono)

### $50k MRR: 디자인 문서화 추가

## 결론

솔로 창업자나 작은 팀으로서 UI 스택은:

1. **빠르게 구축** (몇 주가 아닌 며칠에 출시)
2. **저렴** (이상적으로 $0)
3. **유연** (코드 소유)
4. **전문적** (템플릿처럼 보이지 않음)
5. **유지보수 가능** (카드로 만든 집이 아님)

**FramingUI + shadcn/ui + Tailwind**는 5가지 모두 충족합니다.

얻는 것:
- ✅ 속도 (Tailwind 유틸리티 + 복사-붙여넣기 컴포넌트)
- ✅ 비용 ($0)
- ✅ 유연성 (모든 코드 라인 소유)
- ✅ 전문성 (토큰이 일관성 강제)
- ✅ 유지보수성 (토큰 = single source of truth)

**이것이 2026년에 SaaS를 시작한다면 사용할 스택입니다.**

그리고 2개월이 아닌 2주에 출시할 것입니다.
