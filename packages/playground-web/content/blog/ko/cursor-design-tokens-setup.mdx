Cursor가 알림 카드를 5초 만에 만들어줬어요. 근데 색상, 패딩, 테두리 반경을 디자인 시스템에 맞게 고치는 데 5분이 걸렸죠. 이 비율 — 5초 생성, 5분 수정 — 이 바로 이 가이드가 해결하는 문제예요.

해결책은 `.cursorrules` 파일이에요. 설정에 15분이면 충분하고, 수동 수정 작업 대부분을 없애줘요.

## Cursor가 엉뚱한 걸 추측하는 이유

명시적인 규칙이 없으면 Cursor는 학습 데이터에서 가져와요. 유명 오픈소스 레포, 공개 컴포넌트 라이브러리, 일반적인 Tailwind 패턴들이죠. 내 프로젝트가 `blue-600` 대신 `indigo-600`을 쓰는지, `p-4` 대신 `p-6`을 쓰는지, `bg-white` 대신 `bg-neutral-50`을 쓰는지 Cursor는 알 방법이 없어요.

FramingUI 디자인 토큰은 CSS 변수예요 — `var(--background-page)`, `var(--foreground-primary)`, `var(--foreground-accent)` — JavaScript 객체가 아니고 하드코딩된 hex 값도 아니에요. Cursor에게 알려주지 않으면 절대 쓰지 않아요.

## 설치

pnpm으로 패키지를 설치해요:

```bash
pnpm add @framingui/ui @framingui/core @framingui/tokens tailwindcss-animate
```

앱의 전역 레이아웃에 스타일을 임포트해요:

```tsx
// app/layout.tsx (Next.js App Router)
import '@framingui/ui/styles';
import './globals.css';

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="ko">
      <body>{children}</body>
    </html>
  );
}
```

## Cursor 설정

### `.cursorrules` 만들기

프로젝트 루트에 `.cursorrules`를 만들어요. 이 파일이 Cursor의 코드베이스 설명서가 돼요.

```
# Design System: FramingUI

## 컴포넌트 라이브러리
항상 @framingui/ui에서 UI 컴포넌트를 임포트하세요:
- Button, Input, Card, Form, Dialog, Sheet, Tabs
- Label, Checkbox, RadioGroup, Select, Switch
- Table, Avatar, Badge, Tooltip, Popover

## 디자인 토큰
모든 스타일링에 CSS 커스텀 속성을 사용하세요. 색상이나 간격을 절대 하드코딩하지 마세요.

### 색상
- 주요 배경: var(--background-page)
- 주요 텍스트: var(--foreground-primary)
- 액션 primary: var(--foreground-accent)
- 뮤트 텍스트: var(--foreground-muted)

### 스타일링
토큰을 참조하는 Tailwind 클래스를 사용하세요:
- bg-[var(--background-page)]
- text-[var(--foreground-primary)]

## 코드 생성 규칙
1. 색상을 절대 하드코딩하지 않기 — 항상 var(--*) 사용
2. 로컬 파일이 아닌 @framingui/ui에서 컴포넌트 임포트
3. 색상 이름(blue, red)이 아닌 시맨틱 토큰 이름 사용
4. 원시 HTML 요소보다 FramingUI 컴포넌트 선호

## 예제 컴포넌트
```tsx
import { Button, Card, CardHeader, CardTitle, CardContent } from '@framingui/ui';

export function ExampleCard() {
  return (
    <Card>
      <CardHeader>
        <CardTitle>예제</CardTitle>
      </CardHeader>
      <CardContent>
        <Button variant="default">클릭하세요</Button>
      </CardContent>
    </Card>
  );
}
```
```

### 토큰 자동완성을 위한 VSCode 설정

`.vscode/settings.json`에 추가해요:

```json
{
  "editor.quickSuggestions": {
    "strings": true
  },
  "css.customData": [".vscode/css-custom-data.json"]
}
```

`.vscode/css-custom-data.json`을 만들어요:

```json
{
  "version": 1.1,
  "properties": [
    {
      "name": "--background-page",
      "description": "주요 캔버스 배경색"
    },
    {
      "name": "--foreground-primary",
      "description": "주요 텍스트 색상"
    },
    {
      "name": "--foreground-accent",
      "description": "주요 액션/브랜드 색상"
    }
  ]
}
```

이 파일을 만들고 나서 Cursor를 재시작해요.

## 통합 테스트

Cursor에게 요청해봐요: "이메일과 비밀번호 필드가 있는 로그인 폼을 FramingUI 컴포넌트로 만들어줘."

제대로 설정됐다면 이런 결과가 나와야 해요:

```tsx
import { Button, Input, Label, Card, CardHeader, CardTitle, CardContent } from '@framingui/ui';

export function LoginForm() {
  return (
    <Card className="w-full max-w-md">
      <CardHeader>
        <CardTitle>로그인</CardTitle>
      </CardHeader>
      <CardContent className="space-y-4">
        <div className="space-y-2">
          <Label htmlFor="email">이메일</Label>
          <Input id="email" type="email" placeholder="you@example.com" />
        </div>
        <div className="space-y-2">
          <Label htmlFor="password">비밀번호</Label>
          <Input id="password" type="password" />
        </div>
        <Button className="w-full" variant="default">
          로그인
        </Button>
      </CardContent>
    </Card>
  );
}
```

확인할 것: 하드코딩된 hex 값 없음, `@framingui/ui`에서 컴포넌트 임포트, 임의의 Tailwind 색상 클래스 없음.

## 문제 해결

**Cursor가 토큰 없이 일반 Tailwind를 생성해요:** `.cursorrules`를 업데이트하고 구체적인 예시를 추가해요. "배경색은 `bg-white`가 아닌 `bg-[var(--background-page)]`를 써"라고 명시적으로 적어요.

**토큰 자동완성이 안 돼요:** `.vscode/css-custom-data.json`을 추가한 후 Cursor를 재시작해요. Cursor가 시작하기 전에 파일이 있어야 해요.

**컴포넌트를 찾을 수 없어요:** `pnpm list @framingui/ui`로 설치를 확인해요. 없으면 설치 명령어를 다시 실행해요.

## 토큰 시스템 확장

FramingUI 기본 위에 프로젝트 고유 토큰을 추가할 수 있어요:

```css
/* globals.css */
@import '@framingui/ui/styles';

:root {
  --my-app-header-height: 64px;
  --my-app-sidebar-width: 280px;
}
```

이것도 `.cursorrules`와 `.vscode/css-custom-data.json`에 추가해서 Cursor가 알 수 있게 해요.

15분짜리 설정은 Cursor가 처음으로 올바른 컴포넌트를 생성하는 순간 본전을 찾아요. 그 이후로는 계속 복리로 돌아오는 거예요.
