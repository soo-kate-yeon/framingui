## AI 생성 UI 코드의 문제점

이런 경험 있으실 겁니다. Claude에게 사용자 프로필 카드를 만들어달라고 합니다. 깔끔하고 작동하는 코드가 나오지만 `#3B82F6` 같은 하드코딩된 색상과 `px-4` 같은 임의의 간격이 들어 있습니다. 코드베이스에 복붙하면 디자인 시스템과 전혀 맞지 않습니다.

왜 그럴까요? **AI 도구는 여러분의 디자인 결정에 대한 컨텍스트가 없습니다.** 브랜드의 특정 토큰, 색상 팔레트, 간격 스케일이 아닌 학습 데이터의 일반적인 패턴을 기반으로 생성합니다.

AI가 디자인 시스템을 읽고 매번 완벽하게 브랜드에 맞는 코드를 생성할 수 있다면요?

## 요약

- **Model Context Protocol (MCP)**은 AI 도구가 디자인 토큰에 프로그래밍 방식으로 접근하게 해줍니다
- **framingui**는 Claude Code, Cursor 등 MCP 호환 도구에 디자인 시스템을 노출하는 MCP 서버를 제공합니다
- 설정에 **5분**이 걸리고 모든 디자인 토큰 형식 (CSS 변수, Tailwind, styled-components)과 호환됩니다
- 결과: AI가 하드코딩된 값 대신 실제 디자인 토큰을 사용해 코드를 생성합니다
- 더 이상 수동 토큰 교체가 필요 없습니다. AI가 처음부터 디자인 시스템을 이해합니다

## Model Context Protocol (MCP)란?

Model Context Protocol은 AI 도구를 외부 데이터 소스에 연결하기 위한 Anthropic의 오픈 표준입니다. **AI 어시스턴트를 위한 USB 포트**라고 생각하세요. 매 프롬프트마다 컨텍스트를 복붙하지 않고도 AI가 정보를 쿼리할 수 있는 표준화된 방법을 제공합니다.

### MCP 작동 방식

```
┌─────────────────┐         MCP          ┌─────────────────┐
│   AI 어시스턴트   │ ◄──────────────────► │   MCP 서버      │
│  (Claude Code)  │   JSON-RPC 프로토콜  │ (디자인 토큰)     │
└─────────────────┘                       └─────────────────┘
        ▲                                          │
        │                                          │
        │                                          ▼
        │                                  ┌──────────────┐
        └──────────────────────────────────│  디자인       │
                실제 토큰을 사용해          │   시스템      │
                코드 생성                  └──────────────┘
```

**주요 이점:**

1. **컨텍스트 인식**: AI가 디자인 토큰을 온디맨드로 읽어 항상 최신 값을 가져옵니다
2. **컨텍스트 윈도우 낭비 없음**: 디자인 토큰이 프롬프트 컨텍스트를 소비하지 않습니다. 외부에서 가져옵니다
3. **자동 업데이트**: 토큰을 변경하면 AI가 즉시 새 값을 봅니다 (다시 붙여넣기 불필요)
4. **양방향**: AI가 토큰을 읽고 생성된 코드를 디자인 시스템에 대해 검증할 수 있습니다

## 디자인 토큰에 MCP 통합이 필요한 이유

### 기존 워크플로우 (고통스러움)

1. AI에게 컴포넌트 생성 요청
2. AI가 하드코딩된 값으로 코드 생성 (`bg-blue-500`, `#3B82F6`, `16px`)
3. 모든 값을 수동으로 디자인 토큰으로 교체 (`bg-primary-500`, `var(--color-primary)`, `spacing-4`)
4. AI가 생성하는 모든 컴포넌트에 반복
5. 일부 교체를 놓치면서 디자인 토큰이 점점 어긋남

**컴포넌트당 낭비되는 시간:** 5-10분
**결과:** 일관성 없는 토큰 사용, 디자인 시스템 위반

### MCP 워크플로우 (수월함)

1. AI 어시스턴트가 MCP로 디자인 토큰에 연결
2. AI에게 컴포넌트 생성 요청
3. AI가 사용 가능한 토큰 쿼리 (색상, 간격, 타이포그래피)
4. AI가 처음부터 실제 토큰을 사용해 코드 생성
5. 코드가 즉시 디자인 시스템 준수

**컴포넌트당 낭비되는 시간:** 0분
**결과:** 100% 토큰 준수, 수동 교체 제로

## framingui와 MCP 설정하기

### 사전 요구사항

- Node.js 20+ 설치됨
- Claude Code, Cursor 또는 MCP 호환 AI 도구
- 디자인 토큰이 있는 프로젝트 (또는 framingui로 생성)

### 1단계: framingui 설치

```bash
# framingui 코어 패키지 설치
npm install @framingui/core @framingui/mcp-server

# yarn 사용 시
yarn add @framingui/core @framingui/mcp-server

# pnpm 사용 시
pnpm add @framingui/core @framingui/mcp-server
```

### 2단계: 디자인 토큰 생성 (선택)

디자인 토큰이 아직 없다면 framingui가 생성할 수 있습니다:

```bash
# 대화형 프롬프트로 framingui 초기화
npx @framingui/cli init

# 프롬프트를 따르세요:
# - 주요 색상 (OKLCH 형식)
# - 대비 수준 (AA 또는 AAA)
# - 보더 레디어스 스타일
# - 타이포그래피 스케일
```

생성되는 것:
- `tokens.css` - CSS 커스텀 프로퍼티
- `tokens.json` - MCP용 토큰 메타데이터
- `tailwind.config.js` - Tailwind 통합 (감지된 경우)

**생성된 토큰 예시:**

```css
/* tokens.css */
:root {
  /* 프라이머리 색상 스케일 */
  --color-primary-50: oklch(0.95 0.03 240);
  --color-primary-500: oklch(0.5 0.15 240);
  --color-primary-900: oklch(0.25 0.08 240);

  /* 간격 스케일 */
  --spacing-1: 0.25rem;
  --spacing-4: 1rem;
  --spacing-6: 1.5rem;

  /* 컴포넌트 토큰 */
  --button-bg-primary: var(--color-primary-500);
  --button-fg-primary: oklch(0.98 0.01 240);
}
```

### 3단계: MCP 서버 설정

MCP 설정 파일을 생성하거나 업데이트하세요. 위치는 AI 도구에 따라 다릅니다:

**Claude Code:**

```json
// .claude/mcp.json
{
  "servers": {
    "tekton-design-tokens": {
      "command": "npx",
      "args": ["@framingui/mcp-server"],
      "env": {
        "TEKTON_PROJECT_PATH": ".",
        "TEKTON_TOKEN_PATH": "./tokens.json"
      }
    }
  }
}
```

**Cursor:**

```json
// .cursor/mcp.json
{
  "mcpServers": {
    "tekton-design-tokens": {
      "command": "npx",
      "args": ["@framingui/mcp-server"],
      "env": {
        "TEKTON_PROJECT_PATH": ".",
        "TEKTON_TOKEN_PATH": "./tokens.json"
      }
    }
  }
}
```

**설정 옵션:**

- `TEKTON_PROJECT_PATH`: 프로젝트 루트 디렉토리 (기본값: 현재 디렉토리)
- `TEKTON_TOKEN_PATH`: 토큰 메타데이터 JSON 경로 (기본값: `./tokens.json`)
- `TEKTON_STRICT_MODE`: 토큰이 없으면 실패 (기본값: `false`)

### 4단계: MCP 연결 확인

AI 도구를 재시작하고 MCP 연결을 확인하세요:

**Claude Code에서:**

```
You: 사용 가능한 디자인 토큰 목록을 보여줘

Claude: MCP로 FramingUI 디자인 토큰을 쿼리하겠습니다.

[Claude가 MCP 서버 쿼리]

사용 가능한 디자인 토큰:
- 색상 스케일: primary (10단계), success, warning, error
- 간격: 1, 2, 3, 4, 6, 8, 12, 16
- 타이포그래피: xs, sm, base, lg, xl, 2xl, 3xl
- 컴포넌트 토큰: button, card, input, badge
```

**Cursor에서:**

채팅에서 `@framingui`를 입력하면 사용 가능한 MCP 명령어가 보입니다:
- `@framingui/tokens` - 모든 토큰 목록
- `@framingui/components` - 컴포넌트 지식 목록
- `@framingui/validate` - 디자인 시스템에 대해 코드 검증

### 5단계: AI로 컴포넌트 생성

이제 AI가 실제 디자인 토큰을 사용해 컴포넌트를 생성할 수 있습니다:

**프롬프트:**

```
디자인 토큰을 사용해서 사용자 프로필 카드 컴포넌트 만들어줘
```

**AI 응답 (MCP 없이):**

```tsx
export function ProfileCard() {
  return (
    <div className="p-4 bg-white rounded-lg shadow-md">
      <img src="/avatar.jpg" className="w-16 h-16 rounded-full" />
      <h3 className="text-lg font-semibold text-gray-900">John Doe</h3>
      <p className="text-gray-600">Software Engineer</p>
      <button className="mt-4 px-4 py-2 bg-blue-500 text-white rounded">
        View Profile
      </button>
    </div>
  );
}
```

**AI 응답 (MCP 사용):**

```tsx
import { tokens } from '@/design-system/tokens';

export function ProfileCard() {
  return (
    <div
      style={{
        padding: tokens.spacing[4],
        background: tokens.bg.surface.elevated,
        borderRadius: tokens.radius.lg,
        boxShadow: tokens.shadow.md,
      }}
    >
      <img
        src="/avatar.jpg"
        style={{
          width: tokens.spacing[16],
          height: tokens.spacing[16],
          borderRadius: tokens.radius.full,
        }}
      />
      <h3
        style={{
          fontSize: tokens.fontSize.lg,
          fontWeight: tokens.fontWeight.semibold,
          color: tokens.fg.primary,
        }}
      >
        John Doe
      </h3>
      <p style={{ color: tokens.fg.secondary }}>Software Engineer</p>
      <button
        style={{
          marginTop: tokens.spacing[4],
          padding: `${tokens.spacing[2]} ${tokens.spacing[4]}`,
          background: tokens.button.bg.primary,
          color: tokens.button.fg.primary,
          borderRadius: tokens.radius.md,
        }}
      >
        View Profile
      </button>
    </div>
  );
}
```

주목하세요:
- 하드코딩된 색상이나 간격 없음
- 모든 값이 실제 디자인 토큰 참조
- 컴포넌트가 디자인 시스템과 정확히 일치
- 수동 토큰 교체 불필요

## 고급 MCP 사용 패턴

### 패턴 1: 컴포넌트 지식 통합

framingui의 MCP 서버는 토큰뿐만 아니라 **컴포넌트 지식**도 노출합니다. 컴포넌트를 언제 어떻게 사용하는지에 대한 시맨틱 메타데이터입니다.

**컴포넌트 메타데이터 쿼리:**

```
You: primary 버튼과 ghost 버튼 variant의 차이가 뭐야?

Claude: [MCP 컴포넌트 지식 쿼리]

버튼 Variant:
- primary: 메인 콜투액션, 높은 시각적 강조
  - 사용 사례: 컨텍스트 내 주요 액션 (화면당 최대 하나)
  - 토큰: button.bg.primary, button.fg.primary
  - 제약: 뷰포트당 최대 하나의 primary 버튼

- ghost: 부수적 작업을 위한 낮은 강조 액션
  - 사용 사례: primary CTA와 경쟁하면 안 되는 액션
  - 토큰: button.bg.ghost (투명), button.fg.ghost
  - 시각적 임팩트: 낮음 (배경 채움 없음)
```

**컨텍스트를 활용한 컴포넌트 생성:**

```
You: 저장과 취소 액션이 있는 모달 푸터 만들어줘

Claude: [컴포넌트 지식에서 save = primary, cancel = ghost 이해]
```

```tsx
import { Button } from '@/components/button';
import { tokens } from '@/design-system/tokens';

export function ModalFooter({ onSave, onCancel }) {
  return (
    <div
      style={{
        display: 'flex',
        gap: tokens.spacing[2],
        justifyContent: 'flex-end',
        padding: tokens.spacing[4],
      }}
    >
      <Button variant="ghost" onClick={onCancel}>
        Cancel
      </Button>
      <Button variant="primary" onClick={onSave}>
        Save Changes
      </Button>
    </div>
  );
}
```

AI가 자동으로:
- primary 버튼을 오른쪽에 배치 (모달 관례)
- cancel에 ghost variant 사용 (낮은 강조)
- 버튼 사이에 올바른 간격 적용
- "뷰포트당 최대 하나의 primary" 제약 준수

### 패턴 2: 토큰 검증

MCP를 사용해 기존 코드를 디자인 시스템에 대해 검증하세요:

**프롬프트:**

```
이 컴포넌트의 토큰 준수를 검증해줘:

const Card = () => (
  <div style={{ background: '#ffffff', padding: '16px' }}>
    <h2 style={{ color: '#1a1a1a' }}>Title</h2>
  </div>
);
```

**AI 응답:**

```
토큰 준수 위반 발견:

1. 2번 줄: 하드코딩된 색상 `#ffffff`
   → 교체: tokens.bg.surface.base

2. 2번 줄: 하드코딩된 간격 `16px`
   → 교체: tokens.spacing[4]

3. 3번 줄: 하드코딩된 색상 `#1a1a1a`
   → 교체: tokens.fg.primary

수정 제안:
```

```tsx
const Card = () => (
  <div
    style={{
      background: tokens.bg.surface.base,
      padding: tokens.spacing[4],
    }}
  >
    <h2 style={{ color: tokens.fg.primary }}>Title</h2>
  </div>
);
```

### 패턴 3: 멀티 테마 생성

여러 테마에서 작동하는 컴포넌트 생성:

**프롬프트:**

```
라이트와 다크 테마를 토글하는 테마 스위처 만들어줘
```

**AI 응답 (MCP로 테마 토큰 쿼리):**

```tsx
import { tokens } from '@/design-system/tokens';
import { useState } from 'react';

export function ThemeSwitcher() {
  const [theme, setTheme] = useState<'light' | 'dark'>('light');

  const toggleTheme = () => {
    const newTheme = theme === 'light' ? 'dark' : 'light';
    setTheme(newTheme);
    document.documentElement.dataset.theme = newTheme;
  };

  return (
    <button
      onClick={toggleTheme}
      style={{
        padding: `${tokens.spacing[2]} ${tokens.spacing[3]}`,
        background: tokens.button.bg.ghost,
        color: tokens.button.fg.ghost,
        border: `1px solid ${tokens.border.default}`,
        borderRadius: tokens.radius.md,
        cursor: 'pointer',
      }}
    >
      {theme === 'light' ? '🌙 Dark Mode' : '☀️ Light Mode'}
    </button>
  );
}
```

```css
/* AI가 토큰 메타데이터에서 테마 인식 CSS 생성 */
[data-theme='light'] {
  --bg-surface-base: oklch(0.98 0.01 240);
  --fg-primary: oklch(0.25 0.03 240);
}

[data-theme='dark'] {
  --bg-surface-base: oklch(0.18 0.01 240);
  --fg-primary: oklch(0.92 0.02 240);
}
```

AI가 자동으로:
- 테마 전환 로직 생성
- 테마에 맞게 적응하는 시맨틱 토큰 사용
- 양쪽 테마에 해당하는 CSS 생성
- 양쪽 모드에서 WCAG 대비 요구사항 준수

## 기존 디자인 시스템과 통합

framingui의 토큰 생성기를 사용할 필요 없습니다. MCP는 모든 디자인 토큰 형식과 작동합니다.

### 기존 디자인 토큰에서

이미 디자인 토큰이 있다면 (Figma Tokens 플러그인이나 다른 소스에서):

```bash
# 토큰을 JSON으로 내보내기
# W3C Design Token 형식을 따라 구조화
# 또는 framingui 템플릿을 시작점으로 사용
```

```json
// .claude/mcp.json
{
  "servers": {
    "tekton": {
      "command": "npx",
      "args": ["@framingui/mcp-server"],
      "env": {
        "TEKTON_TOKEN_PATH": "./design-tokens.json"
      }
    }
  }
}
```

### Style Dictionary에서

Style Dictionary를 사용한다면:

```bash
# Style Dictionary로 토큰 빌드
npm run build-tokens

# 출력: build/tokens.json

# MCP를 Style Dictionary 출력으로 지정
```

```json
{
  "servers": {
    "tekton-design-tokens": {
      "command": "npx",
      "args": ["@framingui/mcp-server", "--format", "style-dictionary"],
      "env": {
        "TEKTON_TOKEN_PATH": "./build/tokens.json"
      }
    }
  }
}
```

### Tailwind Config에서

Tailwind CSS를 사용한다면:

```bash
# tailwind.config.js에서 토큰 추출
```

## 일반적인 문제 해결

### 문제 1: MCP 서버가 연결되지 않음

**증상:** AI 도구가 MCP 명령어를 인식하지 못함

**해결책:**

```bash
# MCP 서버가 올바르게 시작되는지 확인
npx @framingui/mcp-server --test

# AI 도구 로그에서 오류 확인
# Claude Code: Help → Toggle Developer Tools → Console
# Cursor: View → Output → "MCP Servers" 선택

# TEKTON_PROJECT_PATH가 절대 경로인지 확인
{
  "env": {
    "TEKTON_PROJECT_PATH": "/absolute/path/to/project"
  }
}
```

### 문제 2: 토큰을 찾을 수 없음

**증상:** AI가 쿼리할 때 "사용 가능한 토큰 없음"이라고 함

**해결책:**

```bash
# tokens.json이 존재하고 유효한 JSON인지 확인
cat tokens.json | jq

# MCP 설정에서 토큰 파일 경로 확인
# 경로는 TEKTON_PROJECT_PATH에 상대적이어야 함
```

### 문제 3: AI가 여전히 하드코딩된 값 생성

**증상:** AI가 토큰 참조 대신 `#ffffff` 생성

**해결책:**

프롬프트에 명시적 지시 추가:

```
MCP 서버의 디자인 토큰만 사용해서 카드 컴포넌트 만들어줘.
하드코딩된 색상, 간격, 디자인 값은 사용하지 마.
```

또는 strict 모드 설정:

```json
{
  "env": {
    "TEKTON_STRICT_MODE": "true"
  }
}
```

이렇게 하면 AI가 토큰 없이 코드를 생성하려 할 때 MCP 서버가 오류를 반환합니다.

## FAQ

### Claude 외의 AI 도구에서도 MCP를 사용할 수 있나요?

네! MCP는 오픈 표준입니다. Model Context Protocol을 지원하는 모든 도구가 framingui의 MCP 서버에 연결할 수 있습니다. 현재 지원:

- Claude Code (공식)
- Cursor (실험적 MCP 지원)
- Cline / Windsurf (커뮤니티 지원)
- MCP 클라이언트 명세를 구현한 모든 도구

### MCP가 AI 응답을 느리게 하나요?

아니요. MCP 쿼리는 AI 생성과 병렬로 수행되며 일반적으로 50ms 미만으로 완료됩니다. AI가 MCP를 기다리지 않습니다. 토큰 데이터를 가져오는 동안 응답을 스트리밍합니다.

### 토큰이 자주 변경되면요?

MCP가 토큰을 온디맨드로 읽으므로 AI는 항상 최신 값을 봅니다. 토큰이 변경될 때 AI 도구를 재시작하거나 MCP를 다시 설정할 필요 없습니다. 토큰 파일을 저장하면 AI가 다음 쿼리에서 변경사항을 가져옵니다.

### AI가 MCP로 새 토큰을 생성할 수 있나요?

아직은 아닙니다. MCP는 현재 디자인 토큰에 대한 읽기 전용 접근을 제공합니다. 토큰 생성은 여전히 framingui CLI나 기존 디자인 시스템 워크플로우를 사용해야 합니다. 양방향 MCP (읽기 + 쓰기)는 향후 릴리스에 계획되어 있습니다.

### shadcn/ui 같은 컴포넌트 라이브러리와 작동하나요?

네! framingui의 MCP 서버는 shadcn/ui 컴포넌트에 대한 컴포넌트 지식을 포함합니다. AI에게 컴포넌트를 생성해달라고 하면 자동으로:

1. MCP로 사용 가능한 shadcn 컴포넌트 쿼리
2. 적절한 컴포넌트 variant 선택
3. 컴포넌트에 디자인 토큰 적용
4. shadcn의 컴포지션 패턴 따르기

예시:

```
You: 정렬 기능이 있는 데이터 테이블 만들어줘

AI: [MCP로 DataTable 컴포넌트 지식 쿼리]
    [토큰이 적용된 shadcn Table 컴포넌트 생성]
```

### 커스텀 컴포넌트 지식을 어떻게 추가하나요?

`component-knowledge.json` 파일을 생성하세요:

```json
{
  "components": [
    {
      "name": "CustomCard",
      "category": "layout",
      "purpose": "elevated 컨테이너에 콘텐츠 표시",
      "variants": [
        {
          "name": "elevated",
          "tokens": {
            "bg": "bg.surface.elevated",
            "shadow": "shadow.md"
          }
        }
      ]
    }
  ]
}
```

MCP가 로드하도록 설정:

```json
{
  "env": {
    "TEKTON_COMPONENT_KNOWLEDGE_PATH": "./component-knowledge.json"
  }
}
```

이제 AI가 커스텀 컴포넌트를 이해하고 생성된 코드에서 사용합니다.
