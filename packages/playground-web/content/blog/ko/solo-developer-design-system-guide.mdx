## 혼자 개발하고 있다 (그래도 괜찮다)

1인 개발자로 일하고 계신가요? SaaS 제품을 만들거나, 사이드 프로젝트를 하거나, 프리랜서로 활동 중이실 겁니다. 코드도 직접 짜고, 배포도 관리하고, 지원도 처리하면서 *동시에* UI까지 전문적으로 만들어야 합니다.

**문제:** 디자인 배경이 없습니다. Tailwind 문서에서 클래스를 복붙합니다. 버튼 스타일이 12페이지에 걸쳐 7가지나 됩니다. 색상 팔레트는 "그때그때 괜찮아 보이는 것"입니다.

**해결책:** 디자이너가 필요한 게 아닙니다. **디자인 시스템**이 필요합니다. 오후 한나절이면 만들 수 있습니다.

## 요약: 2시간 만에 완성하는 디자인 시스템

**만들게 될 것:**
- 전문적인 색상 팔레트 (WCAG AA 준수)
- 일관된 간격과 타이포그래피 스케일
- 재사용 가능한 컴포넌트 토큰
- 자동 토큰 검증 (스타일 불일치 방지)

**얻게 될 것:**
- 디자이너가 만든 것 같은 UI (디자이너가 아니어도)
- 브랜드에 맞는 컴포넌트를 자동 생성하는 AI 도구
- "스타일 조정하다 시간 다 보내기" 제로
- 디자인 리뷰 없이도 UI를 배포할 수 있는 자신감

**시간 투자:** 설정 2시간, 유지보수 0시간 (토큰이 알아서 강제)

**기술 요구사항:**
- 기본적인 React 지식
- 커맨드 라인 사용 경험
- 아무 텍스트 에디터

**디자인 스킬은 필요 없습니다.** 진심입니다.

## 1인 개발자에게 디자인 시스템이 필요한 이유

### 문제: 점점 퍼지는 비일관성

프로젝트 1일차:
```tsx
<button className="bg-blue-500 px-4 py-2 rounded">Submit</button>
```

15일차:
```tsx
<button className="bg-blue-600 px-6 py-3 rounded-lg">Continue</button>
```

30일차:
```tsx
<button style={{ background: '#3B82F6', padding: '12px 24px' }}>Next</button>
```

**파란색이 세 가지. 크기도 세 가지. 보더 레디어스도 세 가지.**

이제 이걸 50개 컴포넌트, 200개 파일, 6개월 개발 기간에 적용해 보세요.

**결과:** UI가 세 명이 각각 디자인한 것처럼 보입니다. 실제로 그렇거든요 (과거의 나, 현재의 나, 새벽 3시에 지친 나).

### 디자인 시스템이 해결하는 것

디자인 토큰을 쓰면 이렇게 작성합니다:

```tsx
import { tokens } from '@/design-system/tokens';

<button style={{
  background: tokens.button.bg.primary,
  padding: `${tokens.spacing[2]} ${tokens.spacing[4]}`,
  borderRadius: tokens.radius.md,
}}>
  Submit
</button>
```

**모든 버튼이 이제:**
- 같은 파란색 (토큰에서)
- 같은 간격 (스케일에서)
- 같은 보더 레디어스 (토큰에서)

**장점:**
- ✅ 앱 전체에서 일관된 UI
- ✅ 토큰 하나 바꾸면 전체 업데이트
- ✅ AI 도구가 브랜드에 맞는 컴포넌트 생성
- ✅ "이 파란색이 저 파란색이랑 맞나?" 더 이상 안 함

## 2시간 설정 가이드

### 1시간차: 디자인 토큰 생성

Figma를 건드리지 않고 전문적인 색상 팔레트, 간격 스케일, 타이포그래피 시스템을 만들겠습니다.

#### 1단계: framingui 설치 (5분)

```bash
# 새 Next.js 프로젝트 생성 (또는 기존 프로젝트 사용)
npx create-next-app@latest my-app
cd my-app

# framingui 설치
npm install @framingui/core

# yarn을 사용한다면
yarn add @framingui/core

# pnpm을 사용한다면
pnpm add @framingui/core
```

#### 2단계: 7가지 간단한 질문에 답하기 (10분)

```bash
npx @framingui/cli init
```

CLI가 질문을 합니다. 디자인 경험 없이 답하는 방법입니다:

**질문 1: 주요 브랜드 색상은?**

모르겠다면 검증된 색상 심리학 가이드라인에서 고르세요:

- **테크/SaaS**: 파란색 (신뢰, 전문성)
  - 추천: `{ l: 0.5, c: 0.15, h: 240 }`

- **건강/웰니스**: 초록색 (건강, 성장)
  - 추천: `{ l: 0.55, c: 0.12, h: 140 }`

- **금융**: 진한 파란색 또는 보라색 (신뢰, 안정)
  - 추천: `{ l: 0.45, c: 0.13, h: 260 }`

- **이커머스**: 주황색 또는 빨간색 (에너지, 긴급성)
  - 추천: `{ l: 0.6, c: 0.15, h: 30 }`

**질문 2: 대비 수준은?**

- **AA** 선택 (표준 접근성)
- 의료/정부 서비스용이라면 AAA 선택 (더 엄격한 규정)

**질문 3: 보더 레디어스 스타일은?**

- **Minimal** (0-4px): 전문적이고 진지한 제품 (핀테크, B2B)
- **Moderate** (6-8px): 표준 SaaS, 균형 잡힌 느낌
- **Rounded** (12-16px): 소비자 앱, 친근한 느낌

**질문 4: 간격 밀도는?**

- **Compact**: 데이터 많은 대시보드, 파워 유저
- **Comfortable** (권장): 표준 앱, 일반 사용
- **Spacious**: 마케팅 사이트, 콘텐츠 중심

**질문 5: 타이포그래피 스케일은?**

- **Medium** 선택 (나중에 커스터마이징 가능)

**질문 6: 다크 모드?**

- **Yes** 선택 (무료이고, 대부분의 사용자가 기대함)

**질문 7: 프레임워크?**

- 프레임워크 선택 (Next.js, Vite, Remix 등)
- framingui가 가능하면 자동 감지함

#### 3단계: 생성된 토큰 확인 (5분)

framingui가 방금 생성한 것:

```
my-app/
├── tokens.css           # CSS 변수로 된 디자인 토큰
├── tokens.json          # 토큰 메타데이터 (도구용)
├── tailwind.config.js   # Tailwind 통합 (감지된 경우)
└── components/
    └── tokens.ts        # TypeScript 토큰 정의
```

`tokens.css`를 열어보세요:

```css
:root {
  /* 프라이머리 색상 스케일 (10단계, WCAG 준수) */
  --color-primary-50: oklch(0.95 0.03 240);
  --color-primary-100: oklch(0.90 0.06 240);
  --color-primary-500: oklch(0.50 0.15 240);  /* 메인 브랜드 색상 */
  --color-primary-900: oklch(0.25 0.08 240);

  /* 간격 스케일 (4px 그리드 기반) */
  --spacing-1: 0.25rem;  /* 4px */
  --spacing-2: 0.5rem;   /* 8px */
  --spacing-4: 1rem;     /* 16px */
  --spacing-6: 1.5rem;   /* 24px */

  /* 컴포넌트 토큰 (시맨틱) */
  --button-bg-primary: var(--color-primary-500);
  --button-fg-primary: oklch(0.98 0.01 240);  /* 대비 보장됨 */
}
```

**방금 일어난 일:**

1. framingui가 주요 색상에서 10단계 색상 스케일 생성
2. 모든 색상이 자동으로 WCAG AA 대비 요구사항 충족
3. 간격 스케일이 수학적 진행을 따름 (임의적이지 않음)
4. 컴포넌트 토큰이 시맨틱 사용 사례에 매핑

**디자인 지식이 필요 없습니다.** 이게 프로급입니다.

#### 4단계: 시맨틱 토큰 추가 (20분)

생성된 토큰은 기본입니다. 이제 시맨틱 의미를 추가합니다:

```css
/* tokens.css (이것들을 추가하세요) */

:root {
  /* 시맨틱 배경 토큰 */
  --bg-page: var(--color-neutral-50);
  --bg-surface: white;
  --bg-surface-elevated: white;

  /* 시맨틱 전경 토큰 */
  --fg-primary: var(--color-neutral-900);
  --fg-secondary: var(--color-neutral-600);
  --fg-tertiary: var(--color-neutral-500);

  /* 시맨틱 테두리 토큰 */
  --border-default: var(--color-neutral-200);
  --border-strong: var(--color-neutral-400);

  /* 시맨틱 상태 토큰 */
  --state-success: var(--color-success-500);
  --state-warning: var(--color-warning-500);
  --state-error: var(--color-error-500);
  --state-info: var(--color-primary-500);
}

/* 다크 모드 오버라이드 */
[data-theme='dark'] {
  --bg-page: var(--color-neutral-900);
  --bg-surface: var(--color-neutral-800);
  --bg-surface-elevated: var(--color-neutral-750);

  --fg-primary: var(--color-neutral-50);
  --fg-secondary: var(--color-neutral-400);
  --fg-tertiary: var(--color-neutral-500);

  --border-default: var(--color-neutral-700);
  --border-strong: var(--color-neutral-600);
}
```

**팁:** 색상 이름(`gray-100`, `blue-500`) 대신 시맨틱 이름(`bg-surface`, `fg-primary`)을 사용하세요. 색상 팔레트를 바꿀 때 토큰만 업데이트하면 됩니다. 코드 변경 없이요.

#### 5단계: 토큰 헬퍼 만들기 (20분)

React에서 토큰을 쉽게 사용하도록 만듭니다:

```typescript
// lib/tokens.ts

export const tokens = {
  // 색상
  bg: {
    page: 'var(--bg-page)',
    surface: {
      base: 'var(--bg-surface)',
      elevated: 'var(--bg-surface-elevated)',
    },
  },
  fg: {
    primary: 'var(--fg-primary)',
    secondary: 'var(--fg-secondary)',
    tertiary: 'var(--fg-tertiary)',
  },
  border: {
    default: 'var(--border-default)',
    strong: 'var(--border-strong)',
  },

  // 간격
  spacing: {
    1: 'var(--spacing-1)',
    2: 'var(--spacing-2)',
    3: 'var(--spacing-3)',
    4: 'var(--spacing-4)',
    6: 'var(--spacing-6)',
    8: 'var(--spacing-8)',
  },

  // 타이포그래피
  fontSize: {
    xs: 'var(--font-size-xs)',
    sm: 'var(--font-size-sm)',
    base: 'var(--font-size-base)',
    lg: 'var(--font-size-lg)',
    xl: 'var(--font-size-xl)',
    '2xl': 'var(--font-size-2xl)',
  },

  // 보더 레디어스
  radius: {
    sm: 'var(--radius-sm)',
    md: 'var(--radius-md)',
    lg: 'var(--radius-lg)',
    full: 'var(--radius-full)',
  },

  // 그림자
  shadow: {
    sm: 'var(--shadow-sm)',
    md: 'var(--shadow-md)',
    lg: 'var(--shadow-lg)',
  },

  // 컴포넌트 토큰
  button: {
    bg: {
      primary: 'var(--button-bg-primary)',
      secondary: 'var(--button-bg-secondary)',
      ghost: 'transparent',
    },
    fg: {
      primary: 'var(--button-fg-primary)',
      secondary: 'var(--button-fg-secondary)',
      ghost: 'var(--fg-primary)',
    },
  },
} as const;

// TypeScript: 안전을 위해 토큰을 readonly로 만듦
export type Tokens = typeof tokens;
```

이제 컴포넌트에서 토큰 사용:

```tsx
import { tokens } from '@/lib/tokens';

export function Card({ children }) {
  return (
    <div style={{
      background: tokens.bg.surface.elevated,
      padding: tokens.spacing[6],
      borderRadius: tokens.radius.lg,
      boxShadow: tokens.shadow.md,
    }}>
      {children}
    </div>
  );
}
```

**축하합니다!** 전문적인 디자인 시스템이 생겼습니다. 1시간차 완료.

#### 8단계: AI 통합 테스트 (10분)

Claude Code 또는 Cursor에서:

```
You: 사용 가능한 디자인 토큰 목록을 보여줘

Claude: [MCP 서버 쿼리]

사용 가능한 디자인 토큰:
- 색상: primary (10단계), success, warning, error
- 간격: 1, 2, 3, 4, 6, 8, 12, 16
- 타이포그래피: xs, sm, base, lg, xl, 2xl
- 컴포넌트 토큰: button, card, input
```

이제 컴포넌트를 생성합니다:

```
You: 가격 카드 컴포넌트 만들어줘

Claude: [MCP로 디자인 토큰 쿼리]
```

import { tokens } from '@/lib/tokens';

export function PricingCard({ tier, price, features }) {
  return (
    <div
      style={{
        background: tokens.bg.surface.elevated,
        padding: tokens.spacing[6],
        borderRadius: tokens.radius.lg,
        boxShadow: tokens.shadow.md,

      }}
    >
      <h3
        style={{
          fontSize: tokens.fontSize.xl,
          fontWeight: 'bold',
          color: tokens.fg.primary,
        }}
      >

      </h3>

      <p
        style={{
          fontSize: tokens.fontSize['2xl'],
          fontWeight: 'bold',
          color: tokens.fg.primary,
          marginTop: tokens.spacing[4],
        }}
      >
        ${price}<span style={{ fontSize: tokens.fontSize.sm }}>/mo</span>
      </p>

      <ul style={{ marginTop: tokens.spacing[6] }}>
        {features.map((feature) => (
          <li
            key={feature}
            style={{
              color: tokens.fg.secondary,
              marginTop: tokens.spacing[2],
            }}
          >
            {feature}
          </li>
        ))}
      </ul>

      <button
        style={{
          marginTop: tokens.spacing[6],
          width: '100%',
          background: tokens.button.bg.primary,
          color: tokens.button.fg.primary,
          padding: `${tokens.spacing[3]} ${tokens.spacing[4]}`,
          borderRadius: tokens.radius.md,
          border: 'none',
          cursor: 'pointer',
        }}
      >
        Get Started
      </button>
    </div>
  );

```

**주목:** 하드코딩된 값이 없습니다. 모든 토큰이 디자인 시스템에서 옵니다. AI가 자동 생성했습니다.

#### 9단계: 토큰 준수 강제 (25분)

본인이나 AI가 앞으로 하드코딩된 값을 추가하는 걸 막습니다.

esbuild 플러그인 설치:

```bash
npm install @framingui/esbuild-plugin
```

빌드 타임 검증 설정:

```typescript
// next.config.js (Next.js용)
const { tektonPlugin } = require('@framingui/esbuild-plugin');

module.exports = {
  experimental: {
    esmExternals: true,
  },
  webpack: (config, { isServer }) => {
    if (!isServer) {
      config.plugins.push(
        tektonPlugin({
          strict: process.env.NODE_ENV === 'production',
          threshold: 90, // 10% 비토큰 스타일 허용 (서드파티 라이브러리용)
        })
      );
    }
    return config;
  },
};
```

이제 본인이나 AI가 이렇게 작성하면:

```tsx
<button style={{ background: '#3B82F6' }}>Click</button>
```

빌드가 실패합니다:

```
❌ 토큰 준수 위반

파일: components/button.tsx:12
문제: 하드코딩된 색상 값 '#3B82F6'
예상: tokens.button.bg.primary 사용

수정:
<button style={{ background: tokens.button.bg.primary }}>Click</button>
```

**결과:** 디자인 시스템이 영원히 일관성을 유지합니다. 수동 강제가 필요 없습니다.

#### 10단계: 시스템 문서화 (10분)

미래의 자신을 위한 간단한 README 작성:

```markdown
# 디자인 시스템

## 색상

- Primary: 파란색 (#[FramingUI가 생성])
- Success: 초록색
- Warning: 주황색
- Error: 빨간색

## 간격 스케일

4px 그리드 기반:
- 1 = 4px
- 2 = 8px
- 4 = 16px
- 6 = 24px

## 사용법

```tsx
import { tokens } from '@/lib/tokens';

<div style={{
  background: tokens.bg.surface.base,
  padding: tokens.spacing[4],
  borderRadius: tokens.radius.md,
}}>
  Content
</div>
```

## 토큰 참조

- `tokens.bg.*` - 배경 색상
- `tokens.fg.*` - 전경 (텍스트) 색상
- `tokens.spacing.*` - 간격 값
- `tokens.fontSize.*` - 타이포그래피 스케일
- `tokens.radius.*` - 보더 레디어스
- `tokens.shadow.*` - 박스 섀도우
- `tokens.button.*` - 버튼 전용 토큰
```

**끝입니다.** 2시간. 전문적인 디자인 시스템이 생겼습니다.

## 1인 개발자를 위한 일반적인 패턴

### 패턴 1: 컴포넌트 라이브러리 지름길

모든 컴포넌트를 처음부터 만들지 마세요. shadcn/ui를 토큰과 함께 사용하세요:

```bash
# shadcn/ui 설치
npx shadcn-ui@latest init

# 컴포넌트 생성
npx shadcn-ui@latest add button card input
```

그다음 토큰으로 오버라이드:

```tsx
// components/ui/button.tsx
import { tokens } from '@/lib/tokens';

export function Button({ children, variant = 'primary' }) {
  const styles = {
    primary: {
      background: tokens.button.bg.primary,
      color: tokens.button.fg.primary,
    },
    secondary: {
      background: tokens.button.bg.secondary,
      color: tokens.button.fg.secondary,
    },
  };

  return (
    <button
      style={{
        ...styles[variant],
        padding: `${tokens.spacing[2]} ${tokens.spacing[4]}`,
        borderRadius: tokens.radius.md,
        border: 'none',
        cursor: 'pointer',
      }}
    >
      {children}
    </button>
  );
}
```

**결과:** 토큰을 사용한 전문적인 컴포넌트를 몇 분 만에.

### 패턴 2: 색상 팔레트 검증기

색상이 잘 어울리는지 확신이 없나요? framingui가 자동 검증합니다:

```bash
npx @framingui/cli validate

✅ 모든 색상 조합이 WCAG AA 통과
✅ Primary + 배경: 8.2:1 대비 (4.5:1 요구사항 초과)
✅ Secondary + 배경: 5.1:1 대비
⚠️  Tertiary + 배경: 3.8:1 대비 (일반 텍스트용 4.5:1 미달)

제안:
- Tertiary 색상은 큰 텍스트에만 사용 (3:1 요구사항 통과)
- 또는 일반 텍스트 준수를 위해 tertiary를 5% 더 어둡게
```

**디자인 학위가 필요 없습니다.** 수학이 접근성을 처리합니다.

### 패턴 3: 테마 스위처

라이트 + 다크 모드가 필요하신가요? 이미 처리됐습니다:

```tsx
// components/theme-switcher.tsx
export function ThemeSwitcher() {
  const [theme, setTheme] = useState('light');

  const toggle = () => {
    const newTheme = theme === 'light' ? 'dark' : 'light';
    setTheme(newTheme);
    document.documentElement.dataset.theme = newTheme;
  };

  return (
    <button onClick={toggle}>
      {theme === 'light' ? '🌙' : '☀️'}
    </button>
  );
}
```

토큰이 자동으로 전환됩니다:

```css
/* 라이트 모드 (기본) */
:root {
  --bg-page: #fafafa;
  --fg-primary: #1a1a1a;
}

/* 다크 모드 (자동) */
[data-theme='dark'] {
  --bg-page: #1a1a1a;
  --fg-primary: #fafafa;
}
```

모든 컴포넌트가 자동 업데이트됩니다 (하드코딩된 색상이 아닌 토큰을 사용하니까요).

### 패턴 4: AI 디자인 어시스턴트

MCP가 설정되면 AI를 디자인 어시스턴트로 사용하세요:

```
You: "취소" 액션에 primary 버튼을 쓸지 secondary 버튼을 쓸지?

Claude: [MCP로 컴포넌트 지식 쿼리]

Cancel 액션에는 secondary 또는 ghost variant를 사용하세요:
- Primary 버튼은 메인 CTA용입니다 (화면당 최대 하나)
- Cancel 액션은 낮은 우선순위이며 primary 액션과 경쟁하면 안 됩니다
- 권장: ghost variant (가장 낮은 시각적 비중)

코드:
<Button variant="ghost">Cancel</Button>
```

AI가 디자인 모범 사례를 이해합니다. 여러분이 알 필요 없습니다.

## 1인 개발자를 위한 FAQ

### 저는 디자이너가 아닌데 정말 할 수 있나요?

네. framingui는 비디자이너를 위해 설계됐습니다. 간단한 질문에 답하면 (브랜드를 대표하는 색상이 뭐예요?) framingui가 디자인 수학을 처리합니다:

- 색상 이론 (보색, 음영, 틴트)
- 접근성 (WCAG 대비 요구사항)
- 타이포그래피 스케일 (수학적 진행)
- 간격 리듬 (일관된 시각적 계층)

**Figma 필요 없음. 디자인 학위 필요 없음.**

### 비용이 얼마나 드나요?

framingui는 **유연한 가격 정책**으로 Individual 및 Creator 플랜을 제공합니다. 모든 기능을 탐색할 수 있는 무료 체험이 제공됩니다.

또한 필요한 것:
- AI 도구 구독 (Claude, Cursor 등) — 선택이지만 권장
- 호스팅 (다른 프로젝트와 동일)

### 설정이 실제로 얼마나 걸리나요?

실제 소요 시간:
- **처음**: 2-3시간 (학습 포함)
- **두 번째 프로젝트**: 30-45분 (과정을 알고 있음)
- **그 이후**: 15-20분 (대부분 이전 프로젝트에서 복붙)

### 나중에 색상을 바꾸고 싶으면요?

토큰 하나 바꾸면 전부 업데이트됩니다:

```css
/* 전 */
--color-primary-500: oklch(0.5 0.15 240); /* 파란색 */

/* 후 */
--color-primary-500: oklch(0.5 0.15 140); /* 초록색 */
```

모든 버튼, 링크, 뱃지, 하이라이트가 자동 업데이트됩니다.

**찾아 바꾸기 없음. 수동 업데이트 없음. 토큰 시스템이 처리합니다.**

### 나중에 디자이너를 고용해서 개선할 수 있나요?

물론입니다! 디자이너는 디자인 시스템 작업을 좋아합니다. `tokens.css` 파일을 건네주면 이렇게 할 수 있습니다:

- 색상 팔레트 다듬기
- 간격 스케일 조정
- 컴포넌트 변형 추가

그들의 모든 변경이 토큰을 통해 흐릅니다. 코드 변경이 필요 없습니다.

### OKLCH 색상 공간을 배워야 하나요?

아니요. framingui가 OKLCH를 내부적으로 처리합니다. 다음과 같이 색상을 제공할 수 있습니다:
- Hex: `#3B82F6`
- RGB: `rgb(59, 130, 246)`
- HSL: `hsl(217, 91%, 60%)`

framingui가 자동으로 OKLCH로 변환하고 WCAG 준수 스케일을 생성합니다.

### Tailwind CSS를 사용하고 있다면요?

framingui가 Tailwind와 통합됩니다:

```javascript
// tailwind.config.js (framingui가 생성)
module.exports = {
  theme: {
    extend: {
      colors: {
        primary: {
          50: 'var(--color-primary-50)',
          500: 'var(--color-primary-500)',
          900: 'var(--color-primary-900)',
        },
      },
      spacing: {
        1: 'var(--spacing-1)',
        4: 'var(--spacing-4)',
      },
    },
  },
};
```

이제 Tailwind 유틸리티 사용:

```tsx
<div className="bg-primary-500 p-4 rounded-lg">
  디자인 토큰 사용
</div>
```

### Vue/Svelte/Angular에서 사용할 수 있나요?

디자인 토큰 (CSS 변수)은 모든 프레임워크에서 작동합니다:

```css
/* tokens.css */
:root {
  --color-primary: oklch(0.5 0.15 240);
}
```

```html
<!-- Vue -->
<div :style="{ background: 'var(--color-primary)' }">Content</div>

<!-- Svelte -->
<div style:background="var(--color-primary)">Content</div>

<!-- Angular -->
<div [style.background]="'var(--color-primary)'">Content</div>
```

MCP 서버는 현재 React 컴포넌트를 생성하지만, 토큰은 어디서든 작동합니다.
