## 先に結論

- 画面のラフを高速で作るなら `v0`
- プロダクション運用まで一貫させるなら `framingui`

## 比較ポイント

| 観点 | v0 | framingui |
| --- | --- | --- |
| 初速 | 非常に速い | 速い |
| デザイン一貫性 | 追加管理が必要 | トークン中心で管理しやすい |
| 運用拡張 | プロジェクト次第 | MCP 連携で拡張しやすい |
| 長期保守 | ルール整備が必要 | 規約ベースで進めやすい |

## 使い分けの実例

### v0 が向く場面

- アイデア検証
- UI ラフ作成
- 1画面単位の試作

### framingui が向く場面

- 本番 UI の継続開発
- 複数メンバーでの運用
- AI 生成の品質を固定したい

## まとめ

どちらも有用ですが、解く課題が違います。検証フェーズは v0、本番フェーズは framingui という段階的な使い方も効果的です。
