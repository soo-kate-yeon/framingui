## 一貫性はセンスより仕組み

個人開発で UI が崩れる原因は、判断基準が毎回変わることです。まずは次を固定します。

- 色
- 余白
- タイポ
- コンポーネントの使い方

## 最小ルールセット

### 1. 色は 5 役割だけ

- `primary`
- `surface`
- `text`
- `muted`
- `destructive`

### 2. 余白は 4 段階だけ

`4 / 8 / 16 / 24` のように段階を絞ると、画面の密度が揃います。

### 3. 見出し階層を固定

`h1 > h2 > body > caption` を崩さないだけで読みやすさが安定します。

## 実装フロー

1. トークンを定義
2. コンポーネントを制限
3. AI 生成時に制約を明示
4. 差分レビューで逸脱を戻す

## よくある崩れパターン

- 画面ごとに角丸が違う
- ボタン優先度が逆転する
- 同じ情報でも余白の取り方が違う

これらはルール不足で起きます。スタイルの良し悪しより、再現可能性を先に作るのが重要です。

## まとめ

デザイナー不在でも、設計ルールを先に決めれば UI 品質は上げられます。小さな規約を守り続けることが最も効果的です。
