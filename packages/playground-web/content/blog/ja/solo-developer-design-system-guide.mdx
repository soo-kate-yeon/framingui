## 目的を絞る

最初から完璧な体系を作る必要はありません。まずは「新しい画面を迷わず追加できる状態」を目標にします。

## ステップ 1: コアトークンを決める

- Color
- Spacing
- Radius
- Typography

この 4 つだけで十分始められます。

## ステップ 2: 基本コンポーネントを固定

- Button
- Input
- Card
- Table
- Dialog

利用頻度が高い順に定義し、API をぶらさないことが重要です。

## ステップ 3: 画面テンプレートを作る

- 認証画面
- 一覧画面
- 詳細画面

先に骨格を作ると、実装速度が一気に上がります。

## ステップ 4: AI 連携を追加

MCP でトークンとコンポーネント仕様を AI に渡します。生成品質が安定し、レビューも速くなります。

## まとめ

ソロ開発のデザインシステムは、規模より継続性が重要です。最小構成で始め、画面追加のたびにルールを強化していくのが現実的です。
