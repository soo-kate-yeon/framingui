## ソロ開発者のジレンマ

一人で(または共同創業者1人と)SaaS製品を作っています。必要なこと:

- 高速リリース(投資家/ユーザーは待たない)
- プロフェッショナルに見せる(汎用UIはコンバートしない)
- 一貫性を保つ(雑なUIは信頼を損なう)
- コスト削減($50kデザインエージェンシーなし)
- メンテナンス地獄を避ける(あなたが唯一のエンジニア)

従来のアドバイス: 「Material-UIやChakraのようなコンポーネントライブラリを使う。」

問題: 汎用ライブラリは汎用的に見えます。すべてのMUIアプリが同じに見えます。ランディングページが「テンプレート」と叫びます。

**2026年の答え:** スピードとブランド差別化の両方を提供する**レイヤードスタック**。

## 要点まとめ: スタック

```
Layer 1: Tailwind CSS(ユーティリティフレームワーク)
         ↓
Layer 2: FramingUI(design tokens + システム)
         ↓
Layer 3: shadcn/ui(コピー&ペーストコンポーネント)
         ↓
Layer 4: Custom components(差別化要素)
```

**セットアップ時間:** 2〜3時間
**メンテナンス:** 月1時間
**コスト:** $0(すべてオープンソース)
**結果:** ユニークに見え、高速リリース、一貫性を保つプロフェッショナルUI

## スタック、レイヤーごと

### Layer 1: Tailwind CSS(基盤)

**役割:** HTMLから離れずにスタイリングするユーティリティクラス

**必須の理由:**
- CSS記述より10倍速い
- 「このクラス名は何にしよう?」決定疲労を排除
- 小さなバンドル(使用クラスのみ含む)
- 業界標準(採用しやすい、学習しやすい)

**セットアップ:**
```bash
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
```

**判決:** 交渉不可。すべての現代のインディーハッカーはTailwindを使うべきです。

---

### Layer 2: FramingUI(デザインシステム)

**役割:** Tailwindの上に位置するdesign tokens + 規約

**必要な理由:**
- Tailwindは1000オプションを提供; FramingUIはあなたのオプションに絞ります
- アプリ全体で一貫性を強制
- リブランディングを5分タスクにします(コードでなくトークン変更)
- AIツール(Cursor、Claude)がブランドに合ったコード生成を可能に

**得られるもの:**
```typescript
// コード化されたデザイン決定
const tokens = {
  colors: {
    brand: {
      primary: '#4338CA',    // ランダムな青でなくブランドカラー
      secondary: '#7C3AED',
    }
  },
  spacing: {
    sm: '0.75rem',           // 任意px値でなくスペーシングスケール
    md: '1rem',
    lg: '1.5rem',
  },
  typography: {
    fontFamily: {
      sans: 'Inter, system-ui',  // システムデフォルトでなくフォント
    }
  }
}
```

**セットアップ:**
```bash
npm install @framingui/core
```

**時間投資:** トークン定義に1〜2時間

**判決:** 価値あり。これがコード形式のデザインシステムです。

---

### Layer 3: shadcn/ui(コンポーネントライブラリ)

**役割:** コピー&ペーストReactコンポーネント(NPMパッケージでない)

**インディーハッカーに完璧な理由:**
- コード所有(ベンダーロックインなし)
- 完全カスタマイズ可能(ソースを直接編集)
- デフォルトでアクセシブル(Radix UI primitives基盤)
- Tailwind + FramingUIと完璧に動作

**得られるもの:**
- 約40のプロダクション対応コンポーネント(Button、Dialog、Form、Tableなど)
- TypeScriptサポート
- キーボードナビゲーションとスクリーンリーダーサポート
- デフォルトでdark mode

**セットアップ:**
```bash
npx shadcn@latest init
```

**コンポーネント追加:**
```bash
npx shadcn@latest add button
npx shadcn@latest add dialog
npx shadcn@latest add form
```

コンポーネントはあなたのコードベースの`components/ui/`に追加されます。自由に編集できます。

**判決:** ゲームチェンジャー。バンドルサイズやロックインなしでコンポーネントライブラリの80%を取得。

---

### Layer 4: Custom Components(差別化要素)

**役割:** 製品固有のコンポーネント

**必要な理由:**
- shadcnは汎用UI提供(ボタン、フォーム、ダイアログ)
- 製品にはドメイン固有コンポーネントが必要(分析チャート、ユーザープロファイル、価格表)

**例:**
```tsx
// components/features/PricingCard.tsx
// components/features/AnalyticsDashboard.tsx
// components/features/UserAvatar.tsx
```

FramingUIトークンとshadcnコンポーネントをビルディングブロックとして使用:

```tsx
// components/features/PricingCard.tsx
import { Button } from '@/components/ui/button'
import { tokens } from '@/lib/tokens'

export function PricingCard({ plan, price, features }) {
  return (
    <div className="bg-surface-base border border-neutral-200 rounded-lg p-lg">
      <h3 className="text-2xl font-bold text-text-primary">{plan}</h3>
      <p className="text-4xl font-bold text-brand-primary my-md">${price}/mo</p>
      <ul className="space-y-sm mb-lg">
        {features.map(f => (
          <li key={f} className="flex items-center gap-sm">
            <Check className="w-5 h-5 text-semantic-success" />
            <span className="text-text-secondary">{f}</span>
          </li>
        ))}
      </ul>
      <Button variant="primary" className="w-full">
        Get Started
      </Button>
    </div>
  )
}
```

**判決:** ここがブランドが生きる場所です。汎用コンポーネントが80%まで連れて行く; カスタムコンポーネントが100%にします。

## よくある落とし穴(と避け方)

### 落とし穴1: shadcnコンポーネントを早すぎるカスタマイズ

**間違い:** shadcnインストール後、すぐにButton.tsxを編集開始

**問題:** アップストリーム更新とバグ修正を失う

**解決:** shadcnを2〜4週間そのまま使用。明確で繰り返される必要がある場合のみカスタマイズ。

### 落とし穴2: トークンを事前定義しない

**間違い:** コンポーネント構築開始、進行しながらカラーを決定

**問題:** 一貫性が急速に侵入

**解決:** 1時間かけて定義:
- 2〜3ブランドカラー
- 1完全neutralスケール(50〜900)
- 3セマンティックカラー(success、error、warning)
- 5スペーシング値(xs、sm、md、lg、xl)
- 2フォントファミリー(sans、mono)
