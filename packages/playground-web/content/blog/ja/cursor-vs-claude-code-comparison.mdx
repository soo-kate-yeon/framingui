## AI コーディングツールの現在地

AI 支援開発は大きく進化しました。プロ向けで特に存在感があるのは **Cursor**（AI ネイティブ IDE）と **Claude Code**（Anthropic の CLI エージェント）です。どちらも強力ですが、得意なワークフローが異なります。

## Cursor: AI ネイティブ IDE

Cursor は VS Code を土台に、AI 機能をエディタ体験に直接組み込んだツールです。

### 強み

- **視覚的な作業フロー** — 差分を見ながら採用・破棄を判断できる
- **Tab 補完** — 自然な予測補完
- **複数ファイル編集** — IDE の文脈を使って横断的に提案
- **導入しやすい** — VS Code に近い操作感

### 向いているケース

従来の IDE フローの中で AI 支援を受けたい場合に強みを発揮します。変更を見てから適用したい人に向いています。

## Claude Code: ターミナル中心のエージェント

Claude Code は CLI で動く自律型のコーディングエージェントです。

### 強み

- **エージェント型ワークフロー** — タスクを渡すと計画・実装・検証を自律的に進める
- **深い文脈理解** — 200K トークン級コンテキストで大規模コードベースにも対応
- **Git ネイティブ** — コミット、ブランチ、PR まで流れに含めやすい
- **組み合わせやすい** — 任意のエディタ、ターミナル、CI/CD と連携可能
- **拡張しやすい** — Skills、Hooks、MCP サーバー統合

### 向いているケース

複数ファイルにまたがる複雑な作業で効果が高くなります。認証基盤の改修や横断的な機能実装、難しいデバッグで時間短縮しやすいです。

## 比較表

| 機能 | Cursor | Claude Code |
|---------|--------|-------------|
| インターフェース | GUI（VS Code 派生） | CLI（Terminal） |
| 利用モデル | 複数（GPT-4、Claude など） | Claude（Opus / Sonnet / Haiku） |
| コンテキスト長 | モデル依存 | 最大 200K トークン |
| 複数ファイル編集 | 可能（視覚的） | 可能（エージェント型） |
| Git 連携 | 基本機能 | 深い連携（コミット、PR） |
| 拡張性 | 拡張機能 | Skills、Hooks、MCP |
| オフライン対応 | 一部可能 | 不可 |
| 料金 | 月額 20 ドル（Pro） | 従量課金 |

## どちらを選ぶべきか

### Cursor が向いている場合

- インライン提案と視覚的フィードバックを重視したい
- 小さく限定された変更を素早く進めたい
- 今の開発フローを大きく変えたくない
- 1 ファイル中心で作業することが多い

### Claude Code が向いている場合

- 複雑で段階の多いタスクが多い
- 多数のファイルを同時に扱う必要がある
- タスクを自律実行させたい
- ターミナル操作に慣れている
- Git ワークフローと深く連携したい

## Tekton チームでの使い分け

Tekton ではタスクごとに両方を使い分けています。軽い UI 調整やスタイル修正は Cursor が速く、新機能実装や複数ファイル改修は Claude Code の生産性が高いです。

両者は競合というより補完関係です。

## まとめ

「絶対にこれが正解」という 1 つのツールはありません。作業内容、難易度、操作スタイルへの慣れに合わせて選ぶのが現実的です。両方試し、タスクごとに最適な方を使うのがいちばん効率的です。
