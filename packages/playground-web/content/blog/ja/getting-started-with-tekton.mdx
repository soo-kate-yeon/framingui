## Tekton とは？

Tekton は、OKLCH 色空間を使って視覚的に均一でアクセシブルなカラーパレットを生成する AI デザイントークンジェネレーターです。ゼロからデザインシステムを作るときも、既存システムを整えるときも、数秒で実運用向けのトークンを作れます。

### なぜ OKLCH？

HSL や RGB のような従来の色空間は、人の見え方を十分に反映できません。HSL の「明度 50%」は色相によって明るさの見え方が大きく変わります。OKLCH は**知覚的な均一性**を持つため、同じ明度値なら実際の見え方もそろいやすくなります。

## クイックスタート

Tekton の導入は 3 ステップです。

### Step 1: Playground を開く

[framingui.com](https://framingui.com) にアクセスして、インタラクティブ Playground を開きます。値を調整すると、トークンの変化をリアルタイムで確認できます。

### Step 2: ベースカラーを設定する

ブランドのメインカラーを選ぶと、Tekton が次を自動生成します。

- **Primary パレット** — 最も明るい色から最も暗い色まで 10 段階
- **Semantic カラー** — Success / Warning / Error / Info
- **Neutral スケール** — テキスト、背景、ボーダー用

### Step 3: トークンを書き出す

デザイントークンは複数形式でエクスポートできます。

- **CSS Custom Properties** — そのままプロジェクトに貼って使える
- **Tailwind CSS** — Tailwind 設定へ直接統合できる
- **JSON** — デザインツールや独自連携向け

## 主な機能

### アクセシビリティ優先

すべての配色は WCAG 2.1 のコントラスト比でチェックします。AA / AAA を満たさない組み合わせは自動で検出するため、リリース前に修正できます。

### ダークモード対応

ライトモードとダークモードのパレットを同時に生成できます。どちらのモードでも同じ情報階層とコントラストを保てるように設計しています。

### コンポーネントプレビュー

生成したトークンを実際の UI コンポーネントに適用して確認できます。ボタン、カード、フォームなど、文脈の中で見た目を判断しやすくなります。

## 次にやること

トークンを作ったら、次の流れで進めるのがおすすめです。

1. **コードベースに組み込む** — 生成した CSS や Tailwind 設定をプロジェクトへ反映
2. **チームで共有する** — トークン仕様をデザイナーと開発者へ共有
3. **調整を繰り返す** — Playground で配色を比較し、最適なパレットを選ぶ

Tekton はプロジェクトと一緒に育てられる設計です。まずはシンプルに始めて、必要に合わせて段階的に拡張してください。
