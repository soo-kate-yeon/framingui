## The Theme Setup Tax

If you've ever set up a design system, you know the drill:

1. Edit `tailwind.config.js` to add brand colors
2. Define CSS variables for light/dark modes
3. Update TypeScript types for autocomplete
4. Sync everything manually when design changes
5. Hope nothing breaks

This "theme setup tax" eats hours before you write a single line of UI code. And every time your designer tweaks a color or spacing value, you repeat the cycle.

**What if you could skip all of this?**

## The Traditional Approach: Configuration Hell

Let's look at what typical theme setup looks like.

### Tailwind Config Editing

```js
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        primary: {
          50: '#f0f9ff',
          100: '#e0f2fe',
          // ...10 more shades
          900: '#0c4a6e',
        },
        // Repeat for secondary, accent, etc.
      },
      spacing: {
        // Custom spacing values
      },
      fontFamily: {
        // Custom fonts
      },
    },
  },
};
```

### CSS Variables Setup

```css
:root {
  --color-primary-50: #f0f9ff;
  --color-primary-100: #e0f2fe;
  /* ...50+ more variables */
}

[data-theme='dark'] {
  --color-primary-50: #0c4a6e;
  /* ...reverse the palette */
}
```

### TypeScript Sync

```ts
// types/theme.d.ts
declare module 'tailwindcss/defaultTheme' {
  interface DefaultColors {
    primary: {
      50: string;
      100: string;
      // ...manually sync with config
    };
  }
}
```

**The problem?** Three sources of truth. Manual sync. No single source. One typo breaks everything.

## The FramingUI Way: Zero-Config Theming

FramingUI eliminates configuration files entirely. You define tokens **once**, and everything else is automatic.

### 1. Define Your Theme (Once)

```typescript
// framingui.config.ts
import { defineConfig } from '@framingui/core';

export default defineConfig({
  themes: {
    light: {
      colors: {
        primary: '#0066cc',
        secondary: '#6b7280',
        background: '#ffffff',
        text: '#111827',
      },
      spacing: {
        base: '1rem',
        scale: 1.5,
      },
    },
    dark: {
      colors: {
        primary: '#3b82f6',
        background: '#1f2937',
        text: '#f9fafb',
      },
    },
  },
});
```

That's it. No Tailwind config. No CSS variables file. No TypeScript declarations.

### 2. Everything Generates Automatically

When you run `npx @framingui/cli build`, FramingUI:

- Generates Tailwind config
- Creates CSS variables for both themes
- Builds TypeScript types with autocomplete
- Outputs color scales (50-900 shades)
- Handles dark mode switching

All from one config file.

### 3. MCP Integration: Theme Without Thinking

When you connect FramingUI's MCP server to Claude Code (or any MCP client), AI agents can access your theme tokens directly.

**Example prompt:**

> "Create a pricing card with primary background and secondary text"

**Claude Code output:**

```tsx
<div className="bg-primary-500 text-secondary-700 dark:bg-primary-600 dark:text-secondary-200">
  {/* Pricing content */}
</div>
```

No hallucinated colors. No manual lookup. Claude knows your exact theme.

## How It Works: Single Source of Truth

FramingUI uses a **token-first architecture**:

1. **You define tokens** in `framingui.config.ts` (semantic, human-readable)
2. **FramingUI compiles** tokens into platform-specific formats (CSS, Tailwind, JS)
3. **MCP exposes tokens** to AI agents for generation
4. **Your UI stays consistent** across design, code, and AI output

```
framingui.config.ts
       ↓
  [FramingUI Build]
       ↓
  ├── tailwind.config.js (auto-generated)
  ├── tokens.css (CSS variables)
  ├── tokens.ts (TypeScript types)
  └── MCP schema (AI-readable)
```

Every output is **derived**, not manually written. Change one token, everything updates.

## Practical Example: Adding a New Theme

Let's say your designer hands you a new "midnight" theme.

### Traditional Approach (30+ minutes)

1. Edit `tailwind.config.js` (add new color palette)
2. Update CSS variables file (light + dark variants)
3. Adjust TypeScript types
4. Test manually
5. Fix typos and mismatches
6. Re-deploy

### FramingUI Approach (2 minutes)

```typescript
// framingui.config.ts
export default defineConfig({
  themes: {
    light: { /* existing */ },
    dark: { /* existing */ },
    midnight: {  // ← Just add this
      colors: {
        primary: '#a78bfa',
        background: '#0f0a1e',
        text: '#e0e7ff',
      },
    },
  },
});
```

Run `npx @framingui/cli build`. Done.

- Tailwind config: updated
- CSS variables: generated
- TypeScript types: in sync
- MCP schema: ready for AI

**No manual edits. No risk of mismatch.**

## Why Zero-Config Matters

### For Developers

- **Faster setup** — minutes, not hours
- **Less maintenance** — one file to update
- **Type safety** — autocomplete always matches reality
- **No manual sync** — everything derives from tokens

### For Teams

- **Single source of truth** — designers and developers use the same tokens
- **Easier onboarding** — new devs don't need to learn fragile config patterns
- **Scalable** — adding themes or tokens doesn't exponentially increase complexity

### For AI Workflows

- **Consistent output** — AI agents use real tokens, not guesses
- **No hallucination** — Claude can't invent `bg-brand-purple` when your brand color is `primary`
- **Instant updates** — change tokens, AI immediately knows new values

## Getting Started

Want to eliminate theme setup pain in your project?

### 1. Install FramingUI

```bash
npm install @framingui/core @framingui/mcp-server
```

### 2. Initialize Config

```bash
npx @framingui/cli init
```

This scaffolds `framingui.config.ts` with starter tokens.

### 3. Build Tokens

```bash
npx @framingui/cli build
```

Your Tailwind config, CSS variables, and TypeScript types are now auto-generated.

### 4. (Optional) Connect MCP

If you use Claude Code or other MCP clients:

```json
{
  "mcpServers": {
    "framingui": {
      "command": "npx",
      "args": ["@framingui/mcp-server"]
    }
  }
}
```

Now AI agents can access your theme tokens directly.
