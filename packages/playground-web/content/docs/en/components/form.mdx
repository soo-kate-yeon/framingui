# Form

`Form` is a powerful form-building system in FramingUI that integrates React Hook Form with Zod validation and accessible field management.

## Import

```tsx
import {
  Form,
  FormControl,
  FormDescription,
  FormField,
  FormItem,
  FormLabel,
  FormMessage,
} from '@framingui/ui';
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import * as z from 'zod';
```

## Basic Usage

```tsx
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import * as z from 'zod';
import {
  Form,
  FormControl,
  FormField,
  FormItem,
  FormLabel,
  FormMessage,
} from '@framingui/ui';
import { Input, Button } from '@framingui/ui';

const formSchema = z.object({
  username: z.string().min(2, {
    message: 'Username must be at least 2 characters.',
  }),
});

export function BasicForm() {
  const form = useForm<z.infer<typeof formSchema>>({
    resolver: zodResolver(formSchema),
    defaultValues: {
      username: '',
    },
  });

  function onSubmit(values: z.infer<typeof formSchema>) {
    console.log(values);
  }

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(onSubmit)} className="space-y-4">
        <FormField
          control={form.control}
          name="username"
          render={({ field }) => (
            <FormItem>
              <FormLabel>Username</FormLabel>
              <FormControl>
                <Input placeholder="johndoe" {...field} />
              </FormControl>
              <FormMessage />
            </FormItem>
          )}
        />
        <Button type="submit">Submit</Button>
      </form>
    </Form>
  );
}
```

## Component API

### Form

`Form` is an alias for `FormProvider` from `react-hook-form`. It accepts all standard `FormProvider` props.

| Prop | Type | Description |
| --- | --- | --- |
| `...props` | `FormProviderProps` | All React Hook Form provider props |

### FormField

| Prop | Type | Description |
| --- | --- | --- |
| `control` | `Control<TFieldValues>` | Form control from `useForm` |
| `name` | `FieldPath<TFieldValues>` | Field name in the form |
| `render` | `({ field }) => React.ReactElement` | Render function |

### FormItem

| Prop | Type | Description |
| --- | --- | --- |
| `className` | `string` | Extra utility classes |
| `...props` | `HTMLAttributes<HTMLDivElement>` | Standard div attributes |

### FormLabel

| Prop | Type | Description |
| --- | --- | --- |
| `className` | `string` | Extra utility classes |
| `...props` | `LabelProps` | Radix Label props |

### FormControl

`FormControl` is a `Slot` component that passes proper accessibility attributes to its child.

### FormDescription

| Prop | Type | Description |
| --- | --- | --- |
| `className` | `string` | Extra utility classes |
| `...props` | `HTMLAttributes<HTMLParagraphElement>` | Standard paragraph attributes |

### FormMessage

| Prop | Type | Description |
| --- | --- | --- |
| `className` | `string` | Extra utility classes |
| `children` | `React.ReactNode` | Custom error message (overrides validation) |

## Design Tokens

Form components use these Tailwind utility classes:

- Spacing: `space-y-[var(--tekton-spacing-2)]`
- Error text: `text-[var(--tekton-bg-destructive)]`
- Description text: `text-[var(--tekton-bg-muted-foreground)]`
- Font size: `text-sm`

## Best Practices

- Always use Zod schemas for type-safe validation.
- Wrap form inputs with `FormControl` for proper accessibility.
- Use `FormDescription` for helpful hints, not critical information.
- Add `FormMessage` after each field for inline error display.
- Set `type="submit"` explicitly on submit buttons.
- Use semantic input types (`email`, `password`, `url`, `tel`).
- Provide clear, actionable error messages in validation schemas.
- Consider UX: show errors after blur or submit, not on every keystroke.
