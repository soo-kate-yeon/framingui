# MCP セットアップ

FramingUI MCP を使うと、AI ツールがデザインシステムのトークン、テーマ、コンポーネント API を理解できるようになります。

## 1) クイックセットアップ

```bash
npx @framingui/mcp-server init
```

このコマンドで依存関係の設定、Tailwind 設定更新、`@framingui/ui/styles` 追加、`.mcp.json` 作成までまとめて行います。

## 2) Claude / Cursor / Windsurf を接続

3つのツールは、プロジェクトルートの `.mcp.json` を読み取れます。

```json
{
  "mcpServers": {
    "tekton": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@framingui/mcp-server"]
    }
  }
}
```

### Claude Code

1. プロジェクトルートで `npx @framingui/mcp-server init` を実行
2. Claude Code を再起動
3. 「利用できる FramingUI テーマを一覧表示して」と質問

### Cursor

1. ワークスペースのルートに `.mcp.json` があることを確認
2. Cursor を再起動（またはウィンドウ再読み込み）
3. トークンベース UI タスクを依頼し、MCP ツールが出るか確認

### Windsurf

1. `.mcp.json` をプロジェクトルートに配置
2. Windsurf を再起動
3. テーマ一覧やコンポーネント一覧を依頼して確認

## 3) `framingui-mcp` コマンド

| コマンド | できること |
| --- | --- |
| `npx @framingui/mcp-server init` | MCP + デザインシステムの初期設定 |
| `framingui-mcp` | MCP サーバーを stdio で起動 |
| `framingui-mcp login` | ブラウザでログイン認証 |
| `framingui-mcp logout` | ローカル認証セッションを削除 |
| `framingui-mcp status` | 現在の認証状態を確認 |

## 4) トラブルシューティング

### MCP サーバーが検出されない

- `.mcp.json` がプロジェクトルートにあるか確認
- JSON 構文が正しいか確認
- 設定変更後に AI ツールを再起動

### `framingui-mcp` が見つからない

まず `npx` で実行するか、依存関係をインストールしてください。

```bash
npx @framingui/mcp-server init
npx @framingui/mcp-server
```

### 認証に失敗する

- `framingui-mcp login` を再実行
- `framingui-mcp status` で状態確認
- ブラウザログインが最後まで完了しているか確認

### セットアップ後にスタイルが崩れる

- グローバル CSS に `@framingui/ui/styles` があるか確認
- Tailwind `content` に `@framingui/ui` のパスが含まれているか確認
- 設定変更後に開発サーバーを再起動
