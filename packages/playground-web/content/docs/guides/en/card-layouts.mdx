# Card Component Layouts

Cards are versatile containers for grouping related content. FramingUI's Card component provides a flexible foundation for building dashboards, product grids, settings panels, and more.

## Basic Card Structure

A card consists of three main parts:

```tsx
import { Card, CardHeader, CardTitle, CardDescription, CardContent, CardFooter } from '@framingui/ui';

<Card>
  <CardHeader>
    <CardTitle>Card Title</CardTitle>
    <CardDescription>Optional description text</CardDescription>
  </CardHeader>
  <CardContent>
    {/* Main content goes here */}
  </CardContent>
  <CardFooter>
    {/* Footer actions or metadata */}
  </CardFooter>
</Card>
```

**Each part is optional** — use only what you need.

## Common Card Patterns

### Simple Content Card

Basic text container:

```tsx
<Card>
  <CardHeader>
    <CardTitle>Getting Started</CardTitle>
    <CardDescription>Install FramingUI in your project</CardDescription>
  </CardHeader>
  <CardContent>
    <p className="text-muted-foreground">
      Run <code className="bg-muted px-1 py-0.5 rounded">npm install @framingui/ui</code> to add FramingUI to your React project.
    </p>
  </CardContent>
</Card>
```

```tsx
  <CardFooter>
```

### Card with Multiple Actions

Footer with primary + secondary actions:

```tsx
<Card>

    <CardTitle>Unsaved Changes</CardTitle>
```

### Card with Header Action

Inline action in header (settings, edit, etc.):

## Grid Layouts

### Equal Column Grid

Responsive card grid:

```tsx
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6">
  <Card>
    <CardHeader>
      <CardTitle>Card 1</CardTitle>
    </CardHeader>
    <CardContent>Content</CardContent>
  </Card>
  <Card>
    <CardHeader>
      <CardTitle>Card 2</CardTitle>
    </CardHeader>
    <CardContent>Content</CardContent>
  </Card>
  <Card>
    <CardHeader>
      <CardTitle>Card 3</CardTitle>
    </CardHeader>
    <CardContent>Content</CardContent>
  </Card>
</div>
```

**Breakpoints:**
- Mobile: 1 column
- Tablet: 2 columns
- Desktop: 3 columns

## Interactive Cards

### Clickable Card

Entire card as clickable element:

```tsx
<Card className="cursor-pointer hover:shadow-lg transition-shadow">
  <a href="/project/123" className="block">
    <CardHeader>
      <CardTitle>Project Name</CardTitle>
      <CardDescription>Click to view details</CardDescription>
    </CardHeader>
    <CardContent>
      <p className="text-sm text-muted-foreground">
        Last updated 2 hours ago
      </p>
    </CardContent>
  </a>
</Card>
```

### Selectable Card

Card with checkbox or radio:

```tsx
import { Checkbox } from '@framingui/ui';

<Card className="cursor-pointer hover:border-primary">
  <CardHeader className="flex flex-row items-start gap-4">
    <Checkbox id="plan-pro" />
    <label htmlFor="plan-pro" className="flex-1 cursor-pointer">
      <CardTitle>Pro Plan</CardTitle>
      <CardDescription>For growing teams</CardDescription>
    </label>
  </CardHeader>
  <CardContent className="pl-10">
    <div className="text-2xl font-bold">$49/mo</div>
  </CardContent>
</Card>
```

### Expandable Card

Show/hide additional content:

```tsx
import { useState } from 'react';
import { IconChevronDown } from '@tabler/icons-react';
import { Button } from '@framingui/ui';

function ExpandableCard() {
  const [expanded, setExpanded] = useState(false);

  return (
    <Card>
      <CardHeader>
        <div className="flex items-center justify-between">
          <CardTitle>Advanced Settings</CardTitle>
          <Button
            size="icon"
            variant="ghost"
            onClick={() => setExpanded(!expanded)}
          >
            <IconChevronDown
              className={`w-4 h-4 transition-transform ${
                expanded ? 'rotate-180' : ''
              }`}
            />
          </Button>
        </div>
      </CardHeader>
      {expanded && (
        <CardContent>
          {/* Additional content */}
          <p className="text-sm text-muted-foreground">
            Configure advanced options here.
          </p>
        </CardContent>
      )}
    </Card>
  );
}
```

## Accessibility

1. **Use semantic HTML** — Cards are `<div>` containers, not landmarks
2. **Make clickable cards keyboard-accessible** — Wrap in `<a>` or `<button>`
3. **Label interactive elements** — Checkboxes, buttons need labels
4. **Maintain contrast** — Text on card backgrounds meets WCAG AA
5. **Screen reader support** — Use proper heading hierarchy

## Best Practices

✅ **Keep card content focused** — One topic per card

✅ **Use consistent spacing** — CardHeader, CardContent, CardFooter have default padding

✅ **Group related cards** — Use grids for visual hierarchy

✅ **Limit nesting** — Avoid cards inside cards (use borders/sections instead)

✅ **Maintain aspect ratios** — Use `aspect-square` or `aspect-video` for images

## Common Mistakes

❌ **Too much content** — Cards should be scannable, not essays

```tsx
{/* Wrong - too much text */}
<Card>
  <CardContent>
    <p>Lorem ipsum dolor sit amet...</p>
    {/* 10 more paragraphs */}
  </CardContent>
</Card>
```

✅ **Use summary + expand pattern**

```tsx
{/* Correct */}
<Card>
  <CardHeader>
    <CardTitle>Article Title</CardTitle>
    <CardDescription>Brief summary in 1-2 lines</CardDescription>
  </CardHeader>
  <CardFooter>
    <Button variant="link">Read More</Button>
  </CardFooter>
</Card>
```
