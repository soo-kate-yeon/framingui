# Design Token Reference

FramingUI design tokens provide a semantic layer between raw design values and implementation. This reference covers all token categories, naming conventions, and usage patterns.

## Token Structure

All FramingUI tokens follow this structure:

```json
{
  "tokenName": {
    "value": "<actual value>",
    "type": "<token type>",
    "description": "<usage description>"
  }
}
```

## Color Tokens

### Text Colors

Used for all text content.

| Token | CSS Variable | Value | Usage |
|-------|--------------|-------|-------|
| `color.text.primary` | `--color-text-primary` | `#1a1a1a` | Primary text, headings |
| `color.text.secondary` | `--color-text-secondary` | `#6b7280` | Secondary text, captions |
| `color.text.tertiary` | `--color-text-tertiary` | `#9ca3af` | Disabled text, placeholders |
| `color.text.inverse` | `--color-text-inverse` | `#ffffff` | Text on dark backgrounds |

**Example:**
```tsx
<p className="text-[var(--color-text-primary)]">Primary text</p>
<span className="text-[var(--color-text-secondary)]">Secondary</span>
```

### Surface Colors

Used for backgrounds and containers.

| Token | CSS Variable | Value | Usage |
|-------|--------------|-------|-------|
| `color.surface.base` | `--color-surface-base` | `#f9fafb` | Page background |
| `color.surface.elevated` | `--color-surface-elevated` | `#ffffff` | Cards, modals, elevated content |
| `color.surface.overlay` | `--color-surface-overlay` | `rgba(0,0,0,0.5)` | Modal overlays, backdrops |

**Example:**
```tsx
<div className="bg-[var(--color-surface-base)]">
  <div className="bg-[var(--color-surface-elevated)] p-4">
    Card content
  </div>
</div>
```

### Interactive Colors

Used for buttons, links, and interactive elements.

| Token | CSS Variable | Value | Usage |
|-------|--------------|-------|-------|
| `color.interactive.primary` | `--color-interactive-primary` | `#2563eb` | Primary buttons, links |
| `color.interactive.hover` | `--color-interactive-hover` | `#1d4ed8` | Hover state |
| `color.interactive.active` | `--color-interactive-active` | `#1e40af` | Active/pressed state |
| `color.interactive.disabled` | `--color-interactive-disabled` | `#d1d5db` | Disabled state |

**Example:**
```tsx
<button className="bg-[var(--color-interactive-primary)]
                   hover:bg-[var(--color-interactive-hover)]
                   active:bg-[var(--color-interactive-active)]
                   disabled:bg-[var(--color-interactive-disabled)]">
  Submit
</button>
```

### Border Colors

Used for borders, dividers, and outlines.

| Token | CSS Variable | Value | Usage |
|-------|--------------|-------|-------|
| `color.border.default` | `--color-border-default` | `#e5e7eb` | Default borders |
| `color.border.subtle` | `--color-border-subtle` | `#f3f4f6` | Subtle dividers |
| `color.border.strong` | `--color-border-strong` | `#9ca3af` | Emphasized borders |
| `color.border.focus` | `--color-border-focus` | `#2563eb` | Focus rings |

**Example:**

## Spacing Tokens

### Component Spacing

Used for internal component spacing (padding).

| Token | CSS Variable | Value | Usage |
|-------|--------------|-------|-------|
| `spacing.component.xs` | `--spacing-component-xs` | `0.5rem` (8px) | Tight padding |
| `spacing.component.sm` | `--spacing-component-sm` | `0.75rem` (12px) | Small padding |
| `spacing.component.md` | `--spacing-component-md` | `1rem` (16px) | Default padding |
| `spacing.component.lg` | `--spacing-component-lg` | `1.5rem` (24px) | Large padding |
| `spacing.component.xl` | `--spacing-component-xl` | `2rem` (32px) | Extra large padding |

**Example:**
```tsx
<div className="p-[var(--spacing-component-md)]">
  Default card padding
</div>
```

### Layout Spacing

Used for spacing between components (margins, gaps).

| Token | CSS Variable | Value | Usage |
|-------|--------------|-------|-------|
| `spacing.layout.xs` | `--spacing-layout-xs` | `0.5rem` (8px) | Tight gaps |
| `spacing.layout.sm` | `--spacing-layout-sm` | `1rem` (16px) | Small gaps |
| `spacing.layout.md` | `--spacing-layout-md` | `1.5rem` (24px) | Default gaps |
| `spacing.layout.lg` | `--spacing-layout-lg` | `2rem` (32px) | Large gaps |
| `spacing.layout.xl` | `--spacing-layout-xl` | `3rem` (48px) | Section spacing |

**Example:**
```tsx
<div className="space-y-[var(--spacing-layout-md)]">
  <div>Section 1</div>
  <div>Section 2</div>
</div>
```

## Dark Mode Support

All color tokens support dark mode via CSS variables:

```css
:root {
  --color-text-primary: #1a1a1a;
  --color-surface-base: #f9fafb;
}

@media (prefers-color-scheme: dark) {
  :root {
    --color-text-primary: #f9fafb;
    --color-surface-base: #1a1a1a;
  }
}
```

## Naming Conventions

FramingUI follows this naming structure:

```
{category}.{role}.{variant}
```

## Using Tokens in AI Prompts

When asking AI to generate components, reference token categories:

> "Create a button using `color.interactive.primary` for background and `spacing.component.sm` for padding"

AI tools with MCP access can automatically apply tokens without explicit prompts.

## Custom Token Extensions

To add project-specific tokens:

```json
{
  "custom": {
    "brand": {
      "accent": {
        "value": "#ff6b6b",
        "type": "color",
        "description": "Brand accent color"
      }
    }
  }
}
```

Reference as:
```tsx
<div className="text-[var(--custom-brand-accent)]">Accent text</div>
```
