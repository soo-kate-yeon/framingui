# MCP 서버 구성 가이드

이 가이드는 Claude Code, Cursor 및 기타 AI 코딩 도구를 위한 지속적인 컨텍스트로 FramingUI 디자인 토큰을 제공하는 MCP (Model Context Protocol) 서버 설정을 안내합니다.

## 사전 요구사항

- Node.js 18 이상
- 프로젝트에 FramingUI 설치
- 디자인 토큰 파일 (`design-tokens.json`)

## 구축할 내용

다음을 노출하는 MCP 서버:
- 쿼리 가능한 리소스로서의 디자인 토큰
- 일관된 생성을 위한 컴포넌트 패턴
- 토큰이 변경될 때 실시간 업데이트

## 1단계: MCP SDK 설치

```bash
npm install @modelcontextprotocol/server-sdk
# 또는
pnpm add @modelcontextprotocol/server-sdk
```

## 2단계: 서버 파일 생성

프로젝트 루트에 `mcp-server/index.ts` 생성:

```typescript
import { McpServer } from '@modelcontextprotocol/server-sdk';
import { readFileSync } from 'fs';
import { resolve } from 'path';

const server = new McpServer({
  name: 'framingui-tokens',
  version: '1.0.0',
  description: 'FramingUI design tokens and patterns'
});

// 디자인 토큰 로드
const tokensPath = resolve(__dirname, '../design-tokens.json');
const tokens = JSON.parse(readFileSync(tokensPath, 'utf-8'));

// 디자인 토큰 리소스 등록
server.resource({
  uri: 'design://tokens',
  name: 'Design Tokens',
  description: 'Semantic design tokens (colors, spacing, typography)',
  mimeType: 'application/json',
  async get() {
    return JSON.stringify(tokens, null, 2);
  }
});

server.start();
console.log('MCP server started');
```

## 3단계: TypeScript 컴파일

```bash
npx tsc mcp-server/index.ts --outDir mcp-server --module commonjs
```

## 4단계: Claude Code 구성

`.claude/mcp.json` 생성:

```json
{
  "mcpServers": {
    "framingui-tokens": {
      "command": "node",
      "args": ["./mcp-server/index.js"],
      "env": {}
    }
  }
}
```

## 5단계: 지침 업데이트

`.claude/instructions.md` 편집:

```markdown
# FramingUI 디자인 시스템

이 프로젝트는 MCP를 통해 제공되는 디자인 토큰과 함께 FramingUI를 사용합니다.

## 토큰 액세스

디자인 토큰은 `design://tokens`에서 사용할 수 있습니다. 항상 다음에 토큰을 사용하세요:
- 색상: `var(--color-*)`
- 간격: `var(--spacing-*)`
- 타이포그래피: `var(--typography-*)`

## 규칙

1. 절대로 하드코딩된 값을 사용하지 마세요 (`bg-blue-600`, `p-4`)
2. 항상 CSS 변수를 통해 토큰을 참조하세요
3. 새 컴포넌트를 생성할 때 `design://tokens`를 가져오세요
```

## 6단계: 서버 테스트

Claude Code를 재시작하고 물어보세요:

> "주요 버튼 컴포넌트 만들어줘"

Claude가 `design://tokens`를 가져와서 다음을 생성해야 합니다:

```tsx
export function Button({ children }: ButtonProps) {
  return (
    <button className="px-[var(--spacing-component-md)]
                       py-[var(--spacing-component-sm)]
                       bg-[var(--color-interactive-primary)]
                       hover:bg-[var(--color-interactive-hover)]
                       rounded-[var(--border-radius-md)]">
      {children}
    </button>
  );
}
```

## 고급 구성

### 컴포넌트 패턴 추가

재사용 가능한 패턴을 제공하도록 서버 확장:

```typescript
server.resource({
  uri: 'design://patterns/button',
  name: 'Button Pattern',
  mimeType: 'application/json',
  async get() {
    return JSON.stringify({
      base: 'inline-flex items-center justify-center',
      padding: 'var(--spacing-component-sm) var(--spacing-component-md)',
      background: 'var(--color-interactive-primary)',
      hoverBackground: 'var(--color-interactive-hover)',
      borderRadius: 'var(--border-radius-md)'
    }, null, 2);
  }
});
```

### 실시간 토큰 업데이트

자동 재로드를 위한 파일 감시 추가:

```typescript
import { watch } from 'fs';

let tokens = loadTokens();

watch(tokensPath, (event) => {
  if (event === 'change') {
    tokens = loadTokens();
    console.log('Tokens reloaded');
  }
});

function loadTokens() {
  return JSON.parse(readFileSync(tokensPath, 'utf-8'));
}
```

### 멀티 브랜드 지원

다른 브랜드를 위한 다른 토큰 세트 제공:

```typescript
server.resource({
  uri: 'design://tokens/brand-a',
  name: 'Brand A Tokens',
  mimeType: 'application/json',
  async get() {
    return JSON.stringify(loadBrandTokens('brand-a'));
  }
});

server.resource({
  uri: 'design://tokens/brand-b',
  name: 'Brand B Tokens',
  mimeType: 'application/json',
  async get() {
    return JSON.stringify(loadBrandTokens('brand-b'));
  }
});
```

## 문제 해결

### 서버가 시작되지 않음

**오류:** `Cannot find module '@modelcontextprotocol/server-sdk'`

**해결책:**
```bash
npm install @modelcontextprotocol/server-sdk
```

### Claude가 토큰을 사용하지 않음

**체크리스트:**
1. `.claude/mcp.json`이 `index.js`에 대한 올바른 경로를 가지고 있는지 확인
2. `.claude/instructions.md`가 `design://tokens`를 참조하는지 확인
3. Claude Code를 재시작하여 MCP 구성 재로드
4. 서버를 수동으로 실행하여 오류 확인: `node mcp-server/index.js`

### 토큰이 업데이트되지 않음

**해결책:** `design-tokens.json` 변경 후 MCP 서버를 재시작하거나, 파일 감시를 구현하세요 (고급 구성 참조).

## 보안 고려사항

### 환경 변수

토큰이 비밀 정보를 참조하는 경우 (API 엔드포인트, 키), 환경 변수를 사용하세요:

```typescript
server.resource({
  uri: 'design://config',
  name: 'Config',
  mimeType: 'application/json',
  async get() {
    return JSON.stringify({
      apiUrl: process.env.API_URL,
      // 비밀 정보는 노출하지 마세요
    });
  }
});
```

### 액세스 제어

MCP 서버는 로컬에서 실행되며 기본적으로 네트워크 포트를 노출하지 않습니다. 팀 설정의 경우 다음을 고려하세요:
- 공유 개발 머신에서 MCP 서버 실행
- 원격 액세스를 위한 SSH 터널링 사용
- 네트워크를 통해 노출하는 경우 인증 구현

## 다른 도구와의 통합

### Cursor

Cursor는 동일한 구성 형식을 통해 MCP를 지원합니다:

```json
{
  "mcpServers": {
    "framingui-tokens": {
      "command": "node",
      "args": ["./mcp-server/index.js"]
    }
  }
}
```

### Windsurf

Windsurf는 `.windsurf/mcp.json`에 MCP 구성을 배치해야 합니다:

```bash
mkdir -p .windsurf
cp .claude/mcp.json .windsurf/mcp.json
```

## 모범 사례

1. **버전 관리에 토큰 보관** — `design-tokens.json`이 커밋되었는지 확인
2. **토큰 사용 문서화** — 토큰 값에 설명 추가
3. **변경 후 테스트** — 토큰 업데이트 후 AI가 올바른 코드를 생성하는지 확인
4. **의미론적 이름 사용** — 토큰 이름에 구현 세부사항 피하기

## 요약

MCP 서버는 AI 코딩 도구를 위한 지속적인 디자인 컨텍스트를 제공합니다. FramingUI 토큰을 MCP 리소스로 노출함으로써, AI가 생성하는 모든 컴포넌트가 모든 프롬프트에 제약을 반복하지 않고도 디자인 시스템과 일치하도록 보장합니다.
