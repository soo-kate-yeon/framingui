# MCP 설정

FramingUI MCP를 연결하면 AI 도구가 디자인 시스템의 토큰, 테마, 컴포넌트 API를 직접 이해하고 코드를 생성할 수 있다.

## 1) 빠른 설정

```bash
npx @framingui/mcp-server init
```

이 명령으로 의존성 설치, Tailwind 설정 보강, `@framingui/ui/styles` 추가, `.mcp.json` 생성까지 한 번에 처리된다.

## 2) Claude / Cursor / Windsurf 연동

세 도구 모두 프로젝트 루트의 `.mcp.json`을 기준으로 MCP 서버를 로드할 수 있다.

```json
{
  "mcpServers": {
    "tekton": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@framingui/mcp-server"]
    }
  }
}
```

### Claude Code

1. 프로젝트 루트에서 `npx @framingui/mcp-server init` 실행
2. Claude Code 재시작
3. "FramingUI 테마 목록 보여줘"로 연결 확인

### Cursor

1. 워크스페이스 루트에 `.mcp.json`이 있는지 확인
2. Cursor 재시작(또는 reload)
3. 토큰 기반 UI 생성 요청으로 MCP 도구 노출 확인

### Windsurf

1. 루트 `.mcp.json` 유지
2. Windsurf 재시작
3. 테마/컴포넌트 목록 요청으로 연결 확인

## 3) `framingui-mcp` 명령어

| 명령어 | 설명 |
| --- | --- |
| `npx @framingui/mcp-server init` | MCP + 디자인 시스템 초기 설정 |
| `framingui-mcp` | stdio 방식 MCP 서버 실행 |
| `framingui-mcp login` | 브라우저 인증 로그인 |
| `framingui-mcp logout` | 로컬 인증 정보 제거 |
| `framingui-mcp status` | 현재 인증 상태 확인 |

## 4) 트러블슈팅

### MCP 서버가 감지되지 않을 때

- `.mcp.json`이 프로젝트 루트에 있는지 확인
- JSON 문법 오류가 없는지 확인
- 설정 변경 후 AI 도구를 반드시 재시작

### `framingui-mcp`를 찾을 수 없을 때

`npx`로 먼저 실행해 확인한다.

```bash
npx @framingui/mcp-server init
npx @framingui/mcp-server
```

### 인증이 실패할 때

- `framingui-mcp login` 재실행
- `framingui-mcp status`로 상태 확인
- 브라우저 인증 완료 탭이 정상 종료됐는지 확인

### 스타일이 깨져 보일 때

- 전역 CSS에 `@framingui/ui/styles`가 들어갔는지 확인
- Tailwind `content`에 `@framingui/ui` 경로가 포함됐는지 확인
- 설정 수정 후 dev 서버 재시작
