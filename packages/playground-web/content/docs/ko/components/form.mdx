# Form

`Form`은 FramingUI에서 React Hook Form과 Zod 검증, 접근성 관리를 통합한 강력한 폼 빌딩 시스템이다.

## Import

```tsx
import {
  Form,
  FormControl,
  FormDescription,
  FormField,
  FormItem,
  FormLabel,
  FormMessage,
} from '@framingui/ui';
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import * as z from 'zod';
```

## 기본 사용법

```tsx
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import * as z from 'zod';
import {
  Form,
  FormControl,
  FormField,
  FormItem,
  FormLabel,
  FormMessage,
} from '@framingui/ui';
import { Input, Button } from '@framingui/ui';

const formSchema = z.object({
  username: z.string().min(2, {
    message: '사용자명은 최소 2글자 이상이어야 합니다.',
  }),
});

export function BasicForm() {
  const form = useForm<z.infer<typeof formSchema>>({
    resolver: zodResolver(formSchema),
    defaultValues: {
      username: '',
    },
  });

  function onSubmit(values: z.infer<typeof formSchema>) {
    console.log(values);
  }

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(onSubmit)} className="space-y-4">
        <FormField
          control={form.control}
          name="username"
          render={({ field }) => (
            <FormItem>
              <FormLabel>사용자명</FormLabel>
              <FormControl>
                <Input placeholder="홍길동" {...field} />
              </FormControl>
              <FormMessage />
            </FormItem>
          )}
        />
        <Button type="submit">제출</Button>
      </form>
    </Form>
  );
}
```

## 컴포넌트 API

### Form

`Form`은 `react-hook-form`의 `FormProvider`에 대한 별칭이다. 모든 표준 `FormProvider` props를 받는다.

| Prop | 타입 | 설명 |
| --- | --- | --- |
| `...props` | `FormProviderProps` | React Hook Form provider의 모든 props |

## 디자인 토큰

Form 컴포넌트는 다음 Tailwind 유틸리티 클래스를 사용한다:

- 간격: `space-y-[var(--tekton-spacing-2)]`
- 오류 텍스트: `text-[var(--tekton-bg-destructive)]`
- 설명 텍스트: `text-[var(--tekton-bg-muted-foreground)]`
- 폰트 크기: `text-sm`

## 사용 팁

- 타입 안전한 검증을 위해 항상 Zod 스키마를 사용한다.
- 적절한 접근성을 위해 폼 입력을 `FormControl`로 감싼다.
- 중요한 정보가 아닌 도움말에는 `FormDescription`을 사용한다.
- 인라인 오류 표시를 위해 각 필드 다음에 `FormMessage`를 추가한다.
- 제출 버튼에는 `type="submit"`을 명시적으로 설정한다.
- 의미론적 입력 타입(`email`, `password`, `url`, `tel`)을 사용한다.
- 검증 스키마에 명확하고 실행 가능한 오류 메시지를 제공한다.
- UX 고려: 키 입력마다가 아닌 blur나 submit 후 오류를 표시한다.
