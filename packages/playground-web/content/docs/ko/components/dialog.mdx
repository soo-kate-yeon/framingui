# Dialog

`Dialog`는 Radix UI 기반의 모달 오버레이 컴포넌트로, FramingUI 디자인 시스템에서 접근성 높은 다이얼로그, 모달, 알림을 만들 때 사용한다.

## Import

```tsx
import {
  Dialog,
  DialogContent,
  DialogDescription,
  DialogHeader,
  DialogFooter,
  DialogTitle,
  DialogTrigger,
} from '@framingui/ui';
```

## Props

### Dialog (Root)

| Prop | 타입 | 기본값 | 설명 |
| --- | --- | --- | --- |
| `open` | `boolean` | `undefined` | 제어된 열림 상태 |
| `defaultOpen` | `boolean` | `false` | 비제어 기본 열림 상태 |
| `onOpenChange` | `(open: boolean) => void` | `undefined` | 열림 상태 변경 시 이벤트 핸들러 |
| `modal` | `boolean` | `true` | 다이얼로그가 페이지 나머지 부분과의 상호작용을 차단할지 여부 |

### DialogContent

| Prop | 타입 | 기본값 | 설명 |
| --- | --- | --- | --- |
| `className` | `string` | `undefined` | 추가 유틸리티 클래스 |
| `children` | `ReactNode` | - | 다이얼로그 콘텐츠 |
| `onEscapeKeyDown` | `(event: KeyboardEvent) => void` | `undefined` | Escape 키 핸들러 |
| `onPointerDownOutside` | `(event: PointerDownOutsideEvent) => void` | `undefined` | 외부 클릭 핸들러 |
| `...props` | `React.HTMLAttributes<HTMLDivElement>` | - | 기본 div 속성 |

### DialogHeader, DialogFooter

| Prop | 타입 | 기본값 | 설명 |
| --- | --- | --- | --- |
| `className` | `string` | `undefined` | 추가 유틸리티 클래스 |
| `...props` | `React.HTMLAttributes<HTMLDivElement>` | - | 기본 div 속성 |

### DialogTitle

| Prop | 타입 | 기본값 | 설명 |
| --- | --- | --- | --- |
| `className` | `string` | `undefined` | 추가 유틸리티 클래스 |
| `...props` | Radix DialogTitle props | - | Radix primitive props |

### DialogDescription

| Prop | 타입 | 기본값 | 설명 |
| --- | --- | --- | --- |
| `className` | `string` | `undefined` | 추가 유틸리티 클래스 |
| `...props` | Radix DialogDescription props | - | Radix primitive props |

## 기본 사용법

```tsx
import {
  Dialog,
  DialogContent,
  DialogDescription,
  DialogHeader,
  DialogTitle,
  DialogTrigger,
} from '@framingui/ui';
import { Button } from '@framingui/ui';

export function BasicDialog() {
  return (
    <Dialog>
      <DialogTrigger asChild>
        <Button>다이얼로그 열기</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>환영합니다</DialogTitle>
          <DialogDescription>
            기본 다이얼로그 예제입니다.
          </DialogDescription>
        </DialogHeader>
      </DialogContent>
    </Dialog>
  );
}
```

## 실전 시나리오

### 1. 확인 다이얼로그

```tsx
import {
  Dialog,
  DialogContent,
  DialogDescription,
  DialogFooter,
  DialogHeader,
  DialogTitle,
  DialogTrigger,
} from '@framingui/ui';
import { Button } from '@framingui/ui';

export function DeleteConfirmation() {
  return (
    <Dialog>
      <DialogTrigger asChild>
        <Button variant="destructive">계정 삭제</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>정말 삭제하시겠습니까?</DialogTitle>
          <DialogDescription>
            이 작업은 되돌릴 수 없습니다. 계정이 영구적으로 삭제됩니다.
          </DialogDescription>
        </DialogHeader>
        <DialogFooter>
          <Button variant="outline">취소</Button>
          <Button variant="destructive">삭제</Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  );
}
```

### 2. 로그인 폼 다이얼로그

```tsx
import { useState } from 'react';
import {
  Dialog,
  DialogContent,
  DialogDescription,
  DialogFooter,
  DialogHeader,
  DialogTitle,
  DialogTrigger,
} from '@framingui/ui';
import { Button, Input, Label } from '@framingui/ui';

export function LoginDialog() {
  const [open, setOpen] = useState(false);

  return (
    <Dialog open={open} onOpenChange={setOpen}>
      <DialogTrigger asChild>
        <Button>로그인</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>계정에 로그인</DialogTitle>
          <DialogDescription>
            계속하려면 인증 정보를 입력하세요.
          </DialogDescription>
        </DialogHeader>
        <div className="grid gap-4 py-4">
          <div className="grid gap-2">
            <Label htmlFor="email">이메일</Label>
            <Input id="email" type="email" placeholder="name@example.com" />
          </div>
          <div className="grid gap-2">
            <Label htmlFor="password">비밀번호</Label>
            <Input id="password" type="password" />
          </div>
        </div>
        <DialogFooter>
          <Button type="submit">로그인</Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  );
}
```

### 8. 다단계 폼 다이얼로그

```tsx
import { useState } from 'react';
import {
  Dialog,
  DialogContent,
  DialogDescription,
  DialogFooter,
  DialogHeader,
  DialogTitle,
  DialogTrigger,
} from '@framingui/ui';
import { Button, Input, Label } from '@framingui/ui';

export function MultiStepDialog() {
  const [step, setStep] = useState(1);

  return (
    <Dialog>
      <DialogTrigger asChild>
        <Button>회원가입</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>계정 만들기 - {step}/2단계</DialogTitle>
          <DialogDescription>
            {step === 1 ? '개인 정보를 입력하세요' : '비밀번호를 선택하세요'}
          </DialogDescription>
        </DialogHeader>
        <div className="grid gap-4 py-4">
          {step === 1 ? (
            <>
              <div className="grid gap-2">
                <Label htmlFor="fullname">이름</Label>
                <Input id="fullname" placeholder="홍길동" />
              </div>
              <div className="grid gap-2">
                <Label htmlFor="email">이메일</Label>
                <Input id="email" type="email" placeholder="hong@example.com" />
              </div>
            </>
          ) : (
            <>
              <div className="grid gap-2">
                <Label htmlFor="password">비밀번호</Label>
                <Input id="password" type="password" />
              </div>
              <div className="grid gap-2">
                <Label htmlFor="confirm">비밀번호 확인</Label>
                <Input id="confirm" type="password" />
              </div>
            </>
          )}
        </div>
        <DialogFooter>
          {step === 2 && (
            <Button variant="outline" onClick={() => setStep(1)}>
              이전
            </Button>
          )}
          <Button onClick={() => step === 1 ? setStep(2) : null}>
            {step === 1 ? '다음' : '계정 만들기'}
          </Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  );
}
```

## 디자인 토큰

Dialog 컴포넌트는 다음 Tailwind 유틸리티 클래스를 사용한다:

**DialogOverlay:**
- 배경: `bg-black/80`
- 백드롭: `backdrop-blur-sm`
- 위치: `fixed inset-0 z-50`
- 애니메이션: `data-[state=open]:animate-in data-[state=closed]:animate-out`

**DialogContent:**
- 배경: `bg-white`
- 테두리: `border border-neutral-200`
- 그림자: `shadow-lg`
- 패딩: `p-6`
- 모서리: `sm:rounded-lg`
- 최대 너비: `max-w-lg`
- 위치: `fixed left-[50%] top-[50%] translate-x-[-50%] translate-y-[-50%]`
- 애니메이션: `zoom-in-95`, `zoom-out-95`, `fade-in-0`, `fade-out-0`

**DialogTitle:**
- 폰트: `text-lg font-semibold leading-none tracking-tight`

**DialogDescription:**
- 색상: `text-sm text-neutral-500`

**DialogHeader:**
- 레이아웃: `flex flex-col space-y-2 text-center sm:text-left`

**DialogFooter:**
- 레이아웃: `flex flex-col-reverse sm:flex-row sm:justify-end sm:space-x-2`

## 사용 팁

- 접근성을 위해 `DialogTitle`을 항상 포함한다 (스크린 리더용).
- `DialogDescription`으로 다이얼로그 목적에 대한 맥락을 제공한다.
- 위험한 작업에는 명확한 취소 옵션이 있는 확인 다이얼로그를 사용한다.
- 다이얼로그 콘텐츠는 단일 작업이나 결정에 집중한다.
- 프로그래밍 방식으로 다이얼로그를 제어해야 할 때는 제어된 상태(`open` + `onOpenChange`)를 사용한다.
- 더 나은 접근성을 위해 닫기 버튼에 적절한 `aria-label`을 추가한다.
- 다이얼로그 안에 다른 다이얼로그를 중첩하지 않는다.
- 복잡한 폼은 여러 단계로 나누는 것을 고려한다.
- `DialogFooter`를 사용하여 액션 버튼을 일관되게 정렬한다.
- 키보드 탐색이 제대로 작동하는지 확인한다 (Escape로 닫기, Tab으로 이동).
