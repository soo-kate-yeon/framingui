# SPEC-UI-002 레퍼런스 가이드

**Primary Reference:** Claude.ai (https://claude.ai)

Claude.ai의 디자인 철학을 기반으로 12개 템플릿을 정의합니다.

---

## Claude.ai 디자인 특징 분석

### 핵심 디자인 원칙
1. **명료함 (Clarity)**: 불필요한 장식 없이 기능에 집중
2. **접근성 (Accessibility)**: 높은 대비, 명확한 계층 구조
3. **범용성 (Universality)**: 다양한 사용자층이 편안하게 사용 가능
4. **절제된 색상**: 주로 중성 색상, 강조색은 최소한으로

### 색상 팔레트
- **배경**: 밝은 회색/흰색 (light mode), 진한 회색 (dark mode)
- **Primary 컬러**: 블랙 (범용적이고 전문적인 느낌)
- **강조색**: 최소한으로 사용 (필요 시에만)
- **텍스트**: 높은 대비 (검정/흰색)

### 타이포그래피
- **폰트**: Sans-serif (깔끔하고 읽기 쉬운)
- **크기**: 명확한 계층 (제목 → 본문 → 캡션)
- **행간**: 넉넉함 (가독성 우선)

### 레이아웃
- **여백**: 충분한 공간 (답답하지 않음)
- **정렬**: 좌측 정렬 기본, 중앙 정렬은 특정 상황만
- **그리드**: 명확한 구조

## 템플릿별 레퍼런스

### Auth (인증)

#### auth.login
**Claude.ai 참고 화면:** 로그인 페이지
**URL:** https://claude.ai/login

**디자인 원칙:**
1. 중앙 정렬 카드 레이아웃
2. 로고 → 제목 → 입력 필드 → 버튼 순서
3. 입력 필드: 높이 48px, 둥근 모서리 (radius-md)
4. 여백: 요소 간 spacing-4, 섹션 간 spacing-8

**필수 요소:**
- 이메일/비밀번호 입력 필드
- 로그인 버튼 (전체 너비)
- 로고 또는 서비스 이름

**선택 요소:**
- 소셜 로그인 (Google, GitHub 등)
- "비밀번호 찾기" 링크
- "회원가입" 링크
- "자동 로그인" 체크박스

**금지 사항:**
- 화려한 배경 이미지
- 여러 개의 CTA 버튼
- 과도한 설명 텍스트

---

#### auth.signup
**Claude.ai 참고 화면:** 회원가입 페이지

**디자인 원칙:**
1. auth.login과 동일한 레이아웃 구조
2. 추가 입력 필드 (이름, 비밀번호 확인)
3. 약관 동의 체크박스

**필수 요소:**
- 이름/이메일/비밀번호/비밀번호 확인
- 회원가입 버튼
- 약관 동의 체크박스

**선택 요소:**
- 소셜 회원가입
- "이미 계정이 있으신가요?" 링크

**금지 사항:**
- 한 화면에 너무 많은 입력 필드 (5개 초과)
- 복잡한 비밀번호 규칙 표시

---

#### auth.forgot-password
**Claude.ai 참고 화면:** 비밀번호 재설정

**디자인 원칙:**
1. 간결한 설명 + 이메일 입력
2. 안내 메시지는 명확하고 친절하게

**필수 요소:**
- 이메일 입력 필드
- "재설정 링크 보내기" 버튼
- 안내 텍스트

**선택 요소:**
- "로그인으로 돌아가기" 링크

**금지 사항:**
- 불필요한 보안 질문

---

#### auth.verification
**Claude.ai 참고 화면:** 이메일 인증

**디자인 원칙:**
1. 중앙 정렬 메시지
2. 명확한 다음 단계 안내

**필수 요소:**
- 인증 상태 메시지
- 다음 단계 버튼

**선택 요소:**
- "인증 메일 재전송" 버튼
- 이메일 주소 표시

**금지 사항:**
- 복잡한 인증 코드 입력 UI

### Core (핵심)

#### home.landing
**Claude.ai 참고 화면:** 메인 대시보드

**디자인 원칙:**
1. 좌측 사이드바 + 메인 컨텐츠 영역
2. 최근 활동/추천 항목 표시
3. 명확한 CTA (새 대화 시작 등)

**필수 요소:**
- 사이드바 네비게이션
- 메인 컨텐츠 영역
- CTA 버튼

**선택 요소:**
- 최근 활동 목록
- 추천/제안 섹션

**금지 사항:**
- 과도한 애니메이션
- 복잡한 그래프/차트

---

#### settings.preferences
**Claude.ai 참고 화면:** 설정 페이지

**디자인 원칙:**
1. 좌측 설정 메뉴 + 우측 설정 패널
2. 섹션별 명확한 구분
3. 토글/선택 UI 일관성

**필수 요소:**
- 설정 카테고리 네비게이션
- 설정 항목 (Switch, Select 등)
- 저장 버튼

**선택 요소:**
- 설정 검색
- 기본값으로 되돌리기

**금지 사항:**
- 한 화면에 모든 설정 나열

---

#### account.profile
**Claude.ai 참고 화면:** 프로필 페이지

**디자인 원칙:**
1. 프로필 이미지 + 기본 정보
2. 편집 가능한 필드 명확히 표시

**필수 요소:**
- 프로필 이미지 (Avatar)
- 이름/이메일 표시
- 편집 버튼

**선택 요소:**
- 프로필 이미지 변경
- 추가 정보 (소개, 링크 등)

**금지 사항:**
- 복잡한 프로필 커스터마이징

## 구현 시 주의사항

### 토큰 사용
- 모든 색상: `var(--tekton-bg-*)`
- 모든 간격: `var(--tekton-spacing-*)`
- 모든 둥근 모서리: `var(--tekton-radius-*)`

### 반응형
- Desktop: 1024px 이상
- Tablet: 768px ~ 1024px
- Mobile: 768px 미만

### 접근성
- 모든 인터랙티브 요소: 최소 44x44px
- 색상 대비: WCAG AA 이상
- 키보드 네비게이션 지원
