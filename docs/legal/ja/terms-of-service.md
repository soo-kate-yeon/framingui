## 2. 定義

- **「サービス」**: Tekton プラットフォーム、その Web サイト、MCP ツール、アカウント画面、ランタイムパッケージ、ドキュメント、および関連ダウンロード資産
- **「ツールユニット」**: MCP ツールの利用量を quota、top-up、従量課金の基準として測定する加重使用単位
- **「プラン」**: 月間ツールユニット、機能アクセス、サポート条件などを含みうる無料または有料の entitlement パッケージ
- **「トップアップ」**: 定期プランとは別に購入する追加のツールユニット
- **「レガシー資産」**: 旧テンプレートライセンスモデルで購入したテーマ、テンプレート、ダウンロードパッケージ
- **「アウトプット」**: サービスから生成または返却されるコード、設定、提案、検証結果など
- **「アカウント」**: 本サービスで登録したユーザーアカウント

## 4. サービスの範囲

### 4.1 Tekton が提供するもの

Tekton は次を提供します。

- discovery、validation、integration を含む MCP ベースの UI ワークフロー
- デザインシステムのランタイム、トークン、テーマ、関連パッケージ
- 使用量、entitlement、shadow billing、請求状態を表示しうるアカウント画面
- 旧プランで購入されたレガシー資産

### 4.2 提供方法

本サービスは、ホスト型アカウント画面、MCP/API アクセス、ダウンロードパッケージを通じてデジタル提供されます。物理商品の発送はありません。

### 4.3 更新と可用性

- 有効なプランには、購入時の pricing page または checkout に表示された範囲でアクセス、更新、サポートが提供されます。
- レガシー資産は、別途 migration policy が明示されない限り、購入当時に案内された更新・サポート期間に従います。
- 移行期間中は、実際の課金開始前に shadow billing や quota 予測が表示される場合があります。

## 5. 価格、課金、支払い

### 5.1 価格モデル

価格、含まれるツールユニット、更新周期、トップアップ可否、超過利用条件は、購入時点の pricing page と checkout に表示された内容に従います。

### 5.2 Shadow Billing

当社は、実課金の開始前に推定使用量または shadow usage を表示することがあります。pricing page、checkout、または製品内通知で実課金開始が明示されない限り、shadow usage は参考情報であり請求額ではありません。

### 5.3 決済処理

すべての支払いは **Paddle** を通じて処理されます。購入時は Paddle の関連規約にも同意したものとみなします。

### 5.4 サブスクリプション条件

- 定期プランは更新日前に解約しない限り自動更新されます。
- 解約は billing 画面または当社への連絡で行えます。
- 法令で別段の定めがない限り、解約は現在の請求期間の終了時に効力を生じます。

### 5.5 通貨と税金

価格は checkout に表示された通貨で決済され、適用税は Paddle が所在地に基づいて計算・徴収します。

ユーザーは次の行為をしてはなりません。

旧テンプレートライセンスモデルで購入したレガシー資産の利用権は、追加の migration policy が示されない限り、購入当時の条件に従って維持されます。
